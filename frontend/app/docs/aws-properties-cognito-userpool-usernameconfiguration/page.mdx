# AWS::Cognito::UserPool 사용자 이름 구성

`UsernameConfiguration` 속성 유형은 선택한 로그인 옵션에 대한 사용자 이름 입력에 대한 대/소문자 구분을 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "CaseSensitive" : Boolean
}
```

## 속성

`CaseSensitive`  
Amazon Cognito API를 통해 사용자 풀의 모든 사용자에 대해 사용자 이름 대/소문자 구분을 적용할지 여부를 지정합니다. 대부분의 사용 사례에서는 대소문자 구분 `False` 을 (대소문자 구분 안 함)으로 설정하는 것이 좋습니다. 사용자 이름과 이메일 주소가 대소문자를 구분하지 않는 경우 사용자는 사용자 이름의 다른 대문자를 입력할 때 동일한 사용자로 로그인할 수 있습니다.  
유효한 값은 다음과 같습니다.  
참  
모든 사용자 이름 입력에 대해 대소문자를 구분할 수 있습니다. 이 옵션을 `True`로 설정하면 사용자는 지정된 사용자 이름의 정확한 대문자(예: "UserName")를 사용하여 로그인해야 합니다. 이 값은 기본값입니다.  
거짓  
모든 사용자 이름 입력에 대해 대소문자를 구분하지 않도록 합니다. 예를 들어 이 옵션을 로 설정하면 사용자는 `False`, `username`, 또는 `USERNAME` 를 사용하여 로그인할 수 있습니다`UserName`. 또한 이 옵션을 사용하면 속성 외에도 `preferred_username` 및 `email` 별칭이 대소문자를 구분하지 않도록 할 수 있습니다 `username` .
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음
