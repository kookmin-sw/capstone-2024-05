# AWS::SageMaker::ModelPackage 변환 리소스

ML 인스턴스 유형 및 ML 인스턴스 수를 포함하여 변환 작업에 사용할 리소스에 대해 설명합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "InstanceCount" : Integer,
  "InstanceType" : String,
  "VolumeKmsKeyId" : String
}
```

## 속성

`InstanceCount`  
변환 작업에 사용할 ML 컴퓨팅 인스턴스의 수입니다. 기본값은 `1`이고 최대값은 `100`입니다. 분산 변환 작업의 경우 보다 큰 값을 지정합니다 `1`.  
_필수 항목 여부_: 예  
_유형_: Integer  
_최소_: `1`  
_업데이트 필요 사항_: 바꿔 놓음

`InstanceType`  
변환 작업에 대한 ML 컴퓨팅 인스턴스 유형입니다. 기본 제공 알고리즘을 사용하여 중간 크기의 데이터 세트를 변환하는 경우 ml.m4.xlarge 또는 인스턴스 유형을 사용하는 것이 좋습니다 `ml.m5.large`.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `ml.c4.2xlarge | ml.c4.4xlarge | ml.c4.8xlarge | ml.c4.xlarge | ml.c5.18xlarge | ml.c5.2xlarge | ml.c5.4xlarge | ml.c5.9xlarge | ml.c5.xlarge | ml.g4dn.12xlarge | ml.g4dn.16xlarge | ml.g4dn.2xlarge | ml.g4dn.4xlarge | ml.g4dn.8xlarge | ml.g4dn.xlarge | ml.m4.10xlarge | ml.m4.16xlarge | ml.m4.2xlarge | ml.m4.4xlarge | ml.m4.xlarge | ml.m5.12xlarge | ml.m5.24xlarge | ml.m5.2xlarge | ml.m5.4xlarge | ml.m5.large | ml.m5.xlarge | ml.p2.16xlarge | ml.p2.8xlarge | ml.p2.xlarge | ml.p3.16xlarge | ml.p3.2xlarge | ml.p3.8xlarge`  
_업데이트 필요 사항_: 바꿔 놓음

`VolumeKmsKeyId`  
Amazon SageMaker가 배치 변환 작업을 실행하는 ML 컴퓨팅 인스턴스에 연결된 스토리지 볼륨의 모델 데이터를 암호화하는 데 사용하는 AWS Key Management Service(AWS KMS) 키입니다.  
특정 Nitro 기반 인스턴스에는 인스턴스 유형에 따라 로컬 스토리지가 포함됩니다. 로컬 스토리지 볼륨은 인스턴스의 하드웨어 모듈을 사용하여 암호화됩니다. `VolumeKmsKeyId` 로컬 스토리지와 함께 인스턴스 유형을 사용하는 경우에는 요청할 수 없습니다.  
로컬 인스턴스 스토리지를 지원하는 인스턴스 유형 목록은 인스턴스 스토어 볼륨 단원을 참조하십시오.  
로컬 인스턴스 스토리지 암호화에 대한 자세한 내용은 SSD 인스턴스 스토어 볼륨 단원을 참조하십시오.
다음 `VolumeKmsKeyId` 형식 중 하나일 수 있습니다.

- 키 ID: `1234abcd-12ab-34cd-56ef-1234567890ab`
- 키 ARN: `arn:aws:kms:us-west-2:111122223333:key/1234abcd-12ab-34cd-56ef-1234567890ab`
- 별칭 이름: `alias/ExampleAlias`
- 별칭 이름 ARN: `arn:aws:kms:us-west-2:111122223333:alias/ExampleAlias`
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _최대_: `2048`  
  _패턴_: `.*`  
  _업데이트 필요 사항_: 바꿔 놓음
