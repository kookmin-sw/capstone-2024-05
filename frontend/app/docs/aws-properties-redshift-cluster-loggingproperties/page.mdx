# AWS::Redshift::클러스터 로깅 속성

지정된 Amazon Redshift 클러스터에 대한 로깅 정보(예: 쿼리 및 연결 시도)를 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "BucketName" : String,
  "S3KeyPrefix" : String
}
```

## 속성

`BucketName` 로그 파일을 저장할 기존 S3 버킷의 이름입니다.  
제약 조건:

- 클러스터와 동일한 지역에 있어야 합니다.
- 클러스터에는 read bucket 및 put object 권한이 있어야 합니다.
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _최대_: `2147483647`  
  _업데이트 필요 사항_: 중단 없음

`S3KeyPrefix` 로그 파일 이름에 적용되는 접두사입니다.  
제약 조건:

- 512자를 초과할 수 없습니다.
- 공백( ), 큰따옴표("), 작은따옴표('), 백슬래시() 또는 제어 문자를 포함할 수 없습니다. 유효하지 않은 문자에 대한 16진수 코드는 다음과 같습니다.
  - x00에서 x20으로
  - 22개
  - 27대
  - x5c
  - x7f 이상
    *필수 항목 *여부: 아니요  
    _유형_: 문자열  
    _최대_: `2147483647`  
    _업데이트 필요 사항_: 중단 없음
