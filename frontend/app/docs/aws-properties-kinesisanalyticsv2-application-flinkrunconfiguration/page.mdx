# AWS::KinesisAnalyticsV2::애플리케이션 FlinkRunConfiguration

Managed Service for Apache Flink 애플리케이션의 시작 매개변수에 대해 설명합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AllowNonRestoredState" : Boolean
}
```

## 속성

`AllowNonRestoredState`  
스냅숏에서 복원할 때 런타임에서 새 프로그램에 매핑할 수 없는 상태를 건너뛸 수 있는지 여부를 지정합니다. 이 문제는 상태 저장 매개 변수를 제거하기 위해 스냅숏 간에 프로그램이 업데이트되고 스냅숏의 상태 데이터가 더 이상 유효한 응용 프로그램 데이터에 해당하지 않는 경우에 발생합니다. 자세한 정보는 Apache Flink 문서의 복원되지 않은 상태 허용을 참조하십시오.  
기본값은 입니다`false`. 이 매개 변수를 지정하지 않고 응용 프로그램을 업데이트하면 `AllowNonRestoredState`이전에 로 설정되었더라도 가 로 `false` 설정됩니다`true`.
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음
