# AWS::AutoScaling::ScalingPolicy PredictiveScalingPredefinedLoadMetric

AWS::AutoScaling::ScalingPolicy PredictiveScalingMetricSpecification 속성 유형의 속성에`PredefinedLoadMetricSpecification` 대한 로드 지표 정보를 포함합니다.

**중요하다**

- 지표 사양에\* 지표 쌍을 사용하는 정책에는 적용되지 않습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "PredefinedMetricType" : String,
  "ResourceLabel" : String
}
```

## 속성

`PredefinedMetricType`  
측정항목 유형입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `ALBTargetGroupRequestCount | ASGTotalCPUUtilization | ASGTotalNetworkIn | ASGTotalNetworkOut`  
_업데이트 필요 사항_: 중단 없음

`ResourceLabel`  
오토 스케일링에서 제공하는 요청 수를 확인할 특정 Application Load Balancer 대상 그룹을 고유하게 식별하는 레이블입니다. 대상 그룹이 Auto Scaling 그룹에 연결되어 있지 않으면 리소스 레이블을 지정할 수 없습니다.  
로드 밸런서 ARN의 마지막 부분과 대상 그룹 ARN의 마지막 부분을 슬래시(/)로 구분된 단일 값에 추가하여 리소스 레이블을 생성합니다. 리소스 레이블의 형식은 다음과 같습니다.  
 `app/my-alb/778d41231b141a0f/targetgroup/my-alb-target-group/943f017f100becff`을 클릭합니다.  
어디:

- app/&lt;load-balancer-name>/&lt;load-balancer-id>는 로드 밸런서 ARN의 마지막 부분입니다.
- targetgroup/&lt;target-group-name>/&lt;target-group-id> 은 대상 그룹 ARN의 마지막 부분입니다.
  Application Load Balancer의 ARN을 찾으려면 DescribeLoadBalancers API 작업을 사용합니다. 대상 그룹의 ARN을 찾으려면 DescribeTargetGroups API 작업을 사용합니다.  
  _필수_: 조건부  
  _유형_: 문자열  
  _업데이트 필요 사항_: 중단 없음
