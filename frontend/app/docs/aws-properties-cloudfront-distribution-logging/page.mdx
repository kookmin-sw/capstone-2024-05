# AWS::CloudFront::D istribution 로깅

배포에 대한 액세스 로그를 기록할지 여부를 제어하는 복합 형식입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Bucket" : String,
  "IncludeCookies" : Boolean,
  "Prefix" : String
}
```

## 속성

`Bucket`  
액세스 로그를 저장할 Amazon S3 버킷(예: `myawslogbucket.s3.amazonaws.com`.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`IncludeCookies`  
CloudFront에서 액세스 로그에 쿠키를 포함할지 여부를 지정하고, `true` 에 대해 `IncludeCookies` 지정합니다. 로그에 쿠키를 포함하도록 선택하는 경우, CloudFront에서는 이 배포에 대한 캐시 동작을 구성하는 방법에 관계없이 모든 쿠키를 로깅합니다. 배포를 생성할 때 쿠키를 포함하지 않으려는 경우 또는 기존 배포에 대해 쿠키 포함을 비활성화하려면 `false` 를 지정합니다`IncludeCookies`.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Prefix`  
CloudFront에서 이 배포의 액세스 로그 앞에 접두사로 붙일 선택적 문자열 `filenames` (예: `myprefix/`). 로깅을 사용하도록 설정하되 접두사를 지정하지 않으려는 경우에도 요소에 빈 요소를 포함해야 합니다 `Prefix` `Logging` .  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
