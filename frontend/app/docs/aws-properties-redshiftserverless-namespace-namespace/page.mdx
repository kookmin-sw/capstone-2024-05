# AWS::RedshiftServerless::네임스페이스 네임스페이스

데이터베이스 개체 및 사용자의 컬렉션입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AdminUsername" : String,
  "CreationDate" : String,
  "DbName" : String,
  "DefaultIamRoleArn" : String,
  "IamRoles" : [ String, ... ],
  "KmsKeyId" : String,
  "LogExports" : [ String, ... ],
  "NamespaceArn" : String,
  "NamespaceId" : String,
  "NamespaceName" : String,
  "Status" : String
}
```

## 속성

`AdminUsername`  
네임스페이스에서 만든 첫 번째 데이터베이스에 대한 관리자의 사용자 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`CreationDate`  
네임스페이스를 만든 날짜입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DbName`  
네임스페이스에서 만든 첫 번째 데이터베이스의 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DefaultIamRoleArn`  
네임스페이스에서 기본값으로 설정할 IAM 역할의 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`IamRoles`  
네임스페이스와 연결할 IAM 역할 목록입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`KmsKeyId`  
데이터를 암호화하는 데 사용되는 AWS Key Management Service 키의 ID입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`LogExports`  
네임스페이스에서 내보낼 수 있는 로그의 유형입니다. 사용 가능한 내보내기 유형은 사용자 로그, 연결 로그 및 사용자 활동 로그입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`NamespaceArn`  
네임스페이스와 연결된 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`NamespaceId`  
네임스페이스의 고유 식별자입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`NamespaceName`  
네임스페이스의 이름입니다. 소문자로 된 3-64자의 영숫자여야 하며 예약어일 수 없습니다. 예약어 목록은 Amazon Redshift 데이터베이스 개발자 안내서의 예약어에서 확인할 수 있습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Status`  
네임스페이스의 상태입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
