# AWS::EC2::SpotFleet 블록 디바이스 매핑

블록 장치 매핑을 지정합니다.

또는 를 `Ebs`지정할 수 `VirtualName`있지만 둘 다 지정할 수는 없습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "DeviceName" : String,
  "Ebs" : EbsBlockDevice,
  "NoDevice" : String,
  "VirtualName" : String
}
```

## 속성

`DeviceName`  
장치 이름(예: `/dev/sdh` 또는 `xvdh`).  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Ebs`  
인스턴스가 시작될 때 EBS 볼륨을 자동으로 설정하는 데 사용되는 파라미터입니다.  
_필수_: 조건부  
_유형_: EbsBlockDevice  
_업데이트 필요 사항_: 바꿔 놓음

`NoDevice`  
블록 장치 매핑에서 장치를 생략하려면 빈 문자열을 지정합니다. 이 특성을 지정하면, 장치는 지정된 값에 관계없이 블록 장치 맵핑에서 제거됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`VirtualName`  
가상 장치 이름(`ephemeral`N)입니다. 인스턴스 스토어 볼륨은 0부터 시작하여 번호가 매겨집니다. 2개의 사용 가능한 인스턴스 스토어 볼륨이 있는 인스턴스 유형은 및 에 대한 매핑을 지정할 수 있습니다 `ephemeral0` `ephemeral1`. 사용 가능한 인스턴스 스토어 볼륨의 수는 인스턴스 유형에 따라 다릅니다. 인스턴스에 연결한 후에는 볼륨을 탑재해야 합니다.  
NVMe 인스턴스 스토어 볼륨은 자동으로 열거되고 장치 이름이 할당됩니다. 블록 장치 매핑에 포함해도 아무런 효과가 없습니다.  
제약 조건: M3 인스턴스의 경우 인스턴스에 대한 블록 장치 매핑에서 인스턴스 저장소 볼륨을 지정해야 합니다. M3 인스턴스를 시작할 때 AMI에 대한 블록 디바이스 매핑에 지정된 모든 인스턴스 스토어 볼륨을 무시합니다.  
_필수_: 조건부  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음
