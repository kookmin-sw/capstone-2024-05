# AWS::람다::URL 코어

함수 URL에 대한 CORS(Cross-Origin Resource Sharing) 설정입니다. CORS를 사용하여 모든 오리진에서 함수 URL에 대한 액세스 권한을 부여합니다. 또한 CORS를 사용하여 함수 URL에 대한 요청에서 특정 HTTP 헤더 및 메서드에 대한 액세스를 제어할 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AllowCredentials" : Boolean,
  "AllowHeaders" : [ String, ... ],
  "AllowMethods" : [ String, ... ],
  "AllowOrigins" : [ String, ... ],
  "ExposeHeaders" : [ String, ... ],
  "MaxAge" : Integer
}
```

## 속성

`AllowCredentials`  
함수 URL에 대한 요청에서 쿠키 또는 기타 자격 증명을 허용할지 여부입니다. 기본값은 `false`입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`AllowHeaders`  
오리진이 함수 URL에 대한 요청에 포함할 수 있는 HTTP 헤더입니다. 예: `Date`, `Keep-Alive`, `X-Custom-Header`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_최대_: `100`  
_업데이트 필요 사항_: 중단 없음

`AllowMethods`  
함수 URL을 호출할 때 허용되는 HTTP 메서드입니다. 예를 들어, `GET`, `POST`, `DELETE`또는 와일드카드 문자()가 있습니다`*`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_최대_: `6`  
_업데이트 필요 사항_: 중단 없음

`AllowOrigins`  
함수 URL에 액세스할 수 있는 오리진입니다. 특정 출처를 쉼표로 구분하여 원하는 수만큼 나열할 수 있습니다. 예: `some-address `some-address  
또는 와일드카드 문자()를 사용하여 모든 원본에 대한 액세스 권한을 부여할 수 있습니다`*`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_최대_: `100`  
_업데이트 필요 사항_: 중단 없음

`ExposeHeaders`  
함수 URL을 호출하는 오리진에 노출하려는 함수 응답의 HTTP 헤더입니다. 예: `Date`, `Keep-Alive`, `X-Custom-Header`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_최대_: `100`  
_업데이트 필요 사항_: 중단 없음

`MaxAge`  
브라우저가 실행 전 요청의 결과를 캐시할 수 있는 최대 시간(초)입니다. 기본적으로 이 값은 로 설정되어 있으며, `0`이는 브라우저가 결과를 캐시하지 않음을 의미합니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_최소_: `0`  
_최대_: `86400`  
_업데이트 필요 사항_: 중단 없음
