# AWS::CloudFront::D istribution DistributionConfig

배포 구성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Aliases" : [ String, ... ],
  "CacheBehaviors" : [ CacheBehavior, ... ],
  "CNAMEs" : [ String, ... ],
  "Comment" : String,
  "ContinuousDeploymentPolicyId" : String,
  "CustomErrorResponses" : [ CustomErrorResponse, ... ],
  "CustomOrigin" : LegacyCustomOrigin,
  "DefaultCacheBehavior" : DefaultCacheBehavior,
  "DefaultRootObject" : String,
  "Enabled" : Boolean,
  "HttpVersion" : String,
  "IPV6Enabled" : Boolean,
  "Logging" : Logging,
  "OriginGroups" : OriginGroups,
  "Origins" : [ Origin, ... ],
  "PriceClass" : String,
  "Restrictions" : Restrictions,
  "S3Origin" : LegacyS3Origin,
  "Staging" : Boolean,
  "ViewerCertificate" : ViewerCertificate,
  "WebACLId" : String
}
```

## 속성

*필수 항목 *여부: 아니요  
_형식_: CacheBehavior 목록  
_업데이트 필요 사항_: 중단 없음

`CustomErrorResponses` 다음을 제어하는 복합 형식입니다.

- CloudFront가 최종 사용자에게 응답을 반환하기 전에 4xx 및 5xx 범위의 HTTP 상태 코드를 사용자 지정 오류 메시지로 바꾸는지 여부입니다.
- CloudFront에서 4xx 및 5xx 범위의 HTTP 상태 코드를 캐싱하는 기간입니다.
  사용자 지정 오류 페이지에 대한 자세한 내용은 _Amazon CloudFront 개발자 안내서의 오류 응답 사용자 지정을 참조하십시오_.  
  *필수 항목 *여부: 아니요  
  _유형_: CustomErrorResponse 목록  
  _업데이트 필요 사항_: 중단 없음

_형식_: LegacyCustomOrigin  
_업데이트 필요 사항_: 중단 없음

`DefaultCacheBehavior`  
요소를 지정하지 않거나 파일이 `CacheBehavior` in 요소의 `PathPattern` 값과 일치하지 않는 경우 `CacheBehavior` 기본 캐시 동작을 설명하는 복합 형식입니다 . 기본 캐시 동작을 하나만 만들어야 합니다.  
_필수 항목 여부_: 예  
_형식_: DefaultCacheBehavior  
_업데이트 필요 사항_: 중단 없음

`DefaultRootObject`  
`index.html`최종 사용자가 배포의 루트 URL을 요청할 때`some-address instead of an object in your distribution (` CloudFront가 오리진에서 요청하도록 할 객체(예: )입니다(some-address 기본 루트 객체를 지정하면 배포 콘텐츠가 노출되는 것을 방지할 수 있습니다.  
객체 이름만 지정합니다(예: `index.html`. 개체 이름 앞에 a를 추가하지 마세요 `/` .  
배포를 생성할 때 기본 루트 객체를 지정하지 않으려면 빈 `DefaultRootObject` 요소를 포함합니다.  
기존 배포에서 기본 루트 객체를 삭제하려면 배포 구성을 업데이트하고 빈 요소를 포함합니다 `DefaultRootObject` .  
기본 루트 객체를 바꾸려면 배포 구성을 업데이트하고 새 객체를 지정합니다.  
기본 루트 객체에 대한 자세한 내용은 _Amazon CloudFront 개발자 안내서의 기본 루트 객체 생성을 참조하십시오_.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Enabled`  
이 필드에서 선택한 배포를 활성화하거나 비활성화할 수 있습니다.  
_필수 항목 여부_: 예  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`HttpVersion`  
(선택 사항) 최종 사용자가 CloudFront와 통신하는 데 사용할 최대 HTTP 버전을 지정합니다. 새 배포의 기본값은 `http1.1`입니다.  
최종 사용자와 CloudFront에서 HTTP/2를 사용하려면 최종 사용자가 TLSv1.2 이상을 지원해야 하며 SNI(서버 이름 표시)를 지원해야 합니다.  
최종 사용자와 CloudFront에서 HTTP/3를 사용하려면 최종 사용자가 TLSv1.3 및 SNI(서버 이름 표시)를 지원해야 합니다. CloudFront는 최종 사용자가 연결 끊김 없이 네트워크를 전환할 수 있도록 HTTP/3 연결 마이그레이션을 지원합니다. 연결 마이그레이션에 대한 자세한 내용은 RFC 9000의 연결 마이그레이션을 참조하십시오. 지원되는 TLSv1.3 암호에 대한 자세한 내용은 최종 사용자와 CloudFront 간에 지원되는 프로토콜 및 암호 섹션을 참조하세요.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `http1.1 | http2 | http2and3 | http3`  
_업데이트 필요 사항_: 중단 없음

`IPV6Enabled`  
CloudFront에서 배포의 IPv6 주소를 사용하여 IPv6 DNS 요청에 응답하도록 하려면 를 지정합니다`true`. 를 지정할 경우`false`, CloudFront는 IP 주소 없이 DNS 응답 코드를 사용하여 IPv6 DNS 요청에 응답 `NOERROR` 합니다 . 이렇게 하면 최종 사용자가 배포의 IPv4 주소에 대한 두 번째 요청을 제출할 수 있습니다.  
일반적으로 IPv6 네트워크에 콘텐츠에 액세스하려는 사용자가 있는 경우 IPv6을 사용하도록 설정해야 합니다. 그러나 서명된 URL 또는 서명된 쿠키를 사용하여 콘텐츠에 대한 액세스를 제한하고 `IpAddress` 콘텐츠에 액세스할 수 있는 IP 주소를 제한하는 파라미터가 포함된 사용자 지정 정책을 사용하는 경우 IPv6를 활성화하지 마십시오. 일부 콘텐츠에 대한 액세스를 IP 주소로 제한하고 다른 콘텐츠에 대한 액세스는 제한하지 않으려는 경우(또는 IP 주소로는 제한하지 않고 액세스를 제한하려는 경우) 두 개의 배포를 생성할 수 있습니다. 자세한 내용은 Amazon CloudFront 개발자 안내서의 사용자 지정 정책을 사용하여 서명된 URL 생성을 참조하십시오\*\*.  
Amazon Route 53 AWS 통합 별칭 리소스 레코드 세트를 사용하여 CloudFront 배포로 트래픽을 라우팅하는 경우, 다음 두 가지 모두에 해당하면 두 번째 별칭 리소스 레코드 세트를 생성해야 합니다.

- 배포에 대해 IPv6를 사용하도록 설정합니다
- 객체의 URL에 대체 도메인 이름을 사용하고 있습니다.
  자세한 내용은 * Amazon Route 53 AWS 통합 개발자 안내서*의 도메인 이름을 사용하여 Amazon CloudFront 웹 배포로 트래픽 라우팅을 참조하십시오.  
  Amazon Route 53 AWS 통합 또는 다른 DNS 서비스를 사용하여 CNAME 리소스 레코드 세트를 생성한 경우 변경할 필요가 없습니다. CNAME 레코드는 최종 사용자 요청의 IP 주소 형식에 관계없이 트래픽을 배포로 라우팅합니다.  
  *필수 항목 *여부: 아니요  
  _유형_: Boolean  
  _업데이트 필요 사항_: 중단 없음

로깅에 대한 자세한 내용은 _Amazon CloudFront 개발자 안내서의 액세스 로그를 참조하십시오_.  
*필수 항목 *여부: 아니요  
_유형_: 로깅  
_업데이트 필요 사항_: 중단 없음

`PriceClass`  
CloudFront 서비스에 대해 지불하려는 최고 가격에 해당하는 가격 등급입니다. 를 지정하는 경우, `PriceClass_All`CloudFront는 모든 CloudFront 엣지 로케이션의 객체에 대한 요청에 응답합니다.  
가 아닌 다른 가격 등급을 지정하는 경우`PriceClass_All`, CloudFront는 가격 등급의 엣지 로케이션 중 지연 시간이 가장 짧은 CloudFront 엣지 로케이션에서 객체를 제공합니다. 지정된 가격 등급에서 제외된 리전에 있거나 그 근처에 있는 최종 사용자는 성능이 저하될 수 있습니다.  
가격 등급에 대한 자세한 내용은 Amazon CloudFront 개발자 안내서의 CloudFront 배포에 대한 가격 등급 선택을 참조하십시오\**. 요금 등급(예: 가격 등급 100)이 CloudFront 리전에 매핑되는 방식을 포함하여 CloudFront 요금에 대한 자세한 내용은 Amazon CloudFront 요금을 참조하십시오.  
*필수 항목 *여부: 아니요  
*유형*: 문자열  
*허용되는 값*: `PriceClass_100 | PriceClass_200 | PriceClass_All`  
*업데이트 필요 사항\*: 중단 없음

`WebACLId`  
이 배포와 연결할 AWS WAF 웹 ACL(있는 경우)을 지정하는 고유 식별자입니다. 최신 버전의 AWS WAF를 사용하여 생성된 웹 ACL을 지정하려면 ACL ARN을 사용합니다(예: `arn:aws:wafv2:us-east-1:123456789012:global/webacl/ExampleWebACL/473e64fd-f30b-4765-81a0-62ad96dd167a`. AWS WAF Classic을 사용하여 생성된 웹 ACL을 지정하려면 ACL ID(예: )를 사용합니다`473e64fd-f30b-4765-81a0-62ad96dd167a`.  
 AWS WAF는 CloudFront로 전달되는 HTTP 및 HTTPS 요청을 모니터링하고 콘텐츠에 대한 액세스를 제어할 수 있는 웹 애플리케이션 방화벽입니다. 요청이 시작되는 IP 주소 또는 쿼리 문자열의 값과 같이 사용자가 지정한 조건에 따라 CloudFront는 요청된 콘텐츠 또는 HTTP 403 상태 코드(금지됨)로 요청에 응답합니다. 요청이 차단될 때 사용자 지정 오류 페이지를 반환하도록 CloudFront를 구성할 수도 있습니다. AWS WAF에 대한 자세한 내용은 AWS WAF 개발자 안내서를 참조하십시오.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
