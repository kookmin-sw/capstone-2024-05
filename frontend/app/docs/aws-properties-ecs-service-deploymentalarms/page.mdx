# AWS::ECS::서비스 배포 경보

배포가 실패한 시기를 신속하게 식별한 다음 필요에 따라 실패를 마지막 작업 배포로 롤백할 수 있는 방법을 제공하는 방법 중 하나입니다.

경보가 생성되면 Amazon ECS는 서비스 배포를 실패로 설정합니다. Amazon ECS가 실패 후 마지막으로 완료된 배포로 서비스를 롤백하도록 rollback 파라미터를 설정합니다.

가 (롤링 업데이트)`DeploymentAlarms`로 설정된 `DeploymentController` 경우에만 이 방법을 사용하여 실패를 감지 `ECS` 할 수 있습니다.

자세한 내용은 _ Amazon Elastic Container Service 개발자 안내서_ \*\*의 롤링 업데이트를 참조하세요.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AlarmNames" : [ String, ... ],
  "Enable" : Boolean,
  "Rollback" : Boolean
}
```

## 속성

`AlarmNames`  
하나 이상의 CloudWatch 경보 이름. ","를 사용하여 알람을 구분합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Enable`  
서비스 배포 프로세스에서 CloudWatch 경보 옵션을 사용할지 여부를 결정합니다.  
_필수 항목 여부_: 예  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Rollback`  
서비스 배포가 실패할 경우 서비스를 롤백하도록 Amazon ECS를 구성할지 여부를 결정합니다. 롤백을 사용하는 경우 서비스 배포가 실패하면 서비스가 성공적으로 완료된 마지막 배포로 롤백됩니다.  
_필수 항목 여부_: 예  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음
