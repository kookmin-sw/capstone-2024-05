# AWS::EC2::NetworkInsights분석 분석AclRule

네트워크 ACL(액세스 제어) 규칙에 대해 설명합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Cidr" : String,
  "Egress" : Boolean,
  "PortRange" : PortRange,
  "Protocol" : String,
  "RuleAction" : String,
  "RuleNumber" : Integer
}
```

## 속성

`Cidr`  
CIDR 표기법의 IPv4 주소 범위입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Egress`  
규칙이 아웃바운드 규칙인지 여부를 나타냅니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`PortRange`  
포트의 범위입니다.  
*필수 항목 *여부: 아니요  
_형식_: PortRange  
_업데이트 필요 사항_: 중단 없음

`Protocol`  
프로토콜입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`RuleAction`  
규칙과 일치하는 트래픽을 허용할지 또는 거부할지를 나타냅니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`RuleNumber`  
규칙 번호입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음
