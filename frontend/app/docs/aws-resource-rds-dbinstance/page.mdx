# AWS::RDS::D B상태

리소스는 `AWS::RDS::DBInstance` Amazon DB 인스턴스를 생성합니다. 새 DB 인스턴스는 RDS DB 인스턴스일 수도 있고 Aurora DB 클러스터의 DB 인스턴스일 수도 있습니다.

RDS DB 인스턴스 생성에 대한 자세한 내용은 _Amazon RDS 사용 설명서의 Amazon RDS DB 인스턴스 생성을 참조하세요_.

Aurora DB 클러스터에서 DB 인스턴스를 생성하는 방법에 대한 자세한 내용은 Amazon Aurora 사용 설명서의 Amazon Aurora DB 클러스터 생성을 참조하세요\*\*.

기존 DB 인스턴스를 가져올 때 템플릿 구성이 DB 인스턴스의 실제 구성과 일치하지 않는 경우 AWS CloudFormation에서는 가져오기 작업 중에 템플릿의 변경 사항을 적용합니다.

**중요하다**  
업데이트 중에 DB 인스턴스가 삭제되거나 교체되는 경우 AWS CloudFormation에서는 모든 자동 스냅샷을 삭제합니다. 그러나 수동 DB 스냅샷은 유지됩니다. 교체가 필요한 업데이트 중에 스택 정책을 적용하여 DB 인스턴스가 교체되지 않도록 할 수 있습니다. 자세한 내용은 Prevent Updates to Stack Resources를 참조하십시오.

**DB 인스턴스 업데이트**

"Update requires:_ Replacement"라는 레이블이 지정된 속성이_ 업데이트되면 AWS CloudFormation에서는 먼저 대체 DB 인스턴스를 생성한 다음 대체 DB 인스턴스를 가리키도록 다른 종속 리소스의 참조를 변경하고 마지막으로 이전 DB 인스턴스를 삭제합니다.

**중요하다**  
스택을 업데이트하기 전에 데이터베이스의 스냅샷을 생성하는 것이 좋습니다. 그렇지 않으면 AWS CloudFormation에서 DB 인스턴스를 교체할 때 데이터가 손실됩니다. 데이터를 보존하려면 다음 절차를 수행합니다.  
DB 인스턴스에 대한 활동이 없도록 DB 인스턴스를 사용 중인 모든 애플리케이션을 비활성화합니다.
DB 인스턴스의 스냅샷을 생성합니다. 자세한 내용은 DB 스냅샷 생성 단원을 참조하십시오.
DB 스냅샷을 사용하여 인스턴스를 복원하려면 DB 인스턴스 변경 사항으로 업데이트된 템플릿을 수정하고 `DBSnapshotIdentifier` 사용할 DB 스냅샷의 ID로 속성을 추가합니다.  
속성이 있는 DB 인스턴스를 복원한 후에는 `DBSnapshotIdentifier` `DBSnapshotIdentifier` DB 인스턴스에 대한 향후 업데이트에 대해 동일한 속성을 지정해야 합니다. 업데이트에 대해 이 속성을 지정하면 DB 인스턴스가 DB 스냅샷에서 다시 복원되지 않으며 데이터베이스의 데이터도 변경되지 않습니다. 그러나 속성을 지정하지 않으면 `DBSnapshotIdentifier` 빈 DB 인스턴스가 생성되고 원래 DB 인스턴스가 삭제됩니다. 이전 스냅샷 복원 속성과 다른 속성을 지정하면 지정된 속성에서 새 DB 인스턴스가 복원 `DBSnapshotIdentifier` 되고 원래 DB 인스턴스가 삭제됩니다.
스택을 업데이트합니다.

이 리소스의 다른 속성을 업데이트하는 방법에 대한 자세한 내용은 을 참조하십시오 ` ModifyDBInstance`. 스택 업데이트에 대한 자세한 내용은 AWS CloudFormation 스택 업데이트 단원을 참조하십시오.

**DB 인스턴스 삭제**

Aurora DB 클러스터의 일부인 DB 인스턴스의 경우 DB 인스턴스에 대한 삭제 정책을 설정하여 스택이 삭제될 때 AWS CloudFormation에서 DB 인스턴스를 처리하는 방법을 제어할 수 있습니다. Amazon RDS DB 인스턴스의 경우 ** DB 인스턴스를 유지하거나, ** 삭제하거나, DB 인스턴스의 *스냅샷을 생성*하도록 선택할 수 있습니다. 기본 AWS CloudFormation 동작은 속성에 따라 다릅니다 `DBClusterIdentifier` .

1.  `AWS::RDS::DBInstance` 속성을 지정하지 않은 리소스의 `DBClusterIdentifier` 경우 AWS CloudFormation은 DB 인스턴스의 스냅샷을 저장합니다.

1.  속성을 지정하는 리소스 `AWS::RDS::DBInstance` 의 경우 AWS CloudFormation에서 DB 인스턴스를 삭제합니다.`DBClusterIdentifier`

자세한 내용은 DeletionPolicy 속성 단원을 참조하십시오.

`AvailabilityZone` 데이터베이스가 생성될 가용 영역(AZ)입니다. AWS 리전 및 가용 영역에 대한 자세한 내용은 리전 및 가용 영역을 참조하십시오.  
Amazon Aurora의 경우 각 Aurora DB 클러스터는 3개의 개별 가용 영역에서 스토리지 사본을 호스팅합니다. 이러한 가용 영역 중 하나를 지정합니다. Aurora는 적절한 가용 영역을 지정하지 않으면 자동으로 선택합니다.  
기본값: 엔드포인트의 AWS 리전에서 시스템이 선택한 임의의 가용 영역입니다.

- 지정된 가용 영역은 현재 엔드포인트와 동일한 AWS 리전에 있어야 합니다.
  예: `us-east-1d`

`CertificateRotationRestart`  
SSL/TLS 인증서를 교체할 때 DB 인스턴스를 다시 시작할지 여부를 지정합니다.  
기본적으로 DB 인스턴스는 SSL/TLS 인증서를 교체할 때 다시 시작됩니다. 인증서는 DB 인스턴스가 다시 시작될 때까지 업데이트되지 않습니다.  
\*\* SSL/TLS를 사용하여 DB 인스턴스에 연결하지 않는 경우에만 이 파라미터를 설정합니다.
SSL/TLS를 사용하여 DB 인스턴스에 연결하는 경우 DB 엔진에 대한 적절한 지침에 따라 SSL/TLS 인증서를 교체합니다.

- RDS DB 엔진용 SSL/TLS 인증서 교체에 대한 자세한 내용은 SSL/TLS 인증서 교체 단원을 참조하십시오. 의 _Amazon RDS 사용 설명서._
- Aurora DB 엔진용 SSL/TLS 인증서 교체에 대한 자세한 내용은 Amazon Aurora 사용 설명서의 SSL/TLS 인증서 교체를 참조하세요\**.
  이 설정은 RDS Custom DB 인스턴스에는 적용되지 않습니다.

DB 인스턴스를 복원하는 데 사용되는 DB 스냅샷의 이름 또는 Amazon 리소스 이름(ARN)입니다. 공유 수동 DB 스냅샷에서 복원하는 경우 스냅샷의 ARN을 지정해야 합니다.

를 지정하면 `DBSecurityGroups`AWS CloudFormation에서는 이 속성을 무시합니다. 보안 그룹과 이 속성을 모두 지정하려면 VPC 보안 그룹을 사용해야 합니다. Amazon RDS 및 VPC에 대한 자세한 내용은 Amazon RDS 사용 설명서의 Amazon VPC와 함께 Amazon RDS 사용을 참조하십시오**.

DB 인스턴스에 대한 스토리지 처리량 값을 지정합니다. 이 설정은 `gp3` 스토리지 유형에만 적용됩니다.  
이 설정은 RDS Custom 또는 Amazon Aurora에는 적용되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`StorageType`  
DB 인스턴스와 연결할 스토리지 유형을 지정합니다.  
 유효한 값: ` gp2 | gp3 | io1 | standard`  
이 `standard` 값은 마그네틱이라고도 합니다.  
 `io1` 또는 `gp3` 를 지정하는 경우 매개 변수의 값도 포함해야 합니다 `Iops` .  
 기본값: `io1` `Iops` 매개 변수가 지정된 경우, 그렇지 않은 `gp2`경우  
자세한 내용은 _Amazon RDS 사용 설명서의 Amazon RDS DB 인스턴스 스토리지를 참조하십시오_.  
 **아마존 오로라**  
해당 사항 없음. Aurora 데이터는 솔리드 스테이트 드라이브(SSD)를 사용하는 단일 가상 볼륨인 클러스터 볼륨에 저장됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 일부 중단

`Tags`

*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`Timezone`

*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

DB 인스턴스의 DB 인스턴스 클래스가 기본 프로세서 기능을 사용하는지 여부를 지정합니다.  
이 설정은 RDS Custom DB 인스턴스에는 적용되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`UseLatestRestorableTime`  
DB 인스턴스가 최근 백업 시간에서 복원되었는지 여부를 나타내는 값입니다. 기본적으로 DB 인스턴스는 최근 백업 시간에서 복원되지 않습니다.  
제약 조건: 매개 변수가 제공된 경우 지정할 수 없습니다 `RestoreTime` .  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 일부 중단

`VPCSecurityGroups`  
DB 인스턴스에 할당할 VPC 보안 그룹 ID의 목록입니다. 목록에는 기존 VPC 보안 그룹의 물리적 ID와 템플릿에서 생성된 AWS::EC2::SecurityGroup 리소스에 대한 참조가 모두 포함될 수 있습니다.  
리소스를 업데이트하려는 경우 공유 VPC에서 VPC 보안 그룹을 지정하지 마십시오.  
 를 설정하는 경우 `VPCSecurityGroups`some-address를 설정해서는 안 되며 그 반대의 경우도 마찬가지입니다.  
스택의 DB 인스턴스를 RDS DB 보안 그룹에서 VPC 보안 그룹으로 마이그레이션할 수 있지만 다음 사항에 유의해야 합니다.

- VPC 보안 그룹 멤버십을 설정한 후에는 RDS 보안 그룹 사용으로 되돌릴 수 없습니다.
- DB 인스턴스를 VPC 보안 그룹으로 마이그레이션할 때 DB 인스턴스 업데이트가 실패하거나 다른 AWS CloudFormation 리소스에서 업데이트가 실패하여 스택 업데이트가 롤백되는 경우 RDS 보안 그룹으로 되돌릴 수 없기 때문에 롤백이 실패합니다.
- VPC 보안 그룹을 사용할 때 사용할 수 있는 속성을 사용하려면 DB 인스턴스를 다시 생성해야 합니다. 그렇지 않은 경우 AWS CloudFormation에서는 some-address 속성에 나열된 속성 값만 제출합니다.
  이러한 상황을 방지하려면 스택 템플릿의 유일한 변경 사항인 경우에만 VPC 보안 그룹을 사용하도록 DB 인스턴스를 마이그레이션하십시오.  
   **아마존 오로라**  
  해당 사항 없음. 연결된 EC2 VPC 보안 그룹 목록은 DB 클러스터에서 관리합니다. 지정된 경우 설정은 DB 클러스터 설정과 일치해야 합니다.  
  *필수 항목 *여부: 아니요  
  _유형_: 문자열 목록  
  _업데이트 필요 사항_: 중단 없음

`CertificateDetails.CAIdentifier` DB 인스턴스의 서버 인증서에 사용되는 CA 인증서의 CA 식별자입니다.

`CertificateDetails.ValidTill` DB 인스턴스 서버 인증서의 만료 날짜입니다.

`DbiResourceId` DB 인스턴스에 대한 AWS 리전 고유의 변경할 수 없는 식별자입니다. 이 식별자는 DB 인스턴스의 AWS KMS 키에 액세스할 때마다 AWS CloudTrail 로그 항목에서 찾을 수 있습니다.

`DBSystemId` 컨테이너 데이터베이스(CDB)의 Oracle 시스템 ID(Oracle SID)입니다. Oracle SID는 CDB의 이름이기도 합니다.

Aurora Serverless DB 클러스터의 경우 연결 엔드포인트는 DB 클러스터에만 적용됩니다.

`Endpoint.HostedZoneId` 호스팅 영역을 생성할 때 Amazon Route 53이 할당하는 ID입니다.

`Endpoint.Port`  
데이터베이스가 연결을 수락하는 포트 번호입니다. 예를 들어: `3306`

다음 예제에서는 관리형 마스터 사용자 암호 기능이 있는 AWS::RDS::D BInstance 리소스를 사용하여 AWS CloudFormation 스택을 생성합니다. DB 인스턴스를 인증하는 데 사용되는 보안 암호는 `Secret` 스택 출력에 표시됩니다. 이 값을 다른 CloudFormation 스택에 표시할 필요가 없는 경우 이 출력을 제거할 수 있습니다.

Secrets Manager를 사용한 암호 관리에 대한 자세한 내용은 *Amazon RDS 사용 설명서의 AWS Secrets Manager를 사용한 암호 관리 및 *Amazon Aurora 사용 설명서*의 AWS Secrets Manager를 사용한 암호 관리를 * 참조하세요.
