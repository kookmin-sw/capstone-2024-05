# AWS::ElasticBeanstalk::ConfigurationTemplate

AWS::ElasticBeanstalk::ConfigurationTemplate 리소스는 특정 Elastic Beanstalk 애플리케이션과 연결된 Elastic Beanstalk 구성 템플릿을 지정하는 AWS Elastic Beanstalk 리소스 유형입니다. 구성 템플릿에서 응용 프로그램 구성 설정을 정의합니다. 그런 다음 구성 템플릿을 사용하여 동일한 구성 설정으로 다른 버전의 응용 프로그램을 배포할 수 있습니다.

**메모**  
Elastic Beanstalk 콘솔 및 설명서에서는 구성 템플릿을 저장된 구성이라고 부르는 경우가 많습니다\**. 저장된 구성(구성 템플릿)에서 구성 옵션을 설정하면 Elastic Beanstalk는 여러 소스의 옵션 적용의 일환으로 특정 우선 순위로 구성 옵션을 적용합니다. 자세한 내용은 *AWS Elastic Beanstalk 개발자 안내서의 구성 옵션을 참조하십시오\*.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::ElasticBeanstalk::ConfigurationTemplate",
  "Properties" : {
      "ApplicationName" : String,
      "Description" : String,
      "EnvironmentId" : String,
      "OptionSettings" : [ ConfigurationOptionSetting, ... ],
      "PlatformArn" : String,
      "SolutionStackName" : String,
      "SourceConfiguration" : SourceConfiguration
    }
}
```

## 속성

`ApplicationName`  
이 구성 템플릿과 연결할 Elastic Beanstalk 애플리케이션의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `100`  
_업데이트 필요 사항_: 바꿔 놓음

`Description`  
이 구성에 대한 선택적 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `200`  
_업데이트 필요 사항_: 중단 없음

`EnvironmentId`  
구성 템플릿을 만드는 데 사용할 설정이 있는 환경의 ID입니다. `EnvironmentId`, `PlatformArn`또는 `SolutionStackName` 를 지정하지 않는 경우 `SourceConfiguration` 지정해야 합니다.  
_필수_: 조건부  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`OptionSettings`  
Elastic Beanstalk 구성에 대한 옵션 값(예: 인스턴스 유형)입니다. 지정된 경우 이러한 값은 솔루션 스택 또는 원본 구성 템플릿에서 가져온 값을 재정의합니다. Elastic Beanstalk 구성 옵션의 전체 목록은 * AWS Elastic Beanstalk 개발자 안내서*의 옵션 값을 참조하십시오.  
*필수 항목 *여부: 아니요  
_형식_: ConfigurationOptionSetting 목록  
_업데이트 필요 사항_: 중단 없음

`PlatformArn`  
사용자 지정 플랫폼의 Amazon 리소스 이름(ARN)입니다. 자세한 내용은 * AWS Elastic Beanstalk 개발자 안내서*의 사용자 지정 플랫폼을 참조하십시오.  
를 지정할 경우 를 `PlatformArn`지정하지 마십시오`SolutionStackName`.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`SolutionStackName`  
이 구성에서 사용하는 Elastic Beanstalk 솔루션 스택(플랫폼 버전)의 이름입니다. 예를 들어, `64bit Amazon Linux 2013.09 running Tomcat 7 Java 7`. 솔루션 스택은 구성 템플릿에 대한 운영 체제, 런타임 및 응용 프로그램 서버를 지정합니다. 또한 구성 옵션 집합과 가능한 값과 기본값을 결정합니다. 자세한 내용은 * AWS Elastic Beanstalk 개발자 안내서*의 지원되는 플랫폼을 참조하십시오.  
`SolutionStackName`, `PlatformArn`또는 `EnvironmentId` 를 지정하지 않는 경우 `SourceConfiguration` 지정해야 합니다.  
some-address API를 사용하여 사용 가능한 솔루션 스택 목록을 가져옵니다.  
_필수_: 조건부  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`SourceConfiguration`  
이 템플릿의 기반이 되는 Elastic Beanstalk 구성 템플릿입니다. 지정된 경우 Elastic Beanstalk는 지정된 구성 템플릿의 구성 값을 사용하여 새 구성을 생성합니다.  
에 지정된 값은 에서 `OptionSettings` 가져온 모든 값을 재정의합니다`SourceConfiguration`.  
`SourceConfiguration`, `PlatformArn`또는 `EnvironmentId` 를 지정하지 않는 경우 `SolutionStackName` 지정해야 합니다.  
제약 조건: 솔루션 스택 이름과 소스 구성이 모두 지정된 경우 소스 구성 템플릿의 솔루션 스택이 지정된 솔루션 스택 이름과 일치해야 합니다.  
_필수_: 조건부  
_형식_: SourceConfiguration  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID가 `Ref`내장 함수 에 제공되면 `Ref`리소스 이름을 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

####

`TemplateName` 속성 설명을 사용할 수 없습니다.

## 예제

###

#### JSON (영문)

```
"myConfigTemplate" : {
    "Type" : "AWS::ElasticBeanstalk::ConfigurationTemplate",
    "Properties" : {
      "ApplicationName" :{"Ref" : "myApp"},
      "Description" : "my sample configuration template",
      "EnvironmentId" : "",
      "SourceConfiguration" : {
        "ApplicationName" : {"Ref" : "mySecondApp"},
        "TemplateName" : {"Ref" : "mySourceTemplate"}
      },
      "SolutionStackName" : "64bit Amazon Linux running PHP 5.3",
      "OptionSettings" : [ {
        "Namespace" : "aws:autoscaling:launchconfiguration",
        "OptionName" : "EC2KeyName",
        "Value" : { "Ref" : "KeyName" }
      } ]
    }
  }
```
