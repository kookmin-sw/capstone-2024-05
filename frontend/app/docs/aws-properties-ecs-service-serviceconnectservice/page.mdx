# AWS::ECS::서비스 서비스 연결 서비스

Service Connect 서비스 개체 구성입니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Service Connect를 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "ClientAliases" : [ ServiceConnectClientAlias, ... ],
  "DiscoveryName" : String,
  "IngressPortOverride" : Integer,
  "PortName" : String
}
```

## 속성

`ClientAliases`  
이 Service Connect 서비스에 대한 클라이언트 별칭 목록입니다. 이러한 이름을 사용하여 클라이언트 응용 프로그램에서 사용할 수 있는 이름을 할당할 수 있습니다. 이 목록에 포함할 수 있는 최대 클라이언트 별칭 수는 1개입니다.  
각 별칭("엔드포인트")은 다른 Amazon ECS 태스크("클라이언트")가 이 서비스에 연결하는 데 사용할 수 있는 정규화된 이름 및 포트 번호입니다.  
각 이름 및 포트 매핑은 네임스페이스 내에서 고유해야 합니다.  
각각에 대해 하나 이상의 에 `ServiceConnectService` 대해 하나 `clientAlias`이상의 를 제공해야 합니다`port`.  
*필수 항목 *여부: 아니요  
_유형_: ServiceConnectClientAlias 목록  
_업데이트 필요 사항_: 중단 없음

`DiscoveryName`  
`discoveryName` 는 Amazon ECS가 이 Amazon ECS 서비스에 대해 생성하는 새 AWS Cloud Map 서비스의 이름입니다. 이는 AWS Cloud Map 네임스페이스 내에서 고유해야 합니다. 이름은 최대 64자를 포함할 수 있습니다. 이름에는 소문자, 숫자, 밑줄(\_) 및 하이픈(-)이 포함될 수 있습니다. 이름은 하이픈으로 시작할 수 없습니다.  
지정 `discoveryName` 하지 않으면 작업 정의의 포트 매핑 이름이 에서 사용됩니다`portName.namespace`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`IngressPortOverride`  
수신 대기할 Service Connect 프록시의 포트 번호입니다.  
이 필드의 값을 사용하여 이 애플리케이션의 작업 정의에 명명된 포트 번호의 트래픽에 대한 프록시를 우회 `portMapping` 한 다음 VPC 보안 그룹에서 이 프록시를 사용하여 이 Amazon ECS 서비스의 프록시에 대한 트래픽을 허용합니다.  
`awsvpc` mode 및 Fargate에서 기본값은 컨테이너 포트 번호입니다. 컨테이너 포트 번호는 `portMapping` 작업 정의에 있습니다. 브리지 모드에서 기본값은 Service Connect 프록시의 임시 포트입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_최소_: `0`  
_최대_: `65535`  
_업데이트 필요 사항_: 중단 없음

`PortName`  
 `portName` `portMappings` 이 Amazon ECS 서비스의 작업 정의에 있는 모든 컨테이너 중 하나의 이름과 일치해야 합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
