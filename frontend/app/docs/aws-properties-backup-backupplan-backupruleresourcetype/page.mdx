# AWS::백업::백업 계획 BackupRuleResourceType

선택한 자원을 백업하기 위해 작업을 예약하는 데 사용되는 속성이 포함된 개체를 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "CompletionWindowMinutes" : Double,
  "CopyActions" : [ CopyActionResourceType, ... ],
  "EnableContinuousBackup" : Boolean,
  "Lifecycle" : LifecycleResourceType,
  "RecoveryPointTags" : {Key: Value, ...},
  "RuleName" : String,
  "ScheduleExpression" : String,
  "ScheduleExpressionTimezone" : String,
  "StartWindowMinutes" : Double,
  "TargetBackupVault" : String
}
```

## 속성

`CompletionWindowMinutes`  
백업 작업이 성공적으로 시작된 후 완료해야 하거나 AWS Backup에 의해 취소되기 전의 값(분)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 더블  
_업데이트 필요 사항_: 중단 없음

`CopyActions`  
복사 작업의 세부 정보를 포함하는 CopyAction 개체의 배열입니다.  
*필수 항목 *여부: 아니요  
_Type_: CopyActionResourceType 목록  
_업데이트 필요 사항_: 중단 없음

`EnableContinuousBackup`  
연속 백업 및 특정 시점 복원(PITR)을 지원합니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Lifecycle`  
수명 주기는 보호된 리소스가 콜드 스토리지로 전환되는 시기와 만료되는 시점을 정의합니다. AWS Backup은 사용자가 정의한 수명 주기에 따라 백업을 자동으로 전환하고 만료합니다.  
*필수 항목 *여부: 아니요  
_유형_: LifecycleResourceType  
_업데이트 필요 사항_: 중단 없음

`RecoveryPointTags`  
리소스를 구성하는 데 도움이 되도록 만든 리소스에 고유한 메타데이터를 할당할 수 있습니다. 각 태그는 키-값 페어입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 맵  
_업데이트 필요 사항_: 중단 없음

`RuleName`  
백업 규칙의 표시 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`ScheduleExpression`  
AWS Backup이 백업 작업을 시작하는 시기를 지정하는 CRON 표현식입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`ScheduleExpressionTimezone`  
속성 설명을 사용할 수 없습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`StartWindowMinutes`  
백업이 예약된 후 작업이 성공적으로 시작되지 않을 경우 작업이 취소되기 전에 기간(분)을 지정하는 선택적 값입니다.  
이 값이 포함된 경우 오류를 방지하려면 60분 이상이어야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 더블  
_업데이트 필요 사항_: 중단 없음

`TargetBackupVault`  
백업이 저장되는 논리적 컨테이너의 이름입니다. 백업 저장소는 생성에 사용된 계정과 생성된 AWS 리전에 고유한 이름으로 식별됩니다. 문자, 숫자 및 하이픈으로 구성됩니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
