# AWS::Cognito::UserPool 사용자 속성 업데이트 설정

사용자 특성에 대한 업데이트 설정입니다. 이러한 설정에는 `AttributesRequireVerificationBeforeUpdate`Amazon Cognito에 사용자의 이메일 주소 및 전화번호 속성 값 변경을 처리하는 방법을 알려주는 사용자 풀 설정 속성이 포함됩니다. 자세한 내용은 이메일 주소 및 전화 번호에 대한 업데이트 확인을 참조하세요.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AttributesRequireVerificationBeforeUpdate" : [ String, ... ]
}
```

## 속성

`AttributesRequireVerificationBeforeUpdate`  
Amazon Cognito가 해당 속성의 값을 업데이트하기 전에 사용자가 이메일 주소, 전화번호 또는 둘 다를 확인해야 합니다. 이 옵션이 활성화된 사용자 속성을 업데이트하면 Amazon Cognito에서 새 전화번호 또는 이메일 주소로 확인 메시지를 보냅니다. Amazon Cognito는 사용자가 확인 메시지에 응답하고 새 값을 확인할 때까지 속성 값을 변경하지 않습니다.  
VerifyUserAttribute API 요청을 사용하여 업데이트된 이메일 주소 또는 전화번호를 확인할 수 있습니다. AdminUpdateUserAttributes API를 호출하고 `email_verified` or `phone_number_verified` 를 true로 설정할 수도 있습니다.  
 `AttributesRequireVerificationBeforeUpdate` 가 false인 경우 사용자 풀은 Amazon Cognito가 속성 변경 사항을 업데이트하기 전에 사용자가 속성 변경 사항을 확인하도록 요구하지 않습니다. 가 false인 사용자 풀에서 `AttributesRequireVerificationBeforeUpdate` 속성 값을 변경하는 API 작업은 사용자 또는 속성을 즉시 업데이트할 수 있습니다 `email` `phone_number` .  
_필수 항목 여부_: 예  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음
