# AWS::CloudTrail::Trail 이벤트 선택기

이벤트 선택기를 사용하여 추적에 대한 관리 및 데이터 이벤트 설정을 추가로 지정합니다. 기본적으로 특정 이벤트 선택기 없이 생성된 추적은 모든 읽기 및 쓰기 관리 이벤트를 로깅하고 데이터 이벤트는 로깅하지 않도록 구성됩니다. 계정에서 이벤트가 발생하면 CloudTrail은 모든 추적에 대한 이벤트 선택기를 평가합니다. 각 추적에 대해 이벤트가 이벤트 선택기와 일치하면 추적은 이벤트를 처리하고 로깅합니다. 이벤트가 이벤트 선택기와 일치하지 않으면 추적은 이벤트를 로깅하지 않습니다.

추적에 대해 최대 5개의 이벤트 선택기를 구성할 수 있습니다.

이벤트 선택기와 고급 이벤트 선택기를 모두 추적에 적용할 수는 없습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "DataResources" : [ DataResource, ... ],
  "ExcludeManagementEventSources" : [ String, ... ],
  "IncludeManagementEvents" : Boolean,
  "ReadWriteType" : String
}
```

## 속성

`DataResources`  
CloudTrail은 기본 이벤트 선택기를 사용하여 Amazon S3 객체, AWS Lambda 함수 및 Amazon DynamoDB 테이블에 대한 데이터 이벤트 로깅을 지원합니다. 개별 이벤트 선택기에 대해 최대 250개의 리소스를 지정할 수 있지만, 추적의 모든 이벤트 선택기에서 총 데이터 리소스 수는 250개를 초과할 수 없습니다. 이 제한은 모든 데이터 이벤트에 대해 리소스 로깅을 구성하는 경우에는 적용되지 않습니다.  
자세한 내용은 * AWS CloudTrail User Guide*의 Data Events and Limits in AWS CloudTrail을 참조하십시오.  
_필수_: 조건부  
_형식_: DataResource 목록  
_업데이트 필요 사항_: 중단 없음

`ExcludeManagementEventSources`  
관리 이벤트를 추적에 기록하지 않으려는 서비스 이벤트 소스의 선택적 목록입니다. 이 릴리스에서는 목록을 비워 두거나(필터 비활성화) 또는 를 포함하여 AWS Key Management Service 또는 Amazon RDS Data API 이벤트를 필터링할 수 있습니다 `kms.amazonaws.com` `rdsdata.amazonaws.com`. 기본적으로 `ExcludeManagementEventSources` 는 비어 있으며 AWS KMS 및 Amazon RDS Data API 이벤트가 추적에 로깅됩니다. 관리 이벤트 소스는 이벤트 소스를 지원하는 리전에서만 제외할 수 있습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`IncludeManagementEvents`  
이벤트 선택기에 추적에 대한 관리 이벤트를 포함할지 여부를 지정합니다.  
 자세한 내용은 * AWS CloudTrail User Guide*의 Management Events를 참조하십시오.  
기본적으로 값은 `true`입니다.  
관리 이벤트의 첫 번째 복사본은 무료입니다. 동일한 리전의 후속 추적에 로깅하는 관리 이벤트의 추가 사본에 대해서는 요금이 부과됩니다. CloudTrail 요금에 대한 자세한 내용은 AWS CloudTrail 요금을 참조하십시오.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`ReadWriteType`  
추적이 읽기 전용 이벤트, 쓰기 전용 이벤트 또는 모두를 로깅할지 여부를 지정합니다. 예를 들어, EC2는 `GetConsoleOutput` 읽기 전용 API 작업이며 `RunInstances` 쓰기 전용 API 작업입니다.  
 기본적으로 값은 `All`입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `All | ReadOnly | WriteOnly`  
_업데이트 필요 사항_: 중단 없음
