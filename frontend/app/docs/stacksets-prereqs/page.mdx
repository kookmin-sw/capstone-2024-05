# 스택 세트 작업을 위한 사전 조건

스택 세트는 여러 계정에 걸쳐 스택 작업을 수행하기 때문에 첫 번째 스택 세트를 생성하기 전에 AWS 계정에 정의된 필수 권한이 필요합니다.

자체 관리형 권한**으로 스택 세트를 생성하는 데 필요한 권한을 설정하려면 ** 기본적으로 비활성화된 리전과 관련된 스택 세트 작업 수행 및 자체 관리형 권한 부여 섹션을 참조하세요.

서비스 관리형 권한**으로 스택 세트를 생성하는 데 필요한 권한을 설정하려면 ** 기본적으로 비활성화된 리전과 관련된 스택 세트 작업 수행 및 AWS Organizations를 사용하여 신뢰할 수 있는 액세스 활성화 섹션을 참조하세요.

## 기본적으로 비활성화된 리전과 관련된 스택 세트 작업 수행

2019년 3월 20일 이후에 도입된 AWS 리전(예: 아시아 태평양(홍콩))은 기본적으로 비활성화되어 있습니다. 계정에 대해 이러한 리전을 활성화해야 사용할 수 있습니다. 따라서 기본적으로 비활성화된 리전의 계정과 관련된 스택 세트 작업을 수행하기 전에 다음 사항을 고려하십시오.

- 기본적으로 비활성화되어 있는 리전에서 스택 세트의 관리자 계정(자체 관리형 권한을 사용하는 경우) 또는 조직의 관리 계정(서비스 관리형 권한을 사용하는 경우)에서 스택 세트를 생성하려면 먼저 관리자 또는 관리 계정에 대해 해당 리전을 활성화해야 합니다.
- AWS CloudFormation에서 스택 인스턴스를 성공적으로 생성하거나 업데이트하려면 다음을 수행합니다.
  - 대상 계정은 해당 대상 계정에 대해 현재 활성화된 리전에 있어야 합니다.
  - 스택 세트의 관리자 계정 또는 조직의 관리 계정에는 대상 계정과 동일한 리전이 활성화되어 있어야 합니다.

**중요하다**  
스택 세트 작업 중에 관리자와 대상 계정은 관련된 스택 세트 및 스택 세트 인스턴스 외에도 계정 자체에 대한 메타데이터를 교환합니다.  
또한, 스택 세트 인스턴스가 상주하는 계정이 포함된 리전을 비활성화하는 경우, 원하는 경우 해당 인스턴스 또는 리소스를 삭제할 책임이 있습니다. 또한 비활성화된 리전의 대상 계정에 대한 메타데이터는 관리자 계정에 유지됩니다.

리전 활성화 및 비활성화에 대한 자세한 내용은 _AWS General Reference의 AWS 리전 관리를 참조하십시오_.

**항목**

- 기본적으로 비활성화된 리전과 관련된 스택 세트 작업 수행
- 자체 관리 권한 부여
- AWS Organizations를 통해 신뢰할 수 있는 액세스 활성화
- 위임된 관리자 등록
