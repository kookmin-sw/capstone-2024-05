# AWS::VpcLattice::ServiceNetworkServiceAssociation

서비스를 서비스 네트워크와 연결합니다.

서비스 및 서비스 네트워크가 이미 연결되어 있거나 연결 해제 또는 삭제가 진행 중인 경우 이 작업을 사용할 수 없습니다. 연결이 실패하면 연결을 삭제하고 다시 만들어 작업을 다시 시도할 수 있습니다.

호출자와 공유되는 서비스 및 서비스 네트워크는 연결할 수 없습니다. 호출자는 서비스 또는 서비스 네트워크를 소유해야 합니다.

이 작업의 결과로 서비스 네트워크 계정 및 연결 소유자 계정에 연결이 만들어집니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::VpcLattice::ServiceNetworkServiceAssociation",
  "Properties" : {
      "DnsEntry" : DnsEntry,
      "ServiceIdentifier" : String,
      "ServiceNetworkIdentifier" : String,
      "Tags" : [Tag, ... ]
    }
}
```

## 속성

`DnsEntry`  
서비스의 DNS 정보입니다.  
*필수 항목 *여부: 아니요  
_형식_: DnsEntry  
_업데이트 필요 사항_: 중단 없음

`ServiceIdentifier`  
서비스의 ID 또는 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`ServiceNetworkIdentifier`  
서비스 네트워크의 ID 또는 Amazon 리소스 이름(ARN)입니다. 작업에 지정된 리소스가 서로 다른 계정에 있는 경우 ARN을 사용해야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Tags`  
연결에 대한 태그입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`연결 `Ref`의 Amazon 리소스 이름(ARN)을 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Arn` 서비스 네트워크와 서비스 간 연결의 Amazon 리소스 이름(ARN)입니다.

`CreatedAt` 연결이 생성된 날짜 및 시간으로, ISO-8601 형식으로 지정됩니다.

`DnsEntry.DomainName` 서비스의 도메인 이름입니다.

`DnsEntry.HostedZoneId` 호스팅 영역의 ID입니다.

`Id` 서비스 네트워크와 서비스 간의 연결 ID입니다.

`ServiceArn` 서비스의 Amazon 리소스 이름(ARN)입니다.

`ServiceId` 서비스의 ID입니다.

`ServiceName` 서비스의 이름입니다.

`ServiceNetworkArn` 서비스 네트워크의 Amazon 리소스 이름(ARN)

`ServiceNetworkId` 서비스 네트워크의 ID입니다.

`ServiceNetworkName` 서비스 네트워크의 이름입니다.

`Status` 서비스 네트워크와 서비스 간의 연결 상태입니다.
