# AWS::Route53RecoveryControl::SafetyRule 게이팅 규칙

게이팅 규칙은 게이팅 라우팅 컨트롤 또는 게이팅 라우팅 컨트롤 집합이 지정한 규칙 구성에 따라 true로 평가되는지 확인하여 라우팅 제어 상태 변경 집합을 완료할 수 있도록 합니다.

예를 들어, 하나의 게이팅 라우팅 제어를 지정하고 `Type` 규칙 구성에서 `OR` 를 로 설정하면 규칙이 true로 평가되려면 게이팅 라우팅 제어를 로 설정해야 `On` 합니다 . 즉, 게이팅 제어 스위치를 On으로 설정합니다. 이렇게 하면 게이팅 규칙에서 지정한 대상 라우팅 제어에 대한 라우팅 제어 상태를 업데이트할 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "GatingControls" : [ String, ... ],
  "TargetControls" : [ String, ... ],
  "WaitPeriodMs" : Integer
}
```

## 속성

`GatingControls`  
게이팅 라우팅 제어 Amazon 리소스 이름(ARN)의 배열입니다. 간단한 온-오프 스위치의 경우 하나의 라우팅 제어에 대한 ARN을 지정합니다. 게이팅 라우팅 제어는 대상 라우팅 제어 상태를 변경할 수 있는지 여부를 결정하기 위해 지정한 규칙 구성에 의해 평가됩니다.  
_필수 항목 여부_: 예  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`TargetControls`  
대상 라우팅 제어 Amazon 리소스 이름(ARN)의 배열로, 지정한 규칙 구성이 게이팅 라우팅 제어에 대해 true로 평가되는 경우에만 상태를 업데이트할 수 있습니다. 간단한 예로, 단일 게이팅 컨트롤이 있는 경우 대상 라우팅 컨트롤 집합에 대한 전체 온-오프 스위치 역할을 합니다. 예를 들어 이를 사용하여 자동화된 장애 조치(failover)를 수동으로 재정의할 수 있습니다.  
_필수 항목 여부_: 예  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`WaitPeriodMs`  
대상 라우팅 제어에 대한 모든 요청이 실패하는 평가 기간(밀리초(ms))입니다. 이렇게 하면 상태 플래핑을 방지할 수 있습니다. 대기 기간은 기본적으로 5000ms이지만 사용자 지정 값을 선택할 수 있습니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음
