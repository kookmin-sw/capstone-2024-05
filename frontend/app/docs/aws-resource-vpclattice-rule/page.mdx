# AWS::VpcLattice::규칙

수신기 규칙을 만듭니다. 각 수신기에는 연결 요청을 확인하기 위한 기본 규칙이 있지만 추가 규칙을 정의할 수 있습니다. 각 규칙은 우선 순위, 하나 이상의 작업 및 하나 이상의 조건으로 구성됩니다. 자세한 내용은 _Amazon VPC Lattice 사용 설명서의 리스너 규칙을 참조하세요_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::VpcLattice::Rule",
  "Properties" : {
      "Action" : Action,
      "ListenerIdentifier" : String,
      "Match" : Match,
      "Name" : String,
      "Priority" : Integer,
      "ServiceIdentifier" : String,
      "Tags" : [Tag, ... ]
    }
}
```

## 속성

`Action`  
규칙에 대한 작업을 설명합니다.  
_필수 항목 여부_: 예  
_유형_: 작업  
_업데이트 필요 사항_: 중단 없음

`ListenerIdentifier`  
리스너의 ID 또는 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Match`  
규칙 일치.  
_필수 항목 여부_: 예  
_유형_: 성냥  
_업데이트 필요 사항_: 중단 없음

`Name`  
규칙의 이름입니다. 이름은 수신기 내에서 고유해야 합니다. 유효한 문자는 a-z, 0-9 및 하이픈(-)입니다. 하이픈은 첫 번째 또는 마지막 문자로 사용하거나 다른 하이픈 바로 뒤에 사용할 수 없습니다.  
이름을 지정하지 않으면 CloudFormation에서 이름을 생성합니다. 그러나 이름을 지정하고 나중에 자원을 바꾸려면 새 이름을 지정해야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Priority`  
규칙에 할당된 우선 순위입니다. 특정 수신기에 대한 각 규칙에는 고유한 우선 순위가 있어야 합니다. 우선순위 번호가 낮을수록 우선순위가 높습니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`ServiceIdentifier`  
서비스의 ID 또는 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Tags`  
규칙의 태그입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`규칙 `Ref`의 Amazon 리소스 이름(ARN)을 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Arn` 규칙의 Amazon 리소스 이름(ARN)입니다.

`Id` 수신기의 ID입니다.
