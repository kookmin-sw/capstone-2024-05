# 스택 작업

스택은 단일 단위로 관리할 수 있는 AWS 리소스의 모음입니다. 즉, 스택을 생성, 업데이트 또는 삭제하여 리소스 모음을 생성, 업데이트 또는 삭제할 수 있습니다. 스택의 모든 리소스는 스택의 AWS CloudFormation 템플릿에 의해 정의됩니다. 예를 들어, 스택에는 웹 서버, 데이터베이스 및 네트워킹 규칙과 같은 웹 애플리케이션을 실행하는 데 필요한 모든 리소스가 포함될 수 있습니다. 해당 웹 애플리케이션이 더 이상 필요하지 않은 경우 스택을 삭제하기만 하면 모든 관련 리소스가 삭제됩니다.

AWS CloudFormation에서는 모든 스택 리소스가 적절하게 생성되거나 삭제되도록 합니다. AWS CloudFormation에서는 스택 리소스를 단일 단위로 취급하기 때문에 스택을 생성하거나 삭제하려면 스택을 모두 성공적으로 생성하거나 삭제해야 합니다. 리소스를 생성할 수 없는 경우 AWS CloudFormation은 스택을 롤백하고 생성된 모든 리소스를 자동으로 삭제합니다. 리소스를 삭제할 수 없는 경우 스택을 성공적으로 삭제할 수 있을 때까지 나머지 리소스가 유지됩니다.

AWS CloudFormation 콘솔, API, AWS CLI 또는 Systems Manager Application Manager를 사용하여 스택을 관리할 수 있습니다.

**메모**  
스택 리소스가 작동한 시간에 대한 요금이 부과됩니다(스택을 즉시 삭제한 경우에도 마찬가지임).

**항목**

- AWS CloudFormation 콘솔 사용
- AWS Command Line Interface 사용
- AWS CloudFormation 스택 업데이트
- 스택 및 리소스에 대한 관리되지 않는 구성 변경 감지Detecting unmanaged configuration changes to stacks and resources
- 기존 리소스를 CloudFormation 관리로 가져오기
- 스택 실패 옵션
- 스택 간 리소스 이동
- 가져오기 및 드리프트 감지 작업을 지원하는 리소스Resources that support import and drift detection operations
- 스택 출력 값 내보내기
- 내보낸 출력 값을 가져오는 스택 나열
- 중첩 스택 작업
- AWS CloudFormation에서 Microsoft Windows 스택 작업
