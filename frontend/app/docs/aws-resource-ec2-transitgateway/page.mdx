# AWS::EC2::전송 게이트웨이

Transit Gateway를 지정합니다.

전송 게이트웨이를 사용하여 Virtual Private Cloud(VPC)와 온프레미스 네트워크를 상호 연결할 수 있습니다. Transit Gateway가 상태가 되면 `available` VPC 및 VPN 연결을 Transit Gateway에 연결할 수 있습니다.

VPC를 연결하려면 AWS::EC2::TransitGatewayAttachment를 사용합니다.

VPN 연결을 연결하려면 AWS::EC2::CustomerGateway를 사용하여 고객 게이트웨이를 생성하고 AWS::EC2::VPNConnection 호출에서 고객 게이트웨이의 ID와 전송 게이트웨이의 ID를 지정합니다.

Transit Gateway를 생성하면 기본 Transit Gateway 라우팅 테이블이 생성되어 기본 연결 라우팅 테이블 및 기본 전파 라우팅 테이블로 사용됩니다. AWS::EC2::TransitGatewayRouteTable을 사용하여 추가 전송 게이트웨이 라우팅 테이블을 생성할 수 있습니다. 자동 경로 전파를 비활성화하면 기본 Transit Gateway 라우팅 테이블이 생성되지 않습니다. AWS::EC2::TransitGatewayRouteTablePropagation을 사용하여 리소스 연결에서 전송 게이트웨이 라우팅 테이블로 라우팅을 전파할 수 있습니다. 자동 연결을 비활성화하는 경우 AWS::EC2::TransitGatewayRouteTableAssociation을 사용하여 리소스 연결을 전송 게이트웨이 라우팅 테이블과 연결할 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::TransitGateway",
  "Properties" : {
      "AmazonSideAsn" : Integer,
      "AssociationDefaultRouteTableId" : String,
      "AutoAcceptSharedAttachments" : String,
      "DefaultRouteTableAssociation" : String,
      "DefaultRouteTablePropagation" : String,
      "Description" : String,
      "DnsSupport" : String,
      "MulticastSupport" : String,
      "PropagationDefaultRouteTableId" : String,
      "Tags" : [Tag, ... ],
      "TransitGatewayCidrBlocks" : [ String, ... ],
      "VpnEcmpSupport" : String
    }
}
```

## 속성

`AmazonSideAsn`  
BGP 세션의 Amazon 측에 대한 프라이빗 자율 시스템 번호(ASN)입니다. 범위는 16비트 ASN의 경우 64512에서 65534 사이입니다. 기본값은 64512입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 바꿔 놓음

`AssociationDefaultRouteTableId`  
기본 연결 라우팅 테이블의 ID입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`AutoAcceptSharedAttachments`  
첨부 파일 요청의 자동 수락을 사용하거나 사용하지 않도록 설정합니다. 기본적으로 비활성화되어 있습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `disable | enable`  
_업데이트 필요 사항_: 중단 없음

`DefaultRouteTableAssociation`  
기본 연결 경로 테이블과의 자동 연결을 사용하거나 사용하지 않도록 설정합니다. 기본적으로 사용하도록 설정됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `disable | enable`  
_업데이트 필요 사항_: 중단 없음

`DefaultRouteTablePropagation`  
기본 전파 경로 테이블에 대한 경로의 자동 전파를 활성화하거나 비활성화합니다. 기본적으로 사용하도록 설정됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `disable | enable`  
_업데이트 필요 사항_: 중단 없음

`Description`  
Transit Gateway에 대한 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DnsSupport`  
DNS 지원을 활성화하거나 비활성화합니다. 기본적으로 사용하도록 설정됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `disable | enable`  
_업데이트 필요 사항_: 중단 없음

`MulticastSupport`  
transit gateway에서 멀티캐스트가 활성화되었는지 여부를 나타냅니다  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `disable | enable`  
_업데이트 필요 사항_: 바꿔 놓음

`PropagationDefaultRouteTableId`  
기본 전파 경로 테이블의 ID입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Tags`  
Transit Gateway의 태그입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`TransitGatewayCidrBlocks`  
전송 게이트웨이 CIDR 블록입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`VpnEcmpSupport`  
Equal Cost Multipath Protocol 지원을 활성화하거나 비활성화합니다. 기본적으로 사용하도록 설정됩니다.

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 intrinsic 함수에 전달하면 `Ref` `Ref`transit gateway의 ID를 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Id` 전송 게이트웨이의 ID입니다.

## 예제

### Transit Gateway 생성

```
"myTransitGateway": {

   "Properties": {
      "AmazonSideAsn": 65000,
      "Description": "TGW Route Integration Test",
      "AutoAcceptSharedAttachments": "disable",
      "DefaultRouteTableAssociation": "enable",
      "DnsSupport": "enable",
      "VpnEcmpSupport": "enable",
      "Tags": [
         {
            "Key": "Application",
            "Value": {
               "Ref": "AWS::StackId"
            }
         }
      ]
   }
}
```
