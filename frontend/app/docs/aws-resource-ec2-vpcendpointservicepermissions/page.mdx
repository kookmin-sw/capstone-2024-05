# AWS::EC2::VPCEndpointService권한

서비스 소비자(사용자, IAM 역할 및 AWS 계정)에게 VPC 엔드포인트 서비스에 연결할 수 있는 권한을 부여하거나 취소합니다.

모든 보안 주체에게 권한을 부여하면 서비스가 퍼블릭이 됩니다. 공용 서비스의 이름을 알고 있는 모든 사용자는 엔드포인트 연결 요청을 보낼 수 있습니다. 서비스에 수동 승인이 필요하지 않은 경우 첨부 파일이 자동으로 승인됩니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::VPCEndpointServicePermissions",
  "Properties" : {
      "AllowedPrincipals" : [ String, ... ],
      "ServiceId" : String
    }
}
```

## 속성

`AllowedPrincipals`  
하나 이상의 보안 주체(예: 사용자, IAM 역할 및 AWS 계정)의 Amazon 리소스 이름(ARN)입니다. 이 목록의 보안 주체에게 권한이 부여됩니다. 모든 보안 주체에 권한을 부여하려면 별표(*)를 지정합니다. 이 목록에 없는 보안 주체에 대한 사용 권한이 취소됩니다. 목록이 비어 있으면 모든 권한이 취소됩니다.  
*필수 항목 *여부: 아니요  
*유형*: 문자열 목록  
*업데이트 필요 사항\*: 중단 없음

`ServiceId`  
서비스의 ID입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`VPC 엔드포인트 서비스 권한의 ID를 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.
