# AWS::Redshift::이벤트 구독

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Redshift::EventSubscription",
  "Properties" : {
      "Enabled" : Boolean,
      "EventCategories" : [ String, ... ],
      "Severity" : String,
      "SnsTopicArn" : String,
      "SourceIds" : [ String, ... ],
      "SourceType" : String,
      "SubscriptionName" : String,
      "Tags" : [Tag, ... ]
    }
}
```

## 속성

`Enabled`  
부울 값; 구독 `true` 을 활성화하려면 로 설정하고, `false` 구독을 만들지만 활성화하지는 않으려면 로 설정합니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`EventCategories`  
이벤트 알림 구독에서 게시할 Amazon Redshift 이벤트 범주를 지정합니다.  
값: 구성, 관리, 모니터링, 보안, 보류 중  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Severity`  
이벤트 알림 구독에서 게시할 Amazon Redshift 이벤트 심각도를 지정합니다.  
값: ERROR, INFO  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`SnsTopicArn`  
이벤트 알림을 전송하는 데 사용되는 Amazon SNS 주제의 Amazon 리소스 이름(ARN)입니다. ARN은 주제를 생성하고 구독할 때 Amazon SNS에 의해 생성됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`SourceIds`  
Amazon Redshift 소스 객체의 식별자 하나 이상의 목록입니다. 모든 개체는 원본 형식 매개 변수에 지정된 것과 동일한 형식이어야 합니다. 이벤트 구독은 지정된 개체에서 생성된 이벤트만 반환합니다. 지정하지 않으면 지정된 원본 형식 내의 모든 개체에 대해 이벤트가 반환됩니다.  
예: my-cluster-1, my-cluster-2  
예: my-snapshot-20131010  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`SourceType`  
이벤트를 생성할 원본의 형식입니다. 예를 들어 클러스터에서 생성된 이벤트에 대한 알림을 받으려면 이 매개 변수를 cluster로 설정합니다. 이 값을 지정하지 않으면 AWS 계정의 모든 Amazon Redshift 객체에 대해 이벤트가 반환됩니다. 소스 ID를 지정하려면 소스 유형을 지정해야 합니다.  
유효한 값: cluster, cluster-parameter-group, cluster-security-group, cluster-snapshot 및 scheduled-action.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`SubscriptionName` 만들 이벤트 구독의 이름입니다.  
제약 조건:

- null, 비워 두거나 비워 둘 수 없습니다.
- 1자에서 255자 사이의 영숫자 또는 하이픈을 포함해야 합니다.
- 첫 번째 문자는 문자여야 합니다.
- 하이픈으로 끝나거나 두 개의 연속 하이픈을 포함할 수 없습니다.
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _최대_: `2147483647`  
  _업데이트 필요 사항_: 바꿔 놓음

`Tags`  
태그 인스턴스 목록입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`리소스 이름이 반환됩니다. 예를 들어:

`{ "Ref": "mySubscriptionName" }`

Amazon Redshift의 경우 `EventSubscription`Ref는 구독의 이름을 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`CustomerAwsId` Amazon Redshift 이벤트 알림 구독과 연결된 AWS 계정입니다.

`CustSubscriptionId` Amazon Redshift 이벤트 알림 구독의 이름입니다.

`EventCategoriesList` 이벤트 알림 구독에 지정된 Amazon Redshift 이벤트 범주 목록입니다.  
값: 구성, 관리, 모니터링, 보안, 보류 중

`SourceIdsList` Amazon Redshift 이벤트 알림 구독에 이벤트를 게시하는 소스 목록입니다.

`Status` Amazon Redshift 이벤트 알림 구독의 상태입니다.  
제약 조건:

- 다음 중 하나일 수 있습니다. 활성 | 무허가 | 주제가 존재하지 않음
- "no-permission" 상태는 Amazon Redshift가 더 이상 Amazon SNS 주제에 게시할 권한이 없음을 나타냅니다. "topic-not-exist" 상태는 구독이 생성된 후 항목이 삭제되었음을 나타냅니다.

`SubscriptionCreationTime` Amazon Redshift 이벤트 알림 구독이 생성된 날짜 및 시간입니다.
