# AWS::ECS::서비스 ServiceConnectClient별칭

각 별칭("끝점")은 다른 작업("클라이언트")이 이 서비스에 연결하는 데 사용할 수 있는 정규화된 이름 및 포트 번호입니다.

각 이름 및 포트 매핑은 네임스페이스 내에서 고유해야 합니다.

네임스페이스에서 실행되는 작업은 짧은 이름을 사용하여 네임스페이스의 서비스에 연결할 수 있습니다. 태스크는 네임스페이스의 모든 클러스터에서 서비스에 연결할 수 있습니다. 작업은 가시성을 높이기 위해 로그와 메트릭을 수집하는 관리형 프록시 컨테이너를 통해 연결됩니다. Amazon ECS 서비스가 생성하는 작업만 Service Connect에서 지원됩니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Service Connect를 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "DnsName" : String,
  "Port" : Integer
}
```

## 속성

`DnsName`  
`dnsName` 는 이 서비스에 연결하기 위해 클라이언트 작업의 응용 프로그램에서 사용하는 이름입니다. 이름은 유효한 DNS 이름이어야 하지만 정규화될 필요는 없습니다. 이름은 최대 127자를 포함할 수 있습니다. 이름에는 소문자, 숫자, 밑줄(\_), 하이픈(-) 및 마침표(.)가 포함될 수 있습니다. 이름은 하이픈으로 시작할 수 없습니다.  
이 매개 변수를 지정하지 않으면 기본값이 `discoveryName.namespace` 사용됩니다. 지정 `discoveryName` 하지 않으면 작업 정의의 포트 매핑 이름이 에서 사용됩니다`portName.namespace`.  
클라이언트 Amazon ECS 서비스에서 애플리케이션을 변경하지 않으려면 클라이언트 애플리케이션이 기본적으로 사용하는 것과 동일한 이름으로 설정합니다. 예를 들어, 몇 가지 일반적인 이름은 `database`, `db`또는 또는 와 같은 데이터베이스의 소문자 이름입니다 `mysql` `redis`. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Service Connect를 참조하십시오_.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Port`  
Service Connect 프록시의 수신 대기 포트 번호입니다. 이 포트는 동일한 네임스페이스 내의 모든 작업 내에서 사용할 수 있습니다.  
클라이언트 Amazon ECS 서비스에서 애플리케이션을 변경하지 않으려면 클라이언트 애플리케이션이 기본적으로 사용하는 것과 동일한 포트로 설정합니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Service Connect를 참조하십시오_.  
_필수 항목 여부_: 예  
_유형_: Integer  
_최소_: `0`  
_최대_: `65535`  
_업데이트 필요 사항_: 중단 없음
