# AWS::ECS::TaskDefinition 볼륨

이 `Volume` 속성은 작업 정의에 사용되는 데이터 볼륨을 지정합니다. Docker 볼륨을 사용하는 작업의 경우 `DockerVolumeConfiguration`. 바인드 탑재 호스트 볼륨을 사용하는 작업의 경우 및 `host` 선택 사항 `sourcePath`입니다. 에 대한 자세한 `host` 내용과 선택 사항은 `sourcePath`볼륨 및 작업에서 데이터 볼륨 사용을 참조하십시오.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "DockerVolumeConfiguration" : DockerVolumeConfiguration,
  "EFSVolumeConfiguration" : EFSVolumeConfiguration,
  "Host" : HostVolumeProperties,
  "Name" : String
}
```

## 속성

`DockerVolumeConfiguration`  
이 파라미터는 Docker 볼륨을 사용할 때 지정됩니다.  
Windows 컨테이너는 드라이버 사용만 지원합니다 `local` . 바인드 탑재를 사용하려면 `host` 대신 매개 변수를 지정합니다.  
Docker 볼륨은 AWS Fargate에서 실행되는 태스크에서 지원되지 않습니다.
*필수 항목 *여부: 아니요  
_형식_: DockerVolumeConfiguration  
_업데이트 필요 사항_: 바꿔 놓음

`EFSVolumeConfiguration`  
이 파라미터는 작업 스토리지에 Amazon Elastic File System 파일 시스템을 사용할 때 지정됩니다.  
*필수 항목 *여부: 아니요  
_형식_: EFSVolumeConfiguration  
_업데이트 필요 사항_: 바꿔 놓음

`Host`  
이 매개변수는 바인드 탑재 호스트 볼륨을 사용할 때 지정됩니다. 파라미터의 내용은 `host` 바인드 탑재 호스트 볼륨이 호스트 컨테이너 인스턴스에서 유지되는지 여부와 저장 위치를 결정합니다. `host` 파라미터가 비어 있으면 Docker 데몬이 데이터 볼륨에 대한 호스트 경로를 할당합니다. 그러나 연결된 컨테이너의 실행이 중지된 후에도 데이터가 유지되지 않을 수 있습니다.  
Windows 컨테이너는 와 동일한 드라이브에 전체 디렉터리를 탑재할 수 있습니다`$env:ProgramData`. Windows 컨테이너는 다른 드라이브에 디렉터리를 탑재할 수 없으며 탑재 지점은 드라이브 간에 있을 수 없습니다. 예를 들어, 및 를 마운트할 수 `C:mypath:C:mypath` 있지만 `D::D:` 또는 는 `D:mypath:C:mypath`마운트할 수 없습니다`D::C:mypath`.  
*필수 항목 *여부: 아니요  
_유형_: HostVolumeProperties  
_업데이트 필요 사항_: 바꿔 놓음

`Name`  
볼륨의 이름입니다. 최대 255개의 문자(대문자 및 소문자), 숫자, 밑줄 및 하이픈이 허용됩니다. 이 이름은 `sourceVolume` 컨테이너 정의 의 매개 변수 `mountPoints` 에서 참조됩니다.  
이는 Amazon EFS 볼륨을 사용할 때 필요합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음
