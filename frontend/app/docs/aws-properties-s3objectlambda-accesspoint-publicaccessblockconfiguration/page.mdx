# AWS::S3ObjectLambda::AccessPoint PublicAccessBlockConfiguration

이 `PublicAccessBlock` Amazon S3 계정에 적용할 구성입니다. 어떤 조합으로든 구성 옵션을 활성화할 수 있습니다. Amazon S3가 버킷 또는 객체를 퍼블릭으로 간주하는 경우에 대한 자세한 내용은 _Amazon S3 사용 설명서의 "퍼블릭"의 의미를 참조하십시오_.

이 데이터 형식은 Amazon S3 on Outposts에서 지원되지 않습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "BlockPublicAcls" : Boolean,
  "BlockPublicPolicy" : Boolean,
  "IgnorePublicAcls" : Boolean,
  "RestrictPublicBuckets" : Boolean
}
```

## 속성

`BlockPublicAcls`  
Amazon S3에서 이 계정의 버킷에 대한 퍼블릭 ACL(액세스 제어 목록)을 차단해야 하는지 여부를 지정합니다. 이 요소를 다음과 같이 설정하면 `TRUE` 다음과 같은 동작이 발생합니다.

- `PutBucketAcl` `PutObjectAcl` 지정된 ACL이 공용인 경우 호출이 실패합니다.
- 요청에 퍼블릭 ACL이 포함된 경우 PUT Object 호출이 실패합니다.
- 요청에 퍼블릭 ACL이 포함된 경우 PUT Bucket 호출이 실패합니다.
  이 설정을 사용하도록 설정해도 기존 정책 또는 ACL에는 영향을 주지 않습니다.  
  Amazon S3 on Outposts에서는 이 속성이 지원되지 않습니다.  
  *필수 항목 *여부: 아니요  
  _유형_: Boolean  
  _업데이트 필요 사항_: 중단 없음

`BlockPublicPolicy`  
Amazon S3가 이 계정의 버킷에 대한 퍼블릭 버킷 정책을 차단해야 하는지 여부를 지정합니다. 이 요소를 로 설정하면 `TRUE` 지정된 버킷 정책에서 퍼블릭 액세스를 허용하는 경우 Amazon S3가 PUT 버킷 정책에 대한 호출을 거부합니다.  
이 설정을 활성화해도 기존 버킷 정책에는 영향을 주지 않습니다.  
Amazon S3 on Outposts에서는 이 속성이 지원되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`IgnorePublicAcls`  
Amazon S3가 이 계정의 버킷에 대한 퍼블릭 ACL을 무시해야 하는지 여부를 지정합니다. 이 요소를 설정하면 `TRUE` Amazon S3가 이 계정의 버킷에 있는 모든 퍼블릭 ACL과 버킷에 포함된 모든 객체를 무시합니다.  
이 설정을 사용하도록 설정해도 기존 ACL의 지속성에 영향을 주지 않으며 새 퍼블릭 ACL이 설정되는 것을 방지하지 않습니다.  
Amazon S3 on Outposts에서는 이 속성이 지원되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`RestrictPublicBuckets`  
Amazon S3에서 이 계정의 버킷에 대한 퍼블릭 버킷 정책을 제한해야 하는지 여부를 지정합니다. 이 요소를 설정하여 `TRUE` 퍼블릭 정책이 있는 버킷에 대한 액세스는 이 계정 내의 AWS 서비스 보안 주체 및 권한 있는 사용자로만 제한됩니다.  
이 설정을 활성화해도 특정 계정에 대한 비공개 위임을 포함하여 퍼블릭 버킷 정책 내의 퍼블릭 및 교차 계정 액세스가 차단된다는 점을 제외하고는 이전에 저장된 버킷 정책에 영향을 주지 않습니다.  
Amazon S3 on Outposts에서는 이 속성이 지원되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음
