# AWS::RDS::D BCluster ReadEndpoint

`ReadEndpoint` 반환 값은 DB 클러스터의 리더 엔드포인트를 지정합니다.

DB 클러스터의 리더 엔드포인트는 DB 클러스터에서 사용할 수 있는 Aurora 복제본 간에 연결을 로드 밸런싱합니다. 클라이언트가 리더 엔드포인트에 대한 새 연결을 요청하면 Aurora는 DB 클러스터의 Aurora 복제본 간에 연결 요청을 분산합니다. 이 기능은 DB 클러스터의 여러 Aurora 복제본에 걸쳐 읽기 워크로드의 균형을 맞추는 데 도움이 될 수 있습니다.

장애 조치가 발생하고 연결된 Aurora 복제본이 기본 인스턴스로 승격되면 연결이 끊어집니다. 읽기 워크로드를 클러스터의 다른 Aurora 복제본으로 계속 전송하려면 리더 엔드포인트에 다시 연결할 수 있습니다.

Aurora 엔드포인트에 대한 자세한 내용은 _Amazon Aurora 사용 설명서의 Amazon Aurora 연결 관리를 참조하세요_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Address" : String
}
```

## 속성

`Address`  
판독기 엔드포인트의 호스트 주소입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
