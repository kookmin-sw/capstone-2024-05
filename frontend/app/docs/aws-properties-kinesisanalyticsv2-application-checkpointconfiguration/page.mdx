# AWS::KinesisAnalyticsV2::애플리케이션 체크포인트 구성

응용 프로그램의 검사점 구성에 대해 설명합니다. 검사점은 내결함성을 위해 응용 프로그램 상태를 유지하는 프로세스입니다. 자세한 내용은 Apache Flink 설명서의 Checkpoints for Fault Tolerance를 참조하십시오.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "CheckpointingEnabled" : Boolean,
  "CheckpointInterval" : Integer,
  "ConfigurationType" : String,
  "MinPauseBetweenCheckpoints" : Integer
}
```

## 속성

`CheckpointingEnabled`  
Managed Service for Apache Flink 애플리케이션에 대해 체크포인트가 사용 가능한지 여부를 설명합니다.  
`CheckpointConfiguration.ConfigurationType` is 인 `DEFAULT` 경우 이 API를 사용하거나 응용 프로그램 코드에서 이 값을 다른 값으로 설정하더라도 응용 프로그램은 `CheckpointingEnabled` 값을 `true` 사용합니다.
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`CheckpointInterval`  
검사점 작업 사이의 간격(밀리초)을 설명합니다.  
`CheckpointConfiguration.ConfigurationType` is 인 `DEFAULT` 경우 이 `CheckpointInterval` API를 사용하거나 애플리케이션 코드에서 이 값을 다른 값으로 설정하더라도 애플리케이션은 값 60000을 사용합니다.
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`ConfigurationType`  
애플리케이션이 Managed Service for Apache Flink의 기본 체크포인트 동작을 사용하는지 여부를 설명합니다. `CUSTOM`, `CheckpointingEnabled`또는 `CheckpointInterval` 매개 변수를 설정하려면 `MinPauseBetweenCheckpoints` 이 속성을 로 설정해야 합니다.  
이 값을 로 설정하면 `DEFAULT`API 또는 애플리케이션 코드를 사용하여 다른 값으로 설정된 경우에도 애플리케이션은 다음 값을 사용합니다.

- **CheckpointingEnabled:** 참
- **체크포인트 간격:** 60000
- **MinPauseBetweenCheckpoints:** 5000
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _허용되는 값_: `CUSTOM | DEFAULT`  
  _업데이트 필요 사항_: 중단 없음

`MinPauseBetweenCheckpoints`  
검사점 작업이 완료된 후 새 검사점 작업을 시작할 수 있는 최소 시간(밀리초)을 설명합니다. 검사점 작업이 보다 오래 걸리면 `CheckpointInterval`응용 프로그램은 연속적인 검사점 작업을 수행합니다. 자세한 내용은 Apache Flink 문서의 Tuning Checkpointing을 참조하십시오.  
`CheckpointConfiguration.ConfigurationType` is 인 `DEFAULT` 경우 `MinPauseBetweenCheckpoints` 이 API를 사용하거나 애플리케이션 코드에서 이 값을 설정한 경우에도 애플리케이션은 값 5000을 사용합니다.
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음
