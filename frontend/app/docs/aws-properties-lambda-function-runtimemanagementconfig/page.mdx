# AWS::람다::함수 런타임 관리 구성

함수 버전에 대한 런타임 관리 구성을 설정합니다. 자세한 내용은 런타임 업데이트를 참조하세요.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "RuntimeVersionArn" : String,
  "UpdateRuntimeOn" : String
}
```

## 속성

`RuntimeVersionArn`  
함수에서 사용할 런타임 버전의 ARN입니다.  
**수동** 런타임 업데이트 모드를 사용하는 경우에만 필요합니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `26`  
_최대_: `2048`  
_패턴_: `^arn:(aws[a-zA-Z-]*):lambda:[a-z]{2}((-gov)|(-iso(b?)))?-[a-z]+-d{1}::runtime:.+$`  
_업데이트 필요 사항_: 중단 없음

`UpdateRuntimeOn` 런타임 업데이트 모드를 지정합니다.

- **자동(기본값)** - 2단계 런타임 버전 롤아웃을 사용하여 가장 안전한 최신 런타임 버전으로 자동 업데이트합니다. 이는 대부분의 고객이 런타임 업데이트의 이점을 항상 누릴 수 있도록 하는 최상의 선택입니다.
- **FunctionUpdate** - 함수를 업데이트할 때 Lambda는 함수의 실행 시간을 가장 안전한 최신 실행 시간 버전으로 업데이트합니다. 이 접근 방식은 런타임 업데이트를 함수 배포와 동기화하여 런타임 업데이트가 적용되는 시기를 제어하고 드문 런타임 업데이트 비호환성을 조기에 감지하고 완화할 수 있도록 합니다. 이 설정을 사용하는 경우 런타임을 최신 상태로 유지하기 위해 함수를 정기적으로 업데이트해야 합니다.
- **Manual** - 함수 구성에서 런타임 버전을 지정합니다. 함수는 이 런타임 버전을 무기한으로 사용합니다. 드문 경우지만 새 런타임 버전이 기존 함수와 호환되지 않는 경우 함수를 이전 런타임 버전으로 롤백할 수 있습니다. 자세한 내용은 런타임 버전 롤백을 참조하세요.
  _유효한 값 _: `Auto` | `FunctionUpdate` | `Manual`  
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _업데이트 필요 사항_: 중단 없음
