# AWS::EFS::파일 시스템 수명 주기 정책

EFS 수명 주기 관리 및 EFS Intelligent-Tiering에서 사용하는 정책으로, 파일 시스템의 Infrequent Access(IA) 스토리지 클래스로 파일을 전환하거나 외부로 전환할 시기를 지정합니다. 자세한 내용은 EFS Intelligent‐Tiering 및 EFS 수명 주기 관리 단원을 참조하십시오.

**메모**  
각 `LifecyclePolicy` 오브젝트에는 전환이 하나만 있을 수 있습니다. 즉, 요청 본문에서 `LifecyclePolicies` 각 전환에 대해 하나의 개체인 , `LifecyclePolicy`의 `TransitionToIA` 배열로 구성되어야 합니다`TransitionToPrimaryStorageClass`.
올바른 구조는 AWS::EFS::FileSystem 예제를 참조하십시오 `LifecyclePolicy` . 이 페이지에 표시된 구문을 사용하지 마십시오.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "TransitionToIA" : String,
  "TransitionToPrimaryStorageClass" : String
}
```

## 속성

`TransitionToIA`  
 파일에 액세스하지 않고 IA 스토리지로 전환되는 기간을 설명합니다. 디렉터리의 내용을 나열하는 것과 같은 메타데이터 작업은 파일 액세스 이벤트로 계산되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `AFTER_14_DAYS | AFTER_1_DAY | AFTER_30_DAYS | AFTER_60_DAYS | AFTER_7_DAYS | AFTER_90_DAYS`  
_업데이트 필요 사항_: 중단 없음

`TransitionToPrimaryStorageClass`  
IA 스토리지에서 기본 스토리지로 파일을 전환하는 경우에 대해 설명합니다. 디렉터리의 내용을 나열하는 것과 같은 메타데이터 작업은 파일 액세스 이벤트로 계산되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `AFTER_1_ACCESS`  
_업데이트 필요 사항_: 중단 없음
