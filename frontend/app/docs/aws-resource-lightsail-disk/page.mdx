# AWS::Lightsail::D isk

`AWS::Lightsail::Disk` 리소스는 동일한 AWS 리전 및 가용 영역에 있는 Amazon Lightsail 인스턴스에 연결할 수 있는 디스크를 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Lightsail::Disk",
  "Properties" : {
      "AddOns" : [ AddOn, ... ],
      "AvailabilityZone" : String,
      "DiskName" : String,
      "Location" : Location,
      "SizeInGb" : Integer,
      "Tags" : [Tag, ... ]
    }
}
```

## 속성

`AddOns`  
디스크에 대한 추가 기능의 배열입니다.  
디스크 삭제 요청을 수행할 때 디스크에 추가 기능이 활성화되어 있으면 디스크가 삭제되기 전에 추가 기능이 자동으로 비활성화됩니다.
*필수 항목 *여부: 아니요  
_유형_: 애드온 목록  
_업데이트 필요 사항_: 중단 없음

`AvailabilityZone`  
디스크의 AWS 리전 및 가용 영역 위치(예: `us-east-1a`).  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`DiskName`  
디스크의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_패턴_: `w[w-]*w`  
*업데이트 필요 *사항: 바꿔 놓음

`Location`  
속성 설명을 사용할 수 없습니다.  
*필수 항목 *여부: 아니요  
_유형_: 위치  
_업데이트 필요 사항_: 중단 없음

`SizeInGb`  
디스크의 크기(GB)입니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`Tags`  
이 리소스에 적용할 키-값 쌍의 배열입니다.  
자세한 내용은 _AWS CloudFormation 사용 설명서의 태그를 참조하세요_.  
`Value` of `Tags` 는 Lightsail 리소스의 선택 사항입니다.
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`이 `Ref`리소스에 대한 고유 식별자를 반환합니다.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`AttachedTo` 디스크가 연결된 인스턴스입니다.

`AttachmentState`  
(더 이상 사용되지 않음) 디스크의 연결 상태입니다.  
2017년 11월 14일 이전 릴리스에서는 이 매개 변수가 `attached` API 응답의 시스템 디스크에 대해 반환되었습니다. 이제 더 이상 사용되지 않지만 여전히 응답에 포함되어 있습니다. 대신 사용하십시오 `isAttached` .

`DiskArn` 디스크의 Amazon 리소스 이름(ARN)입니다.

`Iops` 디스크의 IOPS(초당 입출력 작업 수)입니다.

`IsAttached` 디스크가 인스턴스에 연결되어 있는지 여부를 나타내는 부울 값입니다.

`Location.AvailabilityZone` 속성 설명을 사용할 수 없습니다.

`Location.RegionName` 속성 설명을 사용할 수 없습니다.

`Path` 디스크의 경로입니다.

`ResourceType`  
디스크의 리소스 종류(예: `Disk`).

`State`  
디스크의 상태(예: `in-use`).

`SupportCode` 디스크의 지원 코드입니다.  
Lightsail의 디스크 또는 다른 리소스에 대한 질문이 있을 때 지원할 수 있도록 이메일에 이 코드를 포함하세요. 이 코드는 지원 팀이 Lightsail 정보를 조회하는 데 도움이 됩니다.

## 발언

_가용 영역_

디스크 생성 요청을 수행할 때 가용 영역을 지정할 수 있습니다. 지정하지 않으면 디스크가 마지막으로 생성한 Lightsail 리소스와 동일한 가용 영역에 생성됩니다.

_디스크 상태_

디스크는 상태일 때만 삭제할 수 있습니다 `available` . `attached` 디스크 삭제 요청을 수행할 때 디스크가 상태인 경우 서비스는 디스크 상태가 로 변경되는지 확인하기 위해 대기 합니다`available`. 디스크 상태가 15분 이내로 변경되지 않으면 디스크 삭제 요청 시간이 초과됩니다 `available` .
