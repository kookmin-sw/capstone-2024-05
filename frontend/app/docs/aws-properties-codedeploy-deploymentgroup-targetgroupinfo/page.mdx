# AWS::CodeDeploy::D eploymentGroup TargetGroupInfo

`TargetGroupInfo` 속성 유형은 배포에 사용할 Elastic Load Balancing의 대상 그룹에 대한 정보를 지정합니다. 인스턴스는 대상 그룹에 대상으로 등록되고 트래픽은 대상 그룹으로 라우팅됩니다. 자세한 내용은 _AWS CodeDeploy API 참조_ 의 TargetGroupInfo를 참조하십시오

속성을 `TargetGroupInfo`지정하는 경우 `DeploymentStyle.DeploymentOption` CodeDeploy가 지정된 대상 그룹을 사용하여 트래픽을 라우팅할 수 있도록`WITH_TRAFFIC_CONTROL` 속성을 설정해야 합니다.

`TargetGroupInfo` 은 LoadBalancerInfo 속성 형식의 속성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Name" : String
}
```

## 속성

`Name`  
블루/그린 배포의 경우, 원래 환경의 인스턴스가 등록 취소된 대상 그룹의 이름과 대체 환경의 인스턴스가 등록됩니다. 인플레이스(in-place) 배포의 경우, 인스턴스가 등록 취소되어 배포 중에 트래픽을 처리하지 않고 배포가 완료된 후 다시 등록되는 대상 그룹의 이름입니다. 중복은 허용되지 않습니다.  
AWS CloudFormation은 AWS Lambda 컴퓨팅 플랫폼에서만 블루/그린 배포를 지원합니다.
이 값은 32자를 초과할 수 없으므로 다음 예제와 같이 `Name` 대상 그룹의 속성 또는 내장 함수 `TargetGroupName` 가 있는 특성을 `Fn::GetAtt` 사용해야 합니다 . 그룹의 Amazon 리소스 이름(ARN) 또는 속성을 사용하지 마십시오 `TargetGroupFullName` .
