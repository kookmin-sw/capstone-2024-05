# AWS::CloudFormation::StackSet 작업 기본 설정

AWS CloudFormation에서 스택 세트 작업을 수행하는 방법에 대한 사용자 지정 기본 설정입니다. 최대 동시 계정 및 장애 허용성에 대한 자세한 내용은 스택 세트 작업 옵션 섹션을 참조하세요.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "FailureToleranceCount" : Integer,
  "FailureTolerancePercentage" : Integer,
  "MaxConcurrentCount" : Integer,
  "MaxConcurrentPercentage" : Integer,
  "RegionConcurrencyType" : String,
  "RegionOrder" : [ String, ... ]
}
```

## 속성

`FailureToleranceCount`  
AWS CloudFormation이 해당 리전에서 작업을 중지하기 전에 이 작업이 실패할 수 있는 리전당 계정 수입니다. 리전에서 작업이 중지된 경우 AWS CloudFormation은 후속 리전에서 작업을 시도하지 않습니다.  
조건부: `FailureToleranceCount` OR 중 하나 `FailureTolerancePercentage` (둘 다 지정하지는 않음)를 지정해야 합니다.  
_필수_: 조건부  
_유형_: Integer  
_최소_: `0`  
_업데이트 필요 사항_: 중단 없음

`FailureTolerancePercentage`  
AWS CloudFormation이 해당 리전에서 작업을 중지하기 전에 이 스택 작업이 실패할 수 있는 리전당 계정의 비율입니다. 리전에서 작업이 중지된 경우 AWS CloudFormation은 후속 리전에서 작업을 시도하지 않습니다.  
지정된 백분율을 기준으로 계정 수를 계산할 때 AWS CloudFormation에서는 \** 다음 정수로 내림합니다.  
조건부: 또는 둘 중 하나를 지정해야 하지만 `FailureToleranceCount` `FailureTolerancePercentage`둘 다 지정해서는 안 됩니다.  
*필수*: 조건부  
*유형*: Integer  
*최소*: `0`  
*최대*: `100`  
*업데이트 필요 사항\*: 중단 없음

`MaxConcurrentCount`  
한 번에 이 작업을 수행할 수 있는 최대 계정 수입니다. 이는 의 값에 따라 달라집니다`FailureToleranceCount`. `MaxConcurrentCount` 는 기껏해야 보다 하나 더 많습니다`FailureToleranceCount`.  
이 설정을 사용하면 작업의 최대값을 지정할 수 \** 있습니다. 대규모 배포의 경우 특정 상황에서 서비스 제한으로 인해 동시에 작동하는 실제 계정 수가 더 적을 수 있습니다.  
조건부: 또는 둘 중 하나를 지정해야 하지만 `MaxConcurrentCount` `MaxConcurrentPercentage`둘 다 지정해서는 안 됩니다.  
*필수*: 조건부  
*유형*: Integer  
*최소*: `1`  
*업데이트 필요 사항\*: 중단 없음

`MaxConcurrentPercentage`  
한 번에 이 작업을 수행할 수 있는 계정의 최대 백분율입니다.  
지정된 백분율을 기준으로 계정 수를 계산할 때 AWS CloudFormation에서는 다음 정수로 내림합니다. 반올림이 0인 경우를 제외하고는 마찬가지입니다. 이 경우 CloudFormation은 숫자를 대신 1로 설정합니다.  
이 설정을 사용하면 작업의 최대값을 지정할 수 \** 있습니다. 대규모 배포의 경우 특정 상황에서 서비스 제한으로 인해 동시에 작동하는 실제 계정 수가 더 적을 수 있습니다.  
조건부: 또는 둘 중 하나를 지정해야 하지만 `MaxConcurrentCount` `MaxConcurrentPercentage`둘 다 지정해서는 안 됩니다.  
*필수*: 조건부  
*유형*: Integer  
*최소*: `1`  
*최대*: `100`  
*업데이트 필요 사항\*: 중단 없음

`RegionConcurrencyType`  
리전에서 StackSets 작업을 배포하는 동시성 유형은 병렬로 또는 한 번에 하나의 리전일 수 있습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `PARALLEL | SEQUENTIAL`  
_업데이트 필요 사항_: 중단 없음

`RegionOrder`  
스택 작업을 수행할 리전의 순서입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음
