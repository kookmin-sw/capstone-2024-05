# AWS::SageMaker::ModelBiasJobDefinition 중지 조건

모델 학습 작업 또는 모델 컴파일 작업을 실행할 수 있는 기간에 대한 제한을 지정합니다. 또한 관리형 스팟 훈련 작업을 완료해야 하는 기간도 지정합니다. 작업이 시간 제한에 도달하면 SageMaker는 훈련 또는 컴파일 작업을 종료합니다. 이 API를 사용하여 모델 학습 비용을 제한합니다.

훈련 작업을 중지하기 위해 SageMaker는 알고리즘에 신호를 보내 `SIGTERM` 작업 종료를 120초 동안 지연시킵니다. 알고리즘은 이 120초 창을 사용하여 모델 아티팩트를 저장할 수 있으므로 학습 결과가 손실되지 않습니다.

SageMaker에서 제공하는 훈련 알고리즘은 가능한 경우 모델 훈련 작업의 중간 결과를 자동으로 저장합니다. 아티팩트를 저장하려는 이 시도는 모델이 저장할 수 있는 상태가 아닐 수 있으므로 최선의 경우일 뿐입니다. 예를 들어 학습이 방금 시작된 경우 모델을 저장할 준비가 되지 않았을 수 있습니다. 저장될 때 이 중간 데이터는 유효한 모델 아티팩트입니다. 이를 사용하여 를 사용하여 모델을 만들 수 있습니다 `CreateModel`.

**메모**  
NTM(Neural Topic Model)은 현재 중간 모델 아티팩트 저장을 지원하지 않습니다. NTM을 학습할 때 최대 런타임이 학습 작업을 완료하기에 충분한지 확인합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "MaxRuntimeInSeconds" : Integer
}
```

## 속성

`MaxRuntimeInSeconds`  
학습 또는 컴파일 작업이 중지되기 전에 실행할 수 있는 최대 시간(초)입니다.  
컴파일 작업의 경우 이 시간 동안 작업이 완료되지 않으면 `TimeOut` 오류가 생성됩니다. 900초부터 시작하여 모델에 따라 필요에 따라 늘리는 것이 좋습니다.  
다른 모든 작업의 경우 이 시간 동안 작업이 완료되지 않으면 SageMaker가 작업을 종료합니다. `RetryStrategy` 작업 요청에 지정된 `MaxRuntimeInSeconds` 경우 각 개별 시도가 아닌 총 모든 시도에 대한 최대 시간을 지정합니다. 기본값은 1일입니다. 최대값은 28일입니다.  
중지된 `TrainingJob` 후 지표를 게시하거나 모델을 보관 및 업로드하는 데 소요된 시간을 포함하여 총 실행할 수 있는 최대 시간은 30일입니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_최소_: `1`  
_업데이트 필요 사항_: 바꿔 놓음
