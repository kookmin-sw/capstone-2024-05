# AWS::S3::버킷 IntelligentTieringConfiguration

Amazon S3 버킷에 대한 S3 Intelligent-Tiering 구성을 지정합니다.

S3 Intelligent-Tiering 스토리지 클래스에 대한 자세한 내용은 자주 액세스하는 객체와 자주 액세스하지 않는 객체를 자동으로 최적화하기 위한 스토리지 클래스 섹션을 참조하세요.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Id" : String,
  "Prefix" : String,
  "Status" : String,
  "TagFilters" : [ TagFilter, ... ],
  "Tierings" : [ Tiering, ... ]
}
```

## 속성

`Id`  
S3 Intelligent-Tiering 구성을 식별하는 데 사용되는 ID입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Prefix`  
규칙이 적용되는 객체의 하위 집합을 식별하는 객체 키 이름 접두사입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Status`  
구성의 상태를 지정합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `Disabled | Enabled`  
_업데이트 필요 사항_: 중단 없음

`TagFilters`  
키-값 페어의 컨테이너입니다.  
*필수 항목 *여부: 아니요  
_유형_: TagFilter 목록  
_업데이트 필요 사항_: 중단 없음

`Tierings`  
구성에서 S3 Intelligent-Tiering 스토리지 클래스 계층 목록을 지정합니다. 목록에 하나 이상의 계층이 정의되어 있어야 합니다. 목록에서 사용 가능한 각 AccessTier에 대해 하나씩 최대 두 개의 계층을 지정할 수 있습니다. `ARCHIVE_ACCESS` `DEEP_ARCHIVE_ACCESS`  
Intelligent-Tiering 스토리지 클래스에 저장된 객체를 Archive Access 또는 Deep Archive Access 계층으로 자동으로 이동하려는 경우에만 버킷에서 Intelligent Tiering Configuration을 활성화하면 됩니다.
_필수 항목 여부_: 예  
_유형_: 계층화 목록  
_업데이트 필요 사항_: 중단 없음
