# AWS::CloudFront::ContinuousDeploymentPolicy SessionStickinessConfig

세션 고정성은 단일 뷰어의 여러 요청을 단일 세션으로 정의할 수 있는 기능을 제공합니다. 이렇게 하면 지정된 사용자의 요청 중 일부는 스테이징 배포로 전송되고 다른 요청은 기본 배포로 전송되는 잠재적으로 일관되지 않은 경험을 방지할 수 있습니다. TTL 값을 사용하여 세션 기간을 정의합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "IdleTTL" : Integer,
  "MaximumTTL" : Integer
}
```

## 속성

`IdleTTL`  
요청이 수신되지 않을 경우 세션을 중단할 때까지의 시간입니다. 허용되는 값은 300-3600초(5-60분)입니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`MaximumTTL`  
최종 사용자의 요청을 동일한 세션의 일부로 간주하는 최대 시간입니다. 허용되는 값은 300-3600초(5-60분)입니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음
