# 위임된 관리자 등록

조직의 관리 계정 외에도 위임된 관리자 권한이 있는 멤버 계정은 조직에 대한 서비스 관리형 권한이 있는 스택 세트를 생성하고 관리할 수 있습니다. 서비스 관리형 권한이 있는 스택 세트는 위임된 관리자가 생성한 스택 세트를 포함하여 관리 계정에 생성됩니다. 조직의 위임된 관리자로 등록하려면 멤버 계정이 조직에 있어야 합니다. 조직 가입에 대한 자세한 내용은 조직에 가입하도록 AWS 계정 초대 섹션을 참조하세요.

조직에는 한 번에 최대 5명의 등록된 위임된 관리자가 있을 수 있습니다. 위임된 관리자는 조직의 모든 계정 또는 특정 OU에 배포하도록 선택할 수 있습니다. AWS Organizations를 통한 신뢰할 수 있는 액세스는 위임된 관리자가 Organizations에서 관리하는 계정에 배포하기 전에 활성화되어야 합니다. 자세한 내용은 AWS Organizations를 사용하여 신뢰할 수 있는 액세스 활성화를 참조하세요.

**중요하다**  
위임된 관리자는 조직의 계정에 배포할 수 있는 모든 권한을 갖습니다. 관리 계정은 특정 OU에 배포하거나 특정 스택 세트 작업을 수행할 수 있는 위임된 관리자 권한을 제한할 수 없습니다.

미국 동부(오하이오), 미국 동부(버지니아 북부), 미국 서부(캘리포니아 북부), 미국 서부(오레곤), 아시아 태평양(뭄바이), 아시아 태평양(서울), 아시아 태평양(싱가포르), 아시아 태평양(시드니), 아시아 태평양(도쿄), 캐나다(중부), 유럽(프랑크푸르트), 유럽(아일랜드), 유럽(런던), 유럽(파리), 유럽(스톡홀름), 남아메리카(상파울루) 리전에서 조직에 대해 위임된 관리자를 등록할 수 있습니다. AWS GovCloud(미국 동부) 및 AWS GovCloud(미국 서부).

AWS CloudFormation 콘솔, AWS CLI 또는 AWS SDK를 사용하여 위임된 관리자를 등록 및 등록 취소할 수 있습니다.

**위임된 관리자를 등록하려면(콘솔)**

1. 관리 계정의 관리자로 AWS에 로그인하고 some-address에서 AWS CloudFormation 콘솔을 엽니다

1. 탐색 창에서 [StackSets**]를 선택합니다\*\*.

1. 위임된 관리자(Delegated administrators**)에서 **위임된 관리자 등록(Register delegated administrator**)을 선택합니다**.

1. **[Register delegated administrator**] 대화 상자에서 **[Register delegated administrator**]를 선택합니다.

성공 메시지는 멤버 계정이 위임된 관리자로 성공적으로 등록되었음을 나타냅니다.

**위임된 관리자의 등록을 취소하려면(콘솔)**

1. 관리 계정의 관리자로 AWS에 로그인하고 some-address에서 AWS CloudFormation 콘솔을 엽니다

1. 탐색 창에서 [StackSets**]를 선택합니다\*\*.

1. **Delegated administrators(위임된 관리자**)에서 등록 취소할 계정을 선택한 다음 **Deregister(등록 취소**)를 선택합니다.

성공 메시지는 멤버 계정이 위임된 관리자로 성공적으로 등록 취소되었음을 나타냅니다.

언제든지 이 계정을 다시 등록할 수 있습니다.

**위임된 관리자를 등록하려면(AWS CLI)**

1. AWS CLI를 엽니다.

1. 명령을 실행합니다 `register-delegated-administrator` .

   ```
   aws organizations register-delegated-administrator \
     --service-principal=member.org.stacksets.cloudformation.amazonaws.com \
     --account-id="memberAccountId"
   ```

1. `list-delegated-administrators` 명령을 실행하여 지정된 멤버 계정이 위임된 관리자로 성공적으로 등록되었는지 확인합니다.

```
aws organizations list-delegated-administrators \
  --service-principal=member.org.stacksets.cloudformation.amazonaws.com
```

**위임된 관리자의 등록을 취소하려면(AWS CLI)**

1. AWS CLI를 엽니다.

1. 명령을 실행합니다 `deregister-delegated-administrator` .

   ```
   aws organizations deregister-delegated-administrator \
     --service-principal=member.org.stacksets.cloudformation.amazonaws.com \
     --account-id="memberAccountId"
   ```

1. 명령을 실행하여 `list-delegated-administrators` 지정된 멤버 계정이 위임된 관리자로 성공적으로 등록 취소되었는지 확인합니다.

   ```
   aws organizations list-delegated-administrators \
     --service-principal=member.org.stacksets.cloudformation.amazonaws.com
   ```

   언제든지 이 계정을 다시 등록할 수 있습니다.
