# AWS::ECS::P rimary태스크셋

서비스에서 기본 작업 집합인 작업 집합을 수정합니다. 서비스의 기본 작업 집합에서 업데이트되는 모든 매개 변수는 서비스로 전환됩니다. 서비스에서 `EXTERNAL` 배포 컨트롤러 유형을 사용할 때 사용됩니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Amazon ECS 배포 유형을 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::ECS::PrimaryTaskSet",
  "Properties" : {
      "Cluster" : String,
      "Service" : String,
      "TaskSetId" : String
    }
}
```

## 속성

`Cluster`  
작업 세트가 있는 서비스를 호스팅하는 클러스터의 짧은 이름 또는 전체 Amazon 리소스 이름(ARN)입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Service`  
작업 세트가 있는 서비스의 짧은 이름 또는 전체 Amazon 리소스 이름(ARN)입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`TaskSetId`  
배포에서 기본 작업 세트로 설정할 작업 세트의 짧은 이름 또는 전체 Amazon 리소스 이름(ARN)입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`리소스 이름이 반환됩니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.
