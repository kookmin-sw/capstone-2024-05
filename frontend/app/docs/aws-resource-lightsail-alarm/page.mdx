# AWS::Lightsail::경보

리소스는 `AWS::Lightsail::Alarm` Lightsail 리소스 중 하나에 대한 단일 지표를 모니터링하는 데 사용할 수 있는 경보를 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Lightsail::Alarm",
  "Properties" : {
      "AlarmName" : String,
      "ComparisonOperator" : String,
      "ContactProtocols" : [ String, ... ],
      "DatapointsToAlarm" : Integer,
      "EvaluationPeriods" : Integer,
      "MetricName" : String,
      "MonitoredResourceName" : String,
      "NotificationEnabled" : Boolean,
      "NotificationTriggers" : [ String, ... ],
      "Threshold" : Double,
      "TreatMissingData" : String
    }
}
```

## 속성

`AlarmName`  
경보의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`ComparisonOperator`  
지정된 통계와 임계값을 비교할 때 사용할 산술 연산입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `GreaterThanOrEqualToThreshold | GreaterThanThreshold | LessThanOrEqualToThreshold | LessThanThreshold`  
_업데이트 필요 사항_: 중단 없음

`ContactProtocols`  
알람에 대한 연락처 프로토콜(예: `Email`, `SMS` (문자 메시지) 또는 둘 다).  
_허용되는 값_: `Email` | `SMS`  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`DatapointsToAlarm`  
평가 기간 내에 경보가 상태로 전환되도록 하기 위해 위반해야 하는 데이터 포인트의 수 입니다 `ALARM` .  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`EvaluationPeriods`  
데이터가 지정된 임계값과 비교되는 기간의 수입니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`MetricName`  
경보와 연결된 지표의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `BurstCapacityPercentage | BurstCapacityTime | ClientTLSNegotiationErrorCount | CPUUtilization | DatabaseConnections | DiskQueueDepth | FreeStorageSpace | HealthyHostCount | HTTPCode_Instance_2XX_Count | HTTPCode_Instance_3XX_Count | HTTPCode_Instance_4XX_Count | HTTPCode_Instance_5XX_Count | HTTPCode_LB_4XX_Count | HTTPCode_LB_5XX_Count | InstanceResponseTime | NetworkIn | NetworkOut | NetworkReceiveThroughput | NetworkTransmitThroughput | RejectedConnectionCount | RequestCount | StatusCheckFailed | StatusCheckFailed_Instance | StatusCheckFailed_System | UnhealthyHostCount`  
_업데이트 필요_: 업데이트가 지원되지 않습니다.

`MonitoredResourceName`  
경보가 모니터링하는 Lightsail 리소스의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`NotificationEnabled`  
경보가 활성화되었는지 여부를 나타내는 부울 값입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`NotificationTriggers`  
알림을 트리거하는 알람 상태입니다.  
`OK` 및 `INSUFFICIENT_DATA` 값을 지정하려면 값도 지정해야 합니다 `ContactProtocols` . 그렇지 않으면 `OK` 및 `INSUFFICIENT_DATA` 값이 적용되지 않고 스택이 드리프트됩니다.
_허용되는 값_: `OK` | `ALARM` | `INSUFFICIENT_DATA`  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Threshold`  
지정된 통계를 비교하는 값입니다.  
_필수 항목 여부_: 예  
_유형_: 더블  
_업데이트 필요 사항_: 중단 없음

`TreatMissingData` 경보가 누락된 데이터 포인트를 처리하는 방법을 지정합니다.  
경보는 다음과 같은 방법으로 누락된 데이터를 처리할 수 있습니다.

- `breaching` - 누락된 데이터가 임계값 내에 있지 않다고 가정합니다. 누락된 데이터는 지표가 임계값 내에 있지 않은 횟수에 포함됩니다.
- `notBreaching` - 누락된 데이터가 임계값 내에 있다고 가정합니다. 누락된 데이터는 지표가 임계값 내에 있지 않은 횟수에 포함되지 않습니다.
- `ignore` - 누락된 데이터를 무시합니다. 현재 알람 상태를 유지합니다.
- `missing` - 누락된 데이터는 누락된 것으로 처리됩니다.
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _허용되는 값_: `breaching | ignore | missing | notBreaching`  
  _업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`이 `Ref`리소스에 대한 고유 식별자를 반환합니다.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`AlarmArn` 경보의 Amazon 리소스 이름(ARN)입니다.

`State` 경보의 현재 상태입니다.  
경보에는 다음과 같은 상태가 있을 수 있습니다.

- `ALARM` - 지표가 정의된 임계값을 벗어났습니다.
- `INSUFFICIENT_DATA` - 경보가 최근에 시작되었거나, 지표를 사용할 수 없거나, 지표가 경보 상태를 결정하는 데 사용할 수 있는 데이터가 충분하지 않습니다.
- `OK` - 지표가 정의된 임계값 내에 있습니다.

## 발언

_알림 트리거_

매개 변수의 및 `OK` 값을 `INSUFFICIENT_DATA`지정하려면 `NotificationTriggers` 값 도 지정해야 합니다 `ContactProtocols` . 그렇지 않으면 매개 변수의 `OK` 및 `INSUFFICIENT_DATA` 값이 `NotificationTriggers` 적용되지 않고 스택이 드리프트됩니다.
