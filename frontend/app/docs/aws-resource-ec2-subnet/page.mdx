# AWS::EC2::서브넷

지정된 VPC에 대한 서브넷을 지정합니다.

IPv4 전용 서브넷의 경우 IPv4 CIDR 블록을 지정합니다. VPC에 IPv6 CIDR 블록이 있는 경우 IPv6 전용 서브넷 또는 듀얼 스택 서브넷을 대신 생성할 수 있습니다. IPv6 전용 서브넷의 경우 IPv6 CIDR 블록을 지정합니다. 이중 스택 서브넷의 경우 IPv4 CIDR 블록과 IPv6 CIDR 블록을 모두 지정합니다.

자세한 내용은 Amazon VPC 사용 설명서*의 VPC용 서브넷을 참조하세요*.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::Subnet",
  "Properties" : {
      "AssignIpv6AddressOnCreation" : Boolean,
      "AvailabilityZone" : String,
      "AvailabilityZoneId" : String,
      "CidrBlock" : String,
      "EnableDns64" : Boolean,
      "Ipv6CidrBlock" : String,
      "Ipv6Native" : Boolean,
      "MapPublicIpOnLaunch" : Boolean,
      "OutpostArn" : String,
      "PrivateDnsNameOptionsOnLaunch" : PrivateDnsNameOptionsOnLaunch,
      "Tags" : [Tag, ... ],
      "VpcId" : String
    }
}
```

## 속성

`AssignIpv6AddressOnCreation`  
이 서브넷에서 만든 네트워크 인터페이스가 IPv6 주소를 수신하는지 여부를 나타냅니다. 기본값은 `false`입니다.  
를 지정하는 경우 `AssignIpv6AddressOnCreation` 도 지정해야 합니다`Ipv6CidrBlock`.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`AvailabilityZone`  
서브넷의 가용 영역입니다.  
이 속성을 업데이트하는 경우 속성도 업데이트해야 합니다 `CidrBlock` .  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`AvailabilityZoneId`  
서브넷의 AZ ID입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`CidrBlock`  
서브넷에 할당된 IPv4 CIDR 블록입니다.  
이 속성을 업데이트하면 새 서브넷이 만들어지고 기존 서브넷이 삭제됩니다.  
_필수_: 조건부  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`EnableDns64`  
이 서브넷의 Amazon 제공 DNS 해석기에 대한 DNS 쿼리가 IPv4 전용 대상에 대한 가상 IPv6 주소를 반환해야 하는지 여부를 나타냅니다. 자세한 내용은 Amazon Virtual Private Cloud 사용 설명서의 DNS64 및 NAT64를 참조하세요\**.  
*필수 항목 *여부: 아니요  
*유형*: Boolean  
*업데이트 필요 사항\*: 중단 없음

`Ipv6CidrBlock`  
IPv6 CIDR 블록입니다.  
를 지정하는 경우 `AssignIpv6AddressOnCreation` 도 지정해야 합니다`Ipv6CidrBlock`.  
_필수_: 조건부  
_유형_: 문자열  
_업데이트 필요 사항_: 일부 중단

`Ipv6Native`  
IPv6 전용 서브넷인지 여부를 나타냅니다. 자세한 내용은 _Amazon Virtual Private Cloud 사용 설명서의 서브넷 기본 사항을 참조하세요_.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 바꿔 놓음

`MapPublicIpOnLaunch`  
이 서브넷에서 시작된 인스턴스가 퍼블릭 IPv4 주소를 수신하는지 여부를 나타냅니다. 기본값은 `false`입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`OutpostArn`  
Outposts의 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`PrivateDnsNameOptionsOnLaunch`  
이 서브넷에서 시작된 EC2 인스턴스의 호스트 이름 유형과 DNS A 및 AAAA 레코드 인스턴스를 처리하는 방법. 자세한 내용은 *Amazon Elastic Compute Cloud 사용 설명서*의 Amazon EC2 인스턴스 호스트 이름 유형을 참조하세요.  
사용 가능한 옵션:

- EnableResourceNameDnsAAAARecord(true | false)
- EnableResourceNameDnsARecord(true | false)
- 호스트 이름 유형(ip-name | 리소스 이름)
  *필수 항목 *여부: 아니요  
  _형식_: PrivateDnsNameOptionsOnLaunch  
  _업데이트 필요 사항_: 중단 없음

`Tags`  
서브넷에 할당된 모든 태그입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`VpcId`  
서브넷이 있는 VPC의 ID입니다.  
이 속성을 업데이트하는 경우 속성도 업데이트해야 합니다 `CidrBlock` .  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`서브넷의 ID가 반환됩니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`AvailabilityZone`  
이 서브넷의 가용 영역입니다. 예를 들어, `us-east-1a`.

`AvailabilityZoneId`  
이 서브넷의 가용 영역 ID입니다. 예를 들어, `use1-az1`.

`CidrBlock` 서브넷과 연결된 IPv4 CIDR 블록입니다.

`Ipv6CidrBlocks` 서브넷과 연결된 IPv6 CIDR 블록입니다.

`NetworkAclAssociationId`  
서브넷의 VPC와 연결된 네트워크 ACL의 ID 입니다`acl-5fb85d36`.

`OutpostArn` Outposts의 Amazon 리소스 이름(ARN)입니다.

`SubnetId` 서브넷의 ID입니다.

`VpcId`  
서브넷 VPC의 ID(예: `vpc-11ad4878`.
