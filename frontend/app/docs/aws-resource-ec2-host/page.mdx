# AWS::EC2::호스트

EC2 인스턴스를 시작하기 위한 완전 전용 물리적 서버를 할당합니다. 호스트는 전적으로 사용자 전용이므로 기존 서버 바인딩 소프트웨어 라이선스를 사용할 수 있도록 하여 규정 준수 요구 사항을 해결하고 비용을 절감하는 데 도움이 될 수 있습니다. 자세한 내용은 _Linux 인스턴스용 Amazon EC2 사용 설명서의 전용 호스트를 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::Host",
  "Properties" : {
      "AssetId" : String,
      "AutoPlacement" : String,
      "AvailabilityZone" : String,
      "HostMaintenance" : String,
      "HostRecovery" : String,
      "InstanceFamily" : String,
      "InstanceType" : String,
      "OutpostArn" : String
    }
}
```

## 속성

`AssetId`  
전용 호스트가 할당된 Outpost 하드웨어 자산의 ID입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`AutoPlacement`  
호스트가 인스턴스 유형 구성과 일치하는 대상 지정되지 않은 인스턴스 시작을 허용하는지 또는 고유한 호스트 ID를 지정하는 호스트 테넌시 인스턴스 시작만 허용하는지 여부를 나타냅니다. 자세한 내용은 _Amazon EC2 사용 설명서의 자동 배치 및 선호도 이해를 참조하세요_.  
기본값: `on`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `off | on`  
_업데이트 필요 사항_: 중단 없음

`AvailabilityZone`  
전용 호스트를 할당할 가용 영역입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`HostMaintenance`  
전용 호스트에 대해 호스트 유지 보수가 활성화되었는지 또는 비활성화되었는지를 나타냅니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `off | on`  
_업데이트 필요 사항_: 중단 없음

`HostRecovery`  
전용 호스트에 대한 호스트 복구를 활성화할지 또는 비활성화할지를 나타냅니다. 호스트 복구는 기본적으로 사용하지 않도록 설정되어 있습니다. 자세한 내용은 _Amazon EC2 사용 설명서의 호스트 복구를 참조하세요_.  
기본값: `off`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `off | on`  
_업데이트 필요 사항_: 중단 없음

`InstanceFamily`  
전용 호스트에서 지원하는 인스턴스 패밀리입니다. 예를 들어, `m5`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`InstanceType`  
전용 호스트에서 지원할 인스턴스 유형을 지정합니다. 인스턴스 유형을 지정하는 경우 전용 호스트는 지정된 인스턴스 유형의 인스턴스만 지원합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`OutpostArn`  
전용 호스트가 할당된 AWS Outposts의 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`는 `Ref`호스트 ID(예: )를 반환합니다 `h-0ab123c45d67ef89`.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`HostId` 호스트의 ID입니다.

## 예제

### 전용 호스트 할당

다음 예제에서는 가용 영역`c3.large`의 인스턴스에 `us-east-1a`대한 전용 호스트를 할당 합니다.

#### JSON (영문)

```
"Host" : {
  "Type" : "AWS::EC2::Host",
  "Properties" : {
    "AutoPlacement" : "on",
    "AvailabilityZone" : "us-east-1a",
    "InstanceType" : "c3.large"
  }
}
```
