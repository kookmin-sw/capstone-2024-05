# AWS::SageMaker::ModelPackage TransformJobDefinition

알고리즘에 지정된 추론 사양을 사용하여 변환 작업을 실행하는 데 필요한 입력을 정의합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "BatchStrategy" : String,
  "Environment" : {Key: Value, ...},
  "MaxConcurrentTransforms" : Integer,
  "MaxPayloadInMB" : Integer,
  "TransformInput" : TransformInput,
  "TransformOutput" : TransformOutput,
  "TransformResources" : TransformResources
}
```

## 속성

`BatchStrategy`  
단일 미니 일괄 처리에 포함된 레코드 수를 결정하는 문자열입니다.  
 `SingleRecord` 는 미니 배치당 하나의 레코드만 사용됨을 의미합니다. `MultiRecord` 는 미니 배치가 한도 내에 들어갈 수 있는 만큼의 레코드를 포함하도록 설정되었음을 의미합니다 `MaxPayloadInMB` .  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `MultiRecord | SingleRecord`  
_업데이트 필요 사항_: 바꿔 놓음

`Environment`  
Docker 컨테이너에서 설정할 환경 변수입니다. 맵에서 최대 16개의 키 및 값 항목을 지원합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 맵  
_업데이트 필요 사항_: 바꿔 놓음

`MaxConcurrentTransforms`  
변환 작업의 각 인스턴스로 전송할 수 있는 최대 병렬 요청 수입니다. 기본값은 1입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_최소_: `0`  
_업데이트 필요 사항_: 바꿔 놓음

`MaxPayloadInMB`  
허용되는 최대 페이로드 크기(MB)입니다. 페이로드는 레코드의 데이터 부분(메타데이터 없음)입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_최소_: `0`  
_업데이트 필요 사항_: 바꿔 놓음

`TransformInput`  
입력 소스와 변환 작업이 입력 소스를 사용하는 방식에 대한 설명입니다.  
_필수 항목 여부_: 예  
_형식_: TransformInput  
_업데이트 필요 사항_: 바꿔 놓음

`TransformOutput`  
Amazon SageMaker에서 변환 작업의 결과를 저장할 Amazon S3 위치를 식별합니다.  
_필수 항목 여부_: 예  
_형식_: TransformOutput  
_업데이트 필요 사항_: 바꿔 놓음

`TransformResources`  
변환 작업에 대한 ML 컴퓨팅 인스턴스를 식별합니다.  
_필수 항목 여부_: 예  
_형식_: TransformResources  
_업데이트 필요 사항_: 바꿔 놓음
