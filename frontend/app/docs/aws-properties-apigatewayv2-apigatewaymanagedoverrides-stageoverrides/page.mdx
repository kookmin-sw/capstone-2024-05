# AWS::ApiGatewayV2::ApiGatewayManagedOverrides 스테이지오버라이드

이 `StageOverrides` 속성은 API Gateway 관리형 단계에 대한 단계 구성을 재정의합니다. 이 속성을 제거하면 API Gateway가 기본값을 복원합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AccessLogSettings" : AccessLogSettings,
  "AutoDeploy" : Boolean,
  "DefaultRouteSettings" : RouteSettings,
  "Description" : String,
  "RouteSettings" : Json,
  "StageVariables" : Json
}
```

## 속성

`AccessLogSettings`  
스테이지에서 액세스를 로깅하기 위한 설정입니다.  
*필수 항목 *여부: 아니요  
_형식_: AccessLogSettings  
_업데이트 필요 사항_: 중단 없음

`AutoDeploy`  
API에 대한 업데이트가 새 배포를 자동으로 트리거하는지 여부를 지정합니다. 기본값은 `true`입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`DefaultRouteSettings`  
스테이지의 기본 경로 설정입니다.  
*필수 항목 *여부: 아니요  
_유형_: RouteSettings  
_업데이트 필요 사항_: 중단 없음

`Description`  
API 단계에 대한 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`RouteSettings`  
스테이지에 대한 경로 설정입니다.  
*필수 항목 *여부: 아니요  
_형식_: Json  
_업데이트 필요 사항_: 중단 없음

`StageVariables`  
에 대한 단계 변수를 정의하는 맵입니다`Stage`. 변수 이름에는 영숫자와 밑줄 문자가 포함될 수 있으며 값은 +와 일치해야 합니다[A-Za-z0-9-._~:/?#&=,].  
*필수 항목 *여부: 아니요  
_형식_: Json  
_업데이트 필요 사항_: 중단 없음
