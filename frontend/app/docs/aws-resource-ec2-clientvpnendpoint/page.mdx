# AWS::EC2::ClientVpnEndpoint

Client VPN 엔드포인트를 지정합니다. Client VPN 엔드포인트는 Client VPN 세션을 활성화하고 관리하기 위해 생성하고 구성하는 리소스입니다. 모든 Client VPN 세션이 종료되는 대상 엔드포인트입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::ClientVpnEndpoint",
  "Properties" : {
      "AuthenticationOptions" : [ ClientAuthenticationRequest, ... ],
      "ClientCidrBlock" : String,
      "ClientConnectOptions" : ClientConnectOptions,
      "ClientLoginBannerOptions" : ClientLoginBannerOptions,
      "ConnectionLogOptions" : ConnectionLogOptions,
      "Description" : String,
      "DnsServers" : [ String, ... ],
      "SecurityGroupIds" : [ String, ... ],
      "SelfServicePortal" : String,
      "ServerCertificateArn" : String,
      "SessionTimeoutHours" : Integer,
      "SplitTunnel" : Boolean,
      "TagSpecifications" : [ TagSpecification, ... ],
      "TransportProtocol" : String,
      "VpcId" : String,
      "VpnPort" : Integer
    }
}
```

## 속성

`AuthenticationOptions`  
클라이언트를 인증하는 데 사용할 인증 방법에 대한 정보입니다.  
_필수 항목 여부_: 예  
_유형_: ClientAuthenticationRequest 목록  
_업데이트 필요 사항_: 바꿔 놓음

`ClientCidrBlock`  
클라이언트 IP 주소를 할당할 IPv4 주소 범위(CIDR 표기법)입니다. 주소 범위는 연결된 서브넷이 있는 VPC의 로컬 CIDR 또는 수동으로 추가한 경로와 겹칠 수 없습니다. Client VPN 엔드포인트가 생성된 후에는 주소 범위를 변경할 수 없습니다. 클라이언트 CIDR 범위의 크기는 /22 이상이어야 하며 /12보다 크지 않아야 합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`ClientConnectOptions`  
새 클라이언트 연결에 대한 연결 권한 부여를 관리하기 위한 옵션입니다.  
*필수 항목 *여부: 아니요  
_형식_: ClientConnectOptions  
_업데이트 필요 사항_: 중단 없음

`ClientLoginBannerOptions`  
VPN 세션이 설정될 때 AWS 제공 클라이언트에 표시되는 사용자 지정 가능한 텍스트 배너를 활성화하기 위한 옵션입니다.  
*필수 항목 *여부: 아니요  
_형식_: ClientLoginBannerOptions  
_업데이트 필요 사항_: 중단 없음

`ConnectionLogOptions` 클라이언트 연결 로깅 옵션에 대한 정보입니다.  
클라이언트 연결 로깅을 활성화하면 클라이언트 연결에 대한 데이터가 Cloudwatch Logs 로그 스트림으로 전송됩니다. 다음 정보가 기록됩니다.

- 클라이언트 연결 요청
- 클라이언트 연결 결과(성공 및 실패)
- 클라이언트 연결 요청이 실패한 이유
- 클라이언트 연결 종료 시간
  _필수 항목 여부_: 예  
  _형식_: ConnectionLogOptions  
  _업데이트 필요 사항_: 중단 없음

`Description`  
Client VPN 엔드포인트에 대한 간략한 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DnsServers`  
DNS 확인에 사용할 DNS 서버에 대한 정보입니다. Client VPN 엔드포인트에는 최대 2개의 DNS 서버가 있을 수 있습니다. DNS 서버가 지정되지 않은 경우 디바이스에 구성된 DNS 주소가 DNS 서버에 사용됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`SecurityGroupIds`  
대상 네트워크에 적용할 하나 이상의 보안 그룹 ID입니다. 보안 그룹이 포함된 VPC의 ID도 지정해야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`SelfServicePortal`  
Client VPN 엔드포인트에 대해 셀프 서비스 포털을 활성화할지 여부를 지정합니다.  
기본값: `enabled`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `disabled | enabled`  
_업데이트 필요 사항_: 중단 없음

`ServerCertificateArn`  
서버 인증서의 ARN입니다. 자세한 내용은 AWS Certificate Manager 사용 설명서를 참조하세요.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`SessionTimeoutHours`  
최대 VPN 세션 지속 시간(시간)입니다.  
유효한 값: `8 | 10 | 12 | 24`  
기본값: `24`  
_필수_: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`SplitTunnel`  
AWS Client VPN 엔드포인트에서 split-tunnel이 활성화되어 있는지 여부를 나타냅니다.  
기본적으로 VPN 엔드포인트의 스플릿 터널은 비활성화되어 있습니다.  
분할 터널 VPN 엔드포인트에 대한 자세한 내용은 * AWS Client VPN 관리자 안내서*의 분할 터널 AWS Client VPN 엔드포인트를 참조하세요.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`TagSpecifications`  
생성 중에 Client VPN 엔드포인트에 적용할 태그입니다.  
*필수 항목 *여부: 아니요  
_유형_: TagSpecification 목록  
_업데이트 필요 사항_: 바꿔 놓음

`TransportProtocol`  
VPN 세션에서 사용할 전송 프로토콜입니다.  
기본값: `udp`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `tcp | udp`  
_업데이트 필요 사항_: 바꿔 놓음

`VpcId`  
Client VPN 엔드포인트와 연결할 VPC의 ID입니다. 요청에 보안 그룹 ID가 지정되지 않은 경우 VPC의 기본 보안 그룹이 적용됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`VpnPort`  
TCP 및 UDP 트래픽에 대해 Client VPN 엔드포인트에 할당할 포트 번호입니다.  
유효한 값 : `443` | `1194`  
기본값: `443`  
_필수_: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`Client VPN 엔드포인트 ID가 반환됩니다. 예를 들면 다음과 같습니다 `cvpn-endpoint-1234567890abcdef0`.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

## 예제

### Client VPN 엔드포인트

다음 예제에서는 Active Directory 인증을 사용하고 CIDR 범위의 클라이언트 IP 주소를 할당하는 Client VPN 엔드포인트를 생성합니다 `10.0.0.0/22` .

#### JSON (영문)

```
"myClientVpnEndpoint": {
    "Type": "AWS::EC2::ClientVpnEndpoint",
    "Properties": {
        "AuthenticationOptions": [
            {
                "Type": "directory-service-authentication",
                "ActiveDirectory": {
                    "DirectoryId": "d-926example"
                }
            }
        ],
        "ClientCidrBlock": "10.0.0.0/22",
        "ConnectionLogOptions": {
            "Enabled": false
        },
        "Description": "My Client VPN Endpoint",
        "DnsServers": [
            "11.11.0.1"
        ],
        "ServerCertificateArn": "arn:aws:acm:us-east-1:111122223333:certificate/12345678-1234-1234-1234-123456789012",
        "TagSpecifications": [
            {
                "ResourceType": "client-vpn-endpoint",
                "Tags": [
                    {
                        "Key": "Purpose",
                        "Value": "Production"
                    }
                ]
            }
        ],
        "TransportProtocol": "udp"
    }
}
```
