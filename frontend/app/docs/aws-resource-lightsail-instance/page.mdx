# AWS::Lightsail::인스턴스

`AWS::Lightsail::Instance` 리소스는 Amazon Lightsail 인스턴스를 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Lightsail::Instance",
  "Properties" : {
      "AddOns" : [ AddOn, ... ],
      "AvailabilityZone" : String,
      "BlueprintId" : String,
      "BundleId" : String,
      "Hardware" : Hardware,
      "InstanceName" : String,
      "KeyPairName" : String,
      "Location" : Location,
      "Networking" : Networking,
      "State" : State,
      "Tags" : [Tag, ... ],
      "UserData" : String
    }
}
```

## 속성

`AddOns`  
인스턴스에 대한 추가 기능의 배열입니다.  
인스턴스 삭제 요청을 수행할 때 인스턴스에 추가 기능이 활성화되어 있으면 인스턴스가 삭제되기 전에 추가 기능이 자동으로 비활성화됩니다.
*필수 항목 *여부: 아니요  
_유형_: 애드온 목록  
_업데이트 필요 사항_: 중단 없음

`AvailabilityZone`  
인스턴스의 가용 영역입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`BlueprintId`  
인스턴스의 블루프린트 ID(예: `os_amlinux_2016_03`).  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`BundleId`  
인스턴스의 번들 ID(예: `micro_1_0`).  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`Hardware`  
인스턴스의 하드웨어 속성(예: vCPU 수, 연결된 디스크 및 RAM 양)입니다.  
디스크 연결 또는 디스크 분리 요청을 수행할 때 인스턴스가 다시 시작됩니다. 이렇게 하면 고정 IP가 연결되지 않은 경우 인스턴스의 퍼블릭 IP 주소가 재설정됩니다.
*필수 항목 *여부: 아니요  
_유형_: 하드웨어  
_업데이트 필요 사항_: 일부 중단

`InstanceName`  
인스턴스의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_패턴_: `w[w-]*w`  
*업데이트 필요 *사항: 바꿔 놓음

`KeyPairName`  
인스턴스에 사용할 키 페어의 이름입니다.  
키 페어 이름을 지정하지 않으면 리전 Lightsail 기본 키 페어가 사용됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Location`  
인스턴스의 위치(예: AWS 리전 및 가용 영역)입니다.  
이 `Location` 속성은 읽기 전용이며 인스턴스 만들기 또는 인스턴스 업데이트 요청에 지정해서는 안 됩니다.
*필수 항목 *여부: 아니요  
_유형_: 위치  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`Networking`  
인스턴스에 할당된 공용 포트 및 월별 데이터 전송량입니다.  
*필수 항목 *여부: 아니요  
_유형_: 네트워킹  
_업데이트 필요 사항_: 중단 없음

`State`  
인스턴스의 상태 코드 및 상태(예: `running`)입니다.  
이 `State` 속성은 읽기 전용이며 인스턴스 만들기 또는 인스턴스 업데이트 요청에 지정해서는 안 됩니다.
*필수 항목 *여부: 아니요  
_유형_: 상태  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`Tags`  
이 리소스에 적용할 키-값 쌍의 배열입니다.  
자세한 내용은 _AWS CloudFormation 사용 설명서의 태그를 참조하세요_.  
`Value` of `Tags` 는 Lightsail 리소스의 선택 사항입니다.
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`UserData`  
인스턴스에 대한 선택적 시작 스크립트입니다.  
시작 스크립트를 지정하여 추가 사용자 데이터로 인스턴스를 구성합니다. 예를 들어 `apt-get -y update` 시작 스크립트로 지정할 수 있습니다.  
인스턴스의 블루프린트에 따라 인스턴스에서 소프트웨어를 가져오는 명령이 달라집니다. Amazon Linux 및 CentOS는 `yum`, Debian 및 Ubuntu 는 를 `apt-get`사용하고, FreeBSD 는 `pkg`를 사용합니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`이 `Ref`리소스에 대한 고유 식별자를 반환합니다.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Hardware.CpuCount` 인스턴스에 있는 vCPU 수입니다.

`Hardware.RamSizeInGb`  
인스턴스의 RAM 용량(GB)입니다(예: `1.0`).

`InstanceArn`  
인스턴스의 Amazon 리소스 이름(ARN)입니다(예 `arn:aws:lightsail:us-east-2:123456789101:Instance/244ad76f-8aad-4741-809f-12345EXAMPLE`: ).

`IsStaticIp` 인스턴스에 고정 IP가 할당되어 있는지 여부를 나타내는 부울 값입니다.

`Location.AvailabilityZone` 인스턴스가 위치한 AWS 리전 및 가용 영역입니다.

`Location.RegionName` 인스턴스의 AWS 리전입니다.

`Networking.MonthlyTransfer.GbPerMonthAllocated` 인스턴스에 할당된 월별 데이터 전송량(GB)입니다.

`PrivateIpAddress` 인스턴스의 프라이빗 IP 주소입니다.

`PublicIpAddress` 인스턴스의 퍼블릭 IP 주소입니다.

`ResourceType`  
인스턴스의 리소스 유형(예: `Instance`).

`SshKeyName` 인스턴스에서 사용하는 SSH 키 페어의 이름입니다.

`State.Code` 인스턴스의 상태 코드입니다.

`State.Name`  
인스턴스의 상태(예: `running` 또는 `pending`)입니다.

`SupportCode` 인스턴스의 지원 코드입니다.  
Lightsail의 인스턴스 또는 다른 리소스에 대한 질문이 있을 때 지원할 수 있도록 이메일에 이 코드를 포함하세요. 이 코드는 지원 팀이 Lightsail 정보를 조회하는 데 도움이 됩니다.

`UserName`  
인스턴스에 연결하기 위한 사용자 이름(예: `ec2-user`).

## 발언

_인스턴스에 고정 IP 연결_

인스턴스 리소스를 사용하여 인스턴스에 고정 IP를 연결할 수 없습니다. 대신 고정 IP 리소스를 사용하여 고정 IP를 인스턴스에 연결해야 합니다. 인스턴스에 고정 IP를 연결하려면 인스턴스가 상태여야 합니다 `running` .

_네트워크 포트_

인스턴스 만들기 요청을 수행할 때 네트워크 포트를 지정하지 않으면 인스턴스가 만들어질 때 기본 네트워크 포트가 열립니다.

인스턴스 생성 요청을 수행할 때 인스턴스에서 포트를 열려면 기본 포트를 포함하여 열려는 모든 포트를 지정해야 합니다. 기본 포트는 열려는 포트를 지정할 때 자동으로 열리지 않습니다.

_디스크 연결 및 분리_

디스크 연결 또는 디스크 분리 요청을 수행할 때 인스턴스가 다시 시작됩니다. 이렇게 하면 고정 IP가 연결되지 않은 경우 인스턴스의 퍼블릭 IP 주소가 재설정됩니다.

동일한 요청에서 디스크를 분리하고(예`DiskA`: )를 분리하고 다른 디스크(예: )를 연결`DiskB`했는데 디스크 연결 요청이 실패하면 AWS CloudFormation은 인스턴스에 다시 연결되도록 변경 사항을 롤백하려고 시도 `DiskA` 합니다. 그러나 `DiskA` AWS CloudFormation에서 롤백을 시도하기 전에 삭제하면 롤백이 실패하고 인스턴스에 두 디스크 모두 연결되지 않습니다.

_읽기 전용 속성_

`State`, `Location`, `CpuCount`및 `RamSizeInGb` 속성은 읽기 전용이며 인스턴스 만들기 또는 인스턴스 업데이트 요청에 지정해서는 안 됩니다.
