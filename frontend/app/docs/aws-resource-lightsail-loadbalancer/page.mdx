# AWS::Lightsail::로드 밸런서

리소스는 `AWS::Lightsail::LoadBalancer` Lightsail 인스턴스와 함께 사용할 수 있는 로드 밸런서를 지정합니다.

**메모**  
리소스 유형을 사용하여 로드 밸런서에 TLS 인증서를 연결할 수 없습니다 `AWS::Lightsail::LoadBalancer` . 대신 리소스 종류를 사용하여 `AWS::Lightsail::LoadBalancerTlsCertificate` 인증서를 만들고 부하 분산 장치에 연결합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Lightsail::LoadBalancer",
  "Properties" : {
      "AttachedInstances" : [ String, ... ],
      "HealthCheckPath" : String,
      "InstancePort" : Integer,
      "IpAddressType" : String,
      "LoadBalancerName" : String,
      "SessionStickinessEnabled" : Boolean,
      "SessionStickinessLBCookieDurationSeconds" : String,
      "Tags" : [Tag, ... ],
      "TlsPolicyName" : String
    }
}
```

## 속성

`AttachedInstances`  
로드 밸런서에 연결할 Lightsail 인스턴스입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`HealthCheckPath`  
상태 확인이 수행될 연결된 인스턴스의 경로입니다. 경로를 지정하지 않으면 로드 밸런서가 기본(루트) 페이지()에 대한 요청을 시도합니다`/index.html`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_패턴_: `.*S.*`  
_업데이트 필요 사항_: 중단 없음

`InstancePort`  
로드 밸런서가 Lightsail 인스턴스로 트래픽을 전달하는 데 사용하는 포트입니다. HTTP 트래픽의 경우 port를 지정합니다 `80`. HTTPS 트래픽의 경우 port 를 지정합니다 `443`.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`IpAddressType`  
로드 밸런서의 IP 주소 유형입니다.  
가능한 값은 `ipv4` IPv4에만 해당되며 `dualstack` IPv4 및 IPv6 모두에 해당됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `dualstack | ipv4`  
_업데이트 필요_: 업데이트가 지원되지 않습니다.

`LoadBalancerName`  
부하 분산 장치의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_패턴_: `w[w-]*w`  
_업데이트 필요 사항_: 업데이트가 지원되지 않습니다.

`SessionStickinessEnabled`  
세션 고정을 사용할 수 있는지 여부를 나타내는 부울 값입니다.  
세션 고정성(*세션 선호도*라고도 함)을 활성화하여 사용자의 세션을 특정 인스턴스에 바인딩합니다. 이렇게 하면 세션 중 사용자의 모든 요청이 동일한 인스턴스로 전송됩니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`SessionStickinessLBCookieDurationSeconds`  
로드 밸런서 세션 고정성 쿠키가 부실한 것으로 간주되어야 하는 기간(초)입니다. 이 매개 변수를 지정하지 않으면 기본값은 0이며, 이는 고정 세션이 브라우저 세션 기간 동안 지속되어야 함을 나타냅니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Tags`  
이 리소스에 적용할 키-값 쌍의 배열입니다.  
자세한 내용은 _AWS CloudFormation 사용 설명서의 태그를 참조하세요_.  
`Value` of `Tags` 는 Lightsail 리소스의 선택 사항입니다.
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`TlsPolicyName`  
로드 밸런서에 대한 TLS 보안 정책의 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_패턴_: `w[w-]*w`  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`이 `Ref`리소스에 대한 고유 식별자를 반환합니다.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`LoadBalancerArn` 로드 밸런서의 Amazon 리소스 이름(ARN)입니다.

## 발언

_부하 분산 장치 TLS 인증서_

리소스 유형을 사용하여 로드 밸런서에 TLS 인증서를 연결할 수 없습니다 `AWS::Lightsail::LoadBalancer` . 대신 리소스 종류를 사용하여 `AWS::Lightsail::LoadBalancerTlsCertificate` 인증서를 만들고 부하 분산 장치에 연결합니다.

_HTTPS 리디렉션 구성_

HTTPS 리디렉션은 부하 분산 장치에 연결된 리소스`HttpsRedirectionEnabled`의 매개 변수를 `AWS::Lightsail::LoadBalancerTlsCertificate`사용해서만 설정할 수 있습니다.
