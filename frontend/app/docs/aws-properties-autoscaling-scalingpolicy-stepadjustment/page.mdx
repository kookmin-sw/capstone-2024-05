# AWS::AutoScaling::ScalingPolicy 단계조정

`StepAdjustment` AWS::AutoScaling::ScalingPolicy 리소스의 속성`StepAdjustments`에 대한 단계 조정을 지정합니다.

다음 예에서는 위반 임계값이 50인 경보가 있다고 가정합니다.

- 지표가 50보다 크거나 같고 60보다 작을 때 단계 조절을 트리거하려면 하한을 0으로, 상한을 10으로 지정합니다.
- 지표가 40보다 크고 50보다 작거나 같을 때 단계 조절을 트리거하려면 하한을 -10으로, 상한을 0으로 지정합니다.

단계 정책의 단계 조정에 대한 몇 가지 규칙이 있습니다.

- 단계 조절 범위는 겹치거나 간격이 있을 수 없습니다.
- 최대 한 단계 조정은 null 하한을 가질 수 있습니다. 한 단계 조정의 하한이 음수인 경우 null 하한이 있는 단계 조정이 있어야 합니다.
- 최대 한 단계 조정은 null 상한을 가질 수 있습니다. 한 단계 조절의 상한이 양수인 경우 null 상한이 있는 단계 조절이 있어야 합니다.
- 상한과 하한은 동일한 단계 조정에서 null이 될 수 없습니다.

자세한 내용은 Amazon EC2 Auto Scaling 사용 설명서*의 단계 조절을 참조하세요*.

샘플 템플릿 코드 조각은 리소스의 예제 섹션에서 찾을 수 있습니다 `AWS::AutoScaling::ScalingPolicy` .

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "MetricIntervalLowerBound" : Double,
  "MetricIntervalUpperBound" : Double,
  "ScalingAdjustment" : Integer
}
```

## 속성

`MetricIntervalLowerBound`  
경보 임계값과 CloudWatch 지표 간의 차이에 대한 하한입니다. 메트릭 값이 위반 임계값을 초과하는 경우 하한은 포함됩니다(메트릭은 임계값에 하한을 더한 값보다 크거나 같아야 함). 그렇지 않으면 배타적입니다(메트릭은 임계값에 하한을 더한 값보다 커야 함). null 값은 음의 무한대를 나타냅니다.  
_필수_: 조건부  
_유형_: 더블  
_업데이트 필요 사항_: 중단 없음

`MetricIntervalUpperBound`  
경보 임계값과 CloudWatch 지표 간의 차이에 대한 상한입니다. 메트릭 값이 위반 임계값보다 높으면 상한은 배타적입니다(메트릭은 임계값에 상한을 더한 값보다 작아야 함). 그렇지 않으면 포함됩니다(메트릭은 임계값에 상한을 더한 값보다 작거나 같아야 함). null 값은 양의 무한대를 나타냅니다.  
상한은 하한보다 커야 합니다.  
_필수_: 조건부  
_유형_: 더블  
_업데이트 필요 사항_: 중단 없음

`ScalingAdjustment`  
지정된 조정 유형에 따라 크기를 조정할 양입니다. 양수 값은 현재 용량에 추가되고 음수는 현재 용량에서 제거됩니다. 정확한 용량의 경우 음수가 아닌 값을 지정해야 합니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음
