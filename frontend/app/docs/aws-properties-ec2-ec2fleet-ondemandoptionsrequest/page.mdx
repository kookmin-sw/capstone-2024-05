# AWS::EC2::EC2Fleet 온디맨드 옵션 요청

EC2 집합에서 온디맨드 인스턴스의 할당 전략을 지정합니다.

`OnDemandOptionsRequest` 은 AWS::EC2::EC2Fleet 리소스의 속성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AllocationStrategy" : String,
  "CapacityReservationOptions" : CapacityReservationOptionsRequest,
  "MaxTotalPrice" : String,
  "MinTargetCapacity" : Integer,
  "SingleAvailabilityZone" : Boolean,
  "SingleInstanceType" : Boolean
}
```

## 속성

`AllocationStrategy`  
시작 템플릿의 순서를 결정하는 전략은 온디맨드 용량을 이행하는 데 사용할 재정의됩니다.  
 `lowest-price` - EC2 집합은 가격을 사용하여 주문을 결정하고 최저 가격을 먼저 시작합니다.  
 `prioritized` - EC2 집합은 각 시작 템플릿 재정의에 할당한 우선 순위를 사용하여 우선 순위가 가장 높은 것을 먼저 시작합니다.  
기본값: `lowest-price`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `lowest-price | prioritized`  
_업데이트 필요 사항_: 바꿔 놓음

`CapacityReservationOptions`  
온디맨드 용량을 이행하기 위해 사용되지 않은 용량 예약을 사용하는 전략입니다.  
유형의 플릿에 대해서만 지원됩니다`instant`.  
*필수 항목 *여부: 아니요  
_유형_: CapacityReservationOptionsRequest  
_업데이트 필요 사항_: 바꿔 놓음

`MaxTotalPrice`  
지불할 의사가 있는 온디맨드 인스턴스의 시간당 최대 금액입니다.  
플릿에 로 구성된 T 인스턴스가 포함되어 `unlimited`있고 평균 CPU 사용량이 기준 사용률을 초과하는 경우 잉여 크레딧에 대한 요금이 발생합니다. 는 `MaxTotalPrice` 잉여 크레딧을 고려하지 않으며, 잉여 크레딧을 사용하는 경우 최종 비용이 에 대해 지정한 비용보다 높을 수 있습니다`MaxTotalPrice`. 자세한 내용은 EC2 사용 설명서의 잉여 크레딧에 요금이 부과될 수 _있음을 참조하세요_.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`MinTargetCapacity`  
플릿의 온디맨드 인스턴스에 대한 최소 목표 용량입니다. 최소 목표 용량에 도달하지 않으면 플릿은 인스턴스를 시작하지 않습니다.  
유형의 플릿에 대해서만 지원됩니다`instant`.  
다음 중 하나 이상을 지정해야 합니다. `SingleAvailabilityZone` `SingleInstanceType`  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 바꿔 놓음

`SingleAvailabilityZone`  
플릿이 모든 온디맨드 인스턴스를 단일 가용 영역으로 시작함을 나타냅니다.  
유형의 플릿에 대해서만 지원됩니다`instant`.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 바꿔 놓음

`SingleInstanceType`  
플릿이 단일 인스턴스 유형을 사용하여 플릿의 모든 온디맨드 인스턴스를 시작함을 나타냅니다.  
유형의 플릿에 대해서만 지원됩니다`instant`.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 바꿔 놓음
