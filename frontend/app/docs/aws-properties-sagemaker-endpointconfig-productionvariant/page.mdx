# AWS::SageMaker::EndpointConfig ProductionVariant

호스팅할 모델과 이 모델을 호스팅하기 위해 배포할 리소스를 지정합니다. 여러 모델을 배포하는 경우 객체를 지정하여 모델 간에 트래픽을 분산하는 방법을 Amazon SageMaker에 지시합니다 `InitialVariantWeight` .

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AcceleratorType" : String,
  "ContainerStartupHealthCheckTimeoutInSeconds" : Integer,
  "EnableSSMAccess" : Boolean,
  "InitialInstanceCount" : Integer,
  "InitialVariantWeight" : Double,
  "InstanceType" : String,
  "ModelDataDownloadTimeoutInSeconds" : Integer,
  "ModelName" : String,
  "ServerlessConfig" : ServerlessConfig,
  "VariantName" : String,
  "VolumeSizeInGB" : Integer
}
```

## 속성

`AcceleratorType`  
프로덕션 변형에 사용할 Elastic Inference(EI) 인스턴스의 크기입니다. EI 인스턴스는 추론을 위한 온디맨드 GPU 컴퓨팅을 제공합니다. 자세한 내용은 Amazon SageMaker에서 Elastic Inference 사용 단원을 참조하십시오. 자세한 내용은 Amazon SageMaker에서 Elastic Inference 사용 단원을 참조하십시오.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `ml.eia1.large | ml.eia1.medium | ml.eia1.xlarge | ml.eia2.large | ml.eia2.medium | ml.eia2.xlarge`  
_업데이트 필요 사항_: 바꿔 놓음

`ContainerStartupHealthCheckTimeoutInSeconds`  
속성 설명을 사용할 수 없습니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 바꿔 놓음

`EnableSSMAccess`  
 이 파라미터를 사용하여 엔드포인트 뒤의 프로덕션 변형에 대한 기본 AWS Systems Manager(SSM) 액세스를 켤 수 있습니다. 기본적으로 SSM 액세스는 엔드포인트 뒤에 있는 모든 프로덕션 변형에 대해 비활성화되어 있습니다. 새 엔드포인트 구성을 생성하고 를 호출하여 기존 엔드포인트 뒤에 있는 프로덕션 변형에 대한 SSM 액세스를 켜거나 끌 수 있습니다`UpdateEndpoint`.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 바꿔 놓음

`InitialInstanceCount`  
처음에 시작할 인스턴스 수입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_최소_: `1`  
_업데이트 필요 사항_: 바꿔 놓음

`InitialVariantWeight`  
엔드포인트 구성에서 지정하는 모든 모델 간의 초기 트래픽 분산을 결정합니다. 프로덕션 변형에 대한 트래픽은 모든 ProductionVariants에서 `VariantWeight` 모든 값의 합계에 대한 `VariantWeight` 비율에 따라 결정됩니다 . 지정하지 않으면 기본값은 1.0입니다.  
_필수 항목 여부_: 예  
_유형_: 더블  
_업데이트 필요 사항_: 바꿔 놓음
