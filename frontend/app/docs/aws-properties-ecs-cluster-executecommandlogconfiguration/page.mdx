# AWS::ECS::클러스터 ExecuteCommandLogConfiguration

execute 명령 조치의 결과에 대한 로그 구성입니다. 로그는 CloudWatch Logs 또는 Amazon S3 버킷으로 전송할 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "CloudWatchEncryptionEnabled" : Boolean,
  "CloudWatchLogGroupName" : String,
  "S3BucketName" : String,
  "S3EncryptionEnabled" : Boolean,
  "S3KeyPrefix" : String
}
```

## 속성

`CloudWatchEncryptionEnabled`  
CloudWatch 로그에서 암호화를 사용할지 여부를 결정합니다. 지정하지 않으면 암호화가 해제됩니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`CloudWatchLogGroupName`  
로그를 전송할 CloudWatch 로그 그룹의 이름입니다.  
CloudWatch 로그 그룹이 이미 생성되어 있어야 합니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`S3BucketName`  
로그를 전송할 S3 버킷의 이름입니다.  
S3 버킷이 이미 생성되어 있어야 합니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`S3EncryptionEnabled`  
S3 로그에서 암호화를 사용할지 여부를 결정합니다. 지정하지 않으면 암호화가 사용되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`S3KeyPrefix`  
로그를 저장할 S3 버킷의 선택적 폴더입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
