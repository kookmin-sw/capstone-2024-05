# AWS::SageMaker::엔드포인트 트래픽라우팅 구성

엔드포인트 배포 중에 트래픽을 이전 플릿에서 새 플릿으로 이동하기 위한 트래픽 라우팅 전략을 정의합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "CanarySize" : CapacitySize,
  "LinearStepSize" : CapacitySize,
  "Type" : String,
  "WaitIntervalInSeconds" : Integer
}
```

## 속성

`CanarySize`  
새 엔드포인트 플릿에서 트래픽을 켜기 위한 첫 번째 단계의 배치 크기입니다. `Value` 변형 총 인스턴스 수의 50% 이하여야 합니다(MUST).  
*필수 항목 *여부: 아니요  
_형식_: CapacitySize  
_업데이트 필요 사항_: 중단 없음

`LinearStepSize`  
새 엔드포인트 플릿에서 트래픽을 켜기 위한 각 단계의 배치 크기입니다. `Value` 변형 전체 인스턴스 수의 10-50%여야 합니다.  
*필수 항목 *여부: 아니요  
_형식_: CapacitySize  
_업데이트 필요 사항_: 중단 없음

`Type` 트래픽 라우팅 전략 유형입니다.

- `ALL_AT_ONCE`: 엔드포인트 트래픽이 한 번에 새 플릿으로 이동합니다.
- `CANARY`: 엔드포인트 트래픽은 두 단계로 새 플릿으로 이동합니다. 첫 번째 단계는 트래픽의 작은 부분인 카나리아입니다. 두 번째 단계는 트래픽의 나머지 부분입니다.
- `LINEAR`: 엔드포인트 트래픽은 구성 가능한 크기의 n단계로 새 플릿으로 이동합니다.
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _허용되는 값_: `ALL_AT_ONCE | CANARY | LINEAR`  
  _업데이트 필요 사항_: 중단 없음

`WaitIntervalInSeconds`  
새 엔드포인트 집합에서 트래픽을 켜기 위한 증분 단계 사이의 대기 시간(초)입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_최소_: `0`  
_최대_: `3600`  
_업데이트 필요 사항_: 중단 없음
