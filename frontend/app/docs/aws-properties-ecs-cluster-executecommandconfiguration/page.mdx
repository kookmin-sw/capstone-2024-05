# AWS::ECS::클러스터 ExecuteCommandConfiguration

실행 명령 구성의 세부 정보입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "KmsKeyId" : String,
  "LogConfiguration" : ExecuteCommandLogConfiguration,
  "Logging" : String
}
```

## 속성

`KmsKeyId`  
AWS Key Management Service 키 ID를 지정하여 로컬 클라이언트와 컨테이너 간의 데이터를 암호화합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`LogConfiguration`  
execute 명령 조치의 결과에 대한 로그 구성입니다. 로그는 CloudWatch Logs 또는 Amazon S3 버킷으로 전송할 수 있습니다. `logging=OVERRIDE` 가 지정되면 a `logConfiguration` 를 제공해야 합니다.  
*필수 항목 *여부: 아니요  
_형식_: ExecuteCommandLogConfiguration  
_업데이트 필요 사항_: 중단 없음

`Logging` 실행 명령 결과에 대한 로그를 리디렉션하는 데 사용할 로그 설정입니다. 다음 로그 설정을 사용할 수 있습니다.

- `NONE`: 실행 명령 세션이 기록되지 않습니다.
- `DEFAULT`: `awslogs` 작업 정의의 구성이 사용됩니다. 로깅 매개 변수를 지정하지 않으면 기본값은 이 값입니다. `awslogs` 작업 정의에 로그 드라이버가 구성되지 않은 경우 출력이 기록되지 않습니다.
- `OVERRIDE`: 의 일부로 로깅 세부 정보를 지정합니다`logConfiguration`. `OVERRIDE` logging 옵션이 지정된 경우 `logConfiguration` 필수입니다.
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _허용되는 값_: `DEFAULT | NONE | OVERRIDE`  
  _업데이트 필요 사항_: 중단 없음
