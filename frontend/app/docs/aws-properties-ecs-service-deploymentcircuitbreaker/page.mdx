# AWS::ECS::서비스 배포회로 차단기

**메모**  
배포 회로 차단기는 롤링 업데이트() 배포 유형을 사용하는 서비스에만 사용할 수 있습니다`ECS`.

**배포 회로 차단기는** 서비스가 안정적인 상태에 도달할 수 없는 경우 서비스 배포가 실패할지 여부를 결정합니다. 이 기능이 켜져 있으면 서비스 배포가 실패 상태로 전환되고 새 작업 시작이 중지됩니다. 또한 실패 후 마지막으로 완료된 배포로 서비스를 롤백하도록 Amazon ECS를 구성할 수 있습니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 롤링 업데이트를 참조하세요_.

API 실패 이유에 대한 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 API 실패 이유를 참조하세요_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Enable" : Boolean,
  "Rollback" : Boolean
}
```

## 속성

`Enable`  
서비스에 배포 회로 차단기 논리를 사용할지 여부를 결정합니다.  
_필수 항목 여부_: 예  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Rollback`  
서비스 배포가 실패할 경우 서비스를 롤백하도록 Amazon ECS를 구성할지 여부를 결정합니다. 롤백이 설정된 경우 서비스 배포가 실패하면 서비스가 성공적으로 완료된 마지막 배포로 롤백됩니다.  
_필수 항목 여부_: 예  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음
