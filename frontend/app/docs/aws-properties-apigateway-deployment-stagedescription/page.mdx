# AWS::ApiGateway::D eployment 단계 설명

`StageDescription` 는 배포 단계를 구성하는 AWS::ApiGateway::D eployment 리소스의 속성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AccessLogSetting" : AccessLogSetting,
  "CacheClusterEnabled" : Boolean,
  "CacheClusterSize" : String,
  "CacheDataEncrypted" : Boolean,
  "CacheTtlInSeconds" : Integer,
  "CachingEnabled" : Boolean,
  "CanarySetting" : CanarySetting,
  "ClientCertificateId" : String,
  "DataTraceEnabled" : Boolean,
  "Description" : String,
  "DocumentationVersion" : String,
  "LoggingLevel" : String,
  "MethodSettings" : [ MethodSetting, ... ],
  "MetricsEnabled" : Boolean,
  "Tags" : [Tag, ... ],
  "ThrottlingBurstLimit" : Integer,
  "ThrottlingRateLimit" : Double,
  "TracingEnabled" : Boolean,
  "Variables" : {Key: Value, ...}
}
```

## 속성

`AccessLogSetting`  
이 단계에서 로깅 액세스에 대한 설정을 지정합니다.  
*필수 항목 *여부: 아니요  
_형식_: AccessLogSetting  
_업데이트 필요 사항_: 중단 없음

`CacheClusterEnabled`  
스테이지에 대해 캐시 클러스터를 사용할 수 있는지 여부를 지정합니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`CacheClusterSize`  
스테이지의 캐시 클러스터 크기입니다. 자세한 내용은 _API Gateway API Reference의 cacheClusterSize를 참조하십시오_.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`CacheDataEncrypted`  
캐시된 응답이 암호화되었는지 여부를 나타냅니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`CacheTtlInSeconds`  
TTL(Time-to-Live) 기간(초)으로, API Gateway가 응답을 캐싱하는 기간을 지정합니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`CachingEnabled`  
요청에 대해 응답이 캐시되고 반환되는지 여부를 나타냅니다. 응답을 캐시하려면 스테이지에서 캐시 클러스터를 활성화해야 합니다. 자세한 내용은 API Gateway 개발자 안내서의 단계에서 API Gateway 캐싱을 활성화하여 API 성능 향상을 참조하십시오\**.  
*필수 항목 *여부: 아니요  
*유형*: Boolean  
*업데이트 필요 사항\*: 중단 없음

`CanarySetting`  
이 단계에서 카나리아 배포에 대한 설정을 지정합니다.  
*필수 항목 *여부: 아니요  
_형식_: CanarySetting  
_업데이트 필요 사항_: 중단 없음

`ClientCertificateId`  
API Gateway가 단계에서 통합 엔드포인트를 호출하는 데 사용하는 클라이언트 인증서의 식별자입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DataTraceEnabled`  
스테이지의 메서드에 대해 데이터 추적 로깅을 사용할 수 있는지 여부를 나타냅니다. API Gateway는 이러한 로그를 Amazon CloudWatch Logs로 푸시합니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Description`  
스테이지의 목적에 대한 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DocumentationVersion`  
API 설명서 스냅샷의 버전 식별자입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`LoggingLevel`  
이 메서드의 로깅 수준입니다. 유효한 값은 `loggingLevel`Amazon API Gateway API Reference*에서 MethodSetting 리소스의 속성을 참조하십시오*.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`MethodSettings`  
스테이지의 모든 메서드에 대한 설정을 구성합니다.  
*필수 항목 *여부: 아니요  
_형식_: MethodSetting 목록  
_업데이트 필요 사항_: 중단 없음

`MetricsEnabled`  
단계의 메서드에 대해 Amazon CloudWatch 지표가 활성화되었는지 여부를 나타냅니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Tags`  
스테이지와 연결할 임의의 태그(키-값 쌍)의 배열입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`ThrottlingBurstLimit`  
대상 요청 버스트 속도 제한입니다. 이렇게 하면 일정 기간 동안 목표 속도 제한보다 더 많은 요청을 통과할 수 있습니다. 자세한 내용은 _API Gateway 개발자 안내서의 API 요청 조절 관리를 참조하십시오_.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`ThrottlingRateLimit`  
대상 요청 안정 상태 속도 제한입니다. 자세한 내용은 _API Gateway 개발자 안내서의 API 요청 조절 관리를 참조하십시오_.  
*필수 항목 *여부: 아니요  
_유형_: 더블  
_업데이트 필요 사항_: 중단 없음

`TracingEnabled`  
이 단계에서 X-ray를 사용한 활성 추적을 사용할지 여부를 지정합니다.  
자세한 내용은 API Gateway 개발자 안내서의 AWS X-Ray를 사용하여 API Gateway API 실행 추적을 참조하십시오\**.  
*필수 항목 *여부: 아니요  
*유형*: Boolean  
*업데이트 필요 사항\*: 중단 없음

`Variables`  
단계 변수를 정의하는 맵입니다. 변수 이름은 영숫자로 구성되어야 하며 값은 다음 정규식과 일치해야 합니다. `[A-Za-z0-9-._~:/?#&=,]+`  
*필수 항목 *여부: 아니요  
_유형_: 문자열 맵  
_업데이트 필요 사항_: 중단 없음
