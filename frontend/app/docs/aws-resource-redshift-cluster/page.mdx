# AWS::Redshift::클러스터

클러스터를 지정합니다. _클러스터_ 는 계산 노드 집합으로 구성된 완전 관리형 데이터 웨어하우스입니다.

Virtual Private Cloud(VPC)에서 클러스터를 생성하려면 클러스터 서브넷 그룹 이름을 제공해야 합니다. 클러스터 서브넷 그룹은 Amazon Redshift가 클러스터를 생성할 때 사용하는 VPC의 서브넷을 식별합니다. 클러스터 관리에 대한 자세한 내용은 Amazon Redshift Cluster Management Guide의 Amazon Redshift Clusters를 참조하십시오\*\*.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Redshift::Cluster",
  "Properties" : {
      "AllowVersionUpgrade" : Boolean,
      "AquaConfigurationStatus" : String,
      "AutomatedSnapshotRetentionPeriod" : Integer,
      "AvailabilityZone" : String,
      "AvailabilityZoneRelocation" : Boolean,
      "AvailabilityZoneRelocationStatus" : String,
      "Classic" : Boolean,
      "ClusterIdentifier" : String,
      "ClusterParameterGroupName" : String,
      "ClusterSecurityGroups" : [ String, ... ],
      "ClusterSubnetGroupName" : String,
      "ClusterType" : String,
      "ClusterVersion" : String,
      "DBName" : String,
      "DeferMaintenance" : Boolean,
      "DeferMaintenanceDuration" : Integer,
      "DeferMaintenanceEndTime" : String,
      "DeferMaintenanceStartTime" : String,
      "DestinationRegion" : String,
      "ElasticIp" : String,

      "Endpoint" : Endpoint,
      "EnhancedVpcRouting" : Boolean,
      "HsmClientCertificateIdentifier" : String,
      "HsmConfigurationIdentifier" : String,
      "IamRoles" : [ String, ... ],
      "KmsKeyId" : String,
      "LoggingProperties" : LoggingProperties,
      "MaintenanceTrackName" : String,
      "ManualSnapshotRetentionPeriod" : Integer,
      "MasterUsername" : String,
      "MasterUserPassword" : String,

}
```

## 속성

`AllowVersionUpgrade`  
의 경우 `true`유지 관리 기간 동안 클러스터에서 실행 중인 Amazon Redshift 엔진에 메이저 버전 업그레이드를 적용할 수 있습니다.  
Amazon Redshift 엔진의 새 메이저 버전이 릴리스되면 유지 관리 기간 동안 클러스터에서 실행 중인 Amazon Redshift 엔진에 업그레이드를 자동으로 적용하도록 서비스에 요청할 수 있습니다.  
기본값: `true`  
_필수_: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`AquaConfigurationStatus`  
이 매개 변수는 더 이상 사용되지 않습니다. AQUA 구성 상태를 설정하지 않습니다. Amazon Redshift는 AQUA(Advanced Query Accelerator)를 사용할지 여부를 자동으로 결정합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `auto | disabled | enabled`  
_업데이트 필요 사항_: 중단 없음

`AutomatedSnapshotRetentionPeriod`  
자동 스냅샷이 보존되는 기간(일)입니다. 값이 0이면 자동 스냅샷이 비활성화됩니다. 자동 스냅샷이 비활성화된 경우에도 Amazon Redshift API Reference의 CreateClusterSnapshot을 사용하여 원할 때 수동 스냅샷을 생성할 수 _있습니다_.  
기본값: `1`  
제약 조건: 0에서 35 사이의 값이어야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`AvailabilityZone`  
Amazon Redshift에서 클러스터를 프로비저닝할 EC2 가용 영역(AZ)입니다. 예를 들어, 특정 가용 영역에서 여러 EC2 인스턴스를 실행 중인 경우 네트워크 지연 시간을 줄이기 위해 클러스터를 동일한 영역에 프로비저닝할 수 있습니다.  
기본값: 엔드포인트에서 지정한 리전에서 시스템에서 선택한 임의의 가용 영역입니다.  
본보기: `us-east-2d`  
제약 조건: 지정된 가용 영역은 현재 엔드포인트와 동일한 지역에 있어야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`AvailabilityZoneRelocation`  
클러스터가 생성된 후 가용 영역 간에 Amazon Redshift 클러스터에 대한 재배치를 활성화하는 옵션입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`AvailabilityZoneRelocationStatus`  
가용 영역 재배치 작업의 상태를 설명합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`Classic`  
크기 조정 작업이 클래식 크기 조정 프로세스를 사용하는지 여부를 나타내는 부울 값입니다. 이 매개 변수를 제공하지 않거나 값을 로 설정하면 `false`크기 조정 유형이 탄력적입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`ClusterIdentifier` 클러스터의 고유 식별자입니다. 이 식별자를 사용하여 삭제 또는 수정과 같은 후속 클러스터 작업을 위해 클러스터를 참조합니다. 식별자는 Amazon Redshift 콘솔에도 표시됩니다.  
제약 조건:

- 1-63자의 영숫자 또는 하이픈을 포함해야 합니다.
- 알파벳 문자는 소문자여야 합니다.
- 첫 번째 문자는 문자여야 합니다.
- 하이픈으로 끝나거나 두 개의 연속 하이픈을 포함할 수 없습니다.
- AWS 계정 내의 모든 클러스터에 대해 고유해야 합니다.
  예: `myexamplecluster`  
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _최대_: `2147483647`  
  _업데이트 필요 사항_: 바꿔 놓음

`ClusterParameterGroupName` 이 클러스터와 연결할 파라미터 그룹의 이름입니다.  
기본값: 기본 Amazon Redshift 클러스터 파라미터 그룹입니다. 기본 파라미터 그룹에 대한 자세한 내용은 Amazon Redshift 파라미터 그룹 작업 단원을 참조하십시오  
제약 조건:

- 1-255자의 영숫자 또는 하이픈이어야 합니다.
- 첫 번째 문자는 문자여야 합니다.
- 하이픈으로 끝나거나 두 개의 연속 하이픈을 포함할 수 없습니다.
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _최대_: `2147483647`  
  _업데이트 필요 사항_: 중단 없음

`ClusterSecurityGroups`  
이 클러스터와 연결할 보안 그룹 목록입니다.  
기본값: Amazon Redshift의 기본 클러스터 보안 그룹입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`ClusterSubnetGroupName`  
이 클러스터와 연결할 클러스터 서브넷 그룹의 이름입니다.  
이 파라미터를 제공하지 않으면 결과 클러스터가 Virtual Private Cloud(VPC) 외부에 배포됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 바꿔 놓음

`ClusterType` 클러스터의 유형입니다. cluster type이

- `single-node`, **NumberOfNodes** 매개 변수는 필요하지 않습니다.
- `multi-node`, **NumberOfNodes** 매개 변수가 필요합니다.
  유효한 값 : `multi-node` | `single-node`  
  기본값: `multi-node`  
  _필수_: 예  
  _유형_: 문자열  
  _최대_: `2147483647`  
  _업데이트 필요 사항_: 중단 없음

`ClusterVersion`  
클러스터에 배포하려는 Amazon Redshift 엔진 소프트웨어의 버전입니다.  
선택한 버전은 클러스터의 모든 노드에서 실행됩니다.  
제약 조건: 현재 버전 1.0만 사용할 수 있습니다.  
예: `1.0`  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`DBName`  
클러스터를 만들 때 만들 첫 번째 데이터베이스의 이름입니다.  
클러스터를 만든 후 추가 데이터베이스를 만들려면 SQL 클라이언트를 사용하여 클러스터에 연결하고 SQL 명령을 사용하여 데이터베이스를 만듭니다. 자세한 내용은 Amazon Redshift 데이터베이스 개발자 안내서의 데이터베이스 생성을 참조하십시오.  
기본값: `dev`  
제약 조건:

- 1-64자의 영숫자를 포함해야 합니다.
- 소문자만 포함해야 합니다.
- 서비스에서 예약한 단어일 수 없습니다. 예약어 목록은 Amazon Redshift 데이터베이스 개발자 안내서의 예약어에서 확인할 수 있습니다.
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _최대_: `2147483647`  
  _업데이트 필요 사항_: 바꿔 놓음

`DeferMaintenance`  
지연된 유지 관리 기간을 사용할지 여부를 나타내는 부울입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`DeferMaintenanceDuration`  
유지 관리 기간(일)을 나타내는 정수입니다. 기간을 지정하면 종료 시간을 지정할 수 없습니다. 기간은 45일 이하여야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`DeferMaintenanceEndTime`  
유지 관리를 연기하는 기간의 종료에 대한 타임스탬프입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DeferMaintenanceStartTime`  
지연된 유지 관리 기간의 시작 시간을 나타내는 타임스탬프입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DestinationRegion`  
교차 리전 스냅샷 복사가 활성화된 경우 스냅샷이 자동으로 복사되는 대상 리전입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`ElasticIp`  
클러스터의 EIP(탄력적 IP) 주소입니다.  
제약 조건: 클러스터는 EC2-VPC에서 프로비저닝되어야 하며 인터넷 게이트웨이를 통해 공개적으로 액세스할 수 있어야 합니다. 가용성 영역 재배치가 켜져 있는 공개적으로 액세스할 수 있는 클러스터에 대한 탄력적 IP 주소를 지정하지 마십시오. EC2-VPC에서 클러스터를 프로비저닝하는 방법에 대한 자세한 내용은 Amazon Redshift Cluster Management Guide의 지원되는 플랫폼에서 클러스터 시작을 참조하십시오.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`Encrypted`  
인 경우 `true`클러스터의 데이터는 미사용 시 암호화됩니다.  
기본값 : false  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Endpoint`  
연결 끝점입니다.  
*필수 항목 *여부: 아니요  
_유형_: 끝점  
_업데이트 필요 사항_: 중단 없음

`EnhancedVpcRouting`  
향상된 VPC 라우팅이 활성화된 클러스터를 생성할지 여부를 지정하는 옵션입니다. 향상된 VPC 라우팅을 사용하는 클러스터를 생성하려면 클러스터가 VPC에 있어야 합니다. 자세한 내용은 Amazon Redshift Cluster Management Guide의 향상된 VPC 라우팅을 참조하십시오.  
이 옵션이 인 경우 `true`향상된 VPC 라우팅이 활성화됩니다.  
기본값 : false  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`HsmClientCertificateIdentifier`  
Amazon Redshift 클러스터가 HSM에 저장된 데이터 암호화 키를 검색하는 데 사용하는 HSM 클라이언트 인증서의 이름을 지정합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2147483647`  
_업데이트 필요 사항_: 중단 없음

`HsmConfigurationIdentifier`  
Amazon Redshift 클러스터가 HSM에서 키를 검색하고 저장하는 데 사용할 수 있는 정보가 포함된 HSM 구성의 이름을 지정합니다.  
*필수 항목 *여부: 아니요

연결할 수 있는 최대 IAM 역할 수에는 할당량이 적용됩니다. 자세한 내용은 _Amazon Redshift Cluster Management Guide의 할당량 및 제한을 참조하십시오_.  
*필수 항목 *여부: 아니요

`KmsKeyId`  
클러스터의 데이터를 암호화하는 데 사용할 암호화 키의 AWS Key Management Service(KMS) 키 ID입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열

`LoggingProperties`

클러스터의 유지 관리 트랙 이름에 대한 선택적 매개 변수입니다. 유지 관리 트랙 이름을 제공하지 않으면 클러스터가 트랙에 할당됩니다 `current` .

_최대_: `2147483647`
