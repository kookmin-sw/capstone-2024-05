# AWS::AutoScalingPlans::ScalingPlan 미리 정의됨ScalingMetric사양

`PredefinedScalingMetricSpecification` 는 AWS Auto Scaling(Auto Scaling 계획)과 함께 사용할 대상 추적 구성에 대한 사용자 지정 조정 지표를 지정하는 TargetTrackingConfiguration의 하위 속성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "PredefinedScalingMetricType" : String,
  "ResourceLabel" : String
}
```

## 속성

`PredefinedScalingMetricType`  
측정항목 유형입니다. `ALBRequestCountPerTarget` 지표 유형은 Auto Scaling 그룹, 스팟 집합 요청 및 ECS 서비스에만 적용됩니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `ALBRequestCountPerTarget | ASGAverageCPUUtilization | ASGAverageNetworkIn | ASGAverageNetworkOut | DynamoDBReadCapacityUtilization | DynamoDBWriteCapacityUtilization | EC2SpotFleetRequestAverageCPUUtilization | EC2SpotFleetRequestAverageNetworkIn | EC2SpotFleetRequestAverageNetworkOut | ECSServiceAverageCPUUtilization | ECSServiceAverageMemoryUtilization | RDSReaderAverageCPUUtilization | RDSReaderAverageDatabaseConnections`  
_업데이트 필요 사항_: 중단 없음

`ResourceLabel`  
메트릭 유형과 연결된 리소스를 식별합니다. 지표 유형이 이고 `ALBRequestCountPerTarget` Auto Scaling 그룹, 스팟 플릿 요청 또는 ECS 서비스에 연결된 Application Load Balancer의 대상 그룹이 없는 한 리소스 레이블을 지정할 수 없습니다.  
로드 밸런서 ARN의 마지막 부분과 대상 그룹 ARN의 마지막 부분을 슬래시(/)로 구분된 단일 값에 추가하여 리소스 레이블을 생성합니다. 형식은 app/&lt;load-balancer-name>/&lt;load-balancer-id>/targetgroup/&lt;target-group-name>/&lt;target-group-id>이며, 여기서

- app/&lt;load-balancer-name>/&lt;load-balancer-id>는 로드 밸런서 ARN의 마지막 부분입니다.
- targetgroup/&lt;target-group-name>/&lt;target-group-id> 은 대상 그룹 ARN의 마지막 부분입니다.
  예: app/EC2Co-EcsEl-1TKLTMITMM0EO/f37c06a68c1748aa/targetgroup/EC2Co-Defau-LDNM7Q3ZH1ZN/6d4ea56ca2d6a18d.  
  Application Load Balancer의 ARN을 찾으려면 DescribeLoadBalancers API 작업을 사용합니다. 대상 그룹의 ARN을 찾으려면 DescribeTargetGroups API 작업을 사용합니다.  
  _필수_: 조건부  
  _유형_: 문자열  
  _최소_: `1`  
  _최대_: `1023`  
  _업데이트 필요 사항_: 중단 없음
