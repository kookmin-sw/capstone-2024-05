# AWS::Route53Resolver::ResolverRuleAssociation

AssociateResolverRule, DisassociateResolverRule 또는 ListResolverRuleAssociations 요청에 대한 응답으로 해석기 규칙과 VPC 간의 연결에 대한 정보를 제공합니다. 연결은 VPC에서 시작된 DNS 쿼리가 네트워크로 전달되는지를 결정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Route53Resolver::ResolverRuleAssociation",
  "Properties" : {
      "Name" : String,
      "ResolverRuleId" : String,
      "VPCId" : String
    }
}
```

## 속성

`Name`  
해석기 규칙과 VPC 간의 연결 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `64`  
_패턴_: `(?!^[0-9]+$)([a-zA-Z0-9-_' ']+)`  
_업데이트 필요 사항_: 바꿔 놓음

`ResolverRuleId`  
로 지정된 VPC와 연결한 확인자 규칙의 ID입니다 `VPCId`.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `64`  
_업데이트 필요 사항_: 바꿔 놓음

`VPCId`  
해석기 규칙과 연결한 VPC의 ID입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `64`  
_업데이트 필요 사항_: 바꿔 놓음

`VPCId`  
해석기 규칙을 연결한 VPC의 ID입니다(예: `vpc-03cf94c75cexample`).
