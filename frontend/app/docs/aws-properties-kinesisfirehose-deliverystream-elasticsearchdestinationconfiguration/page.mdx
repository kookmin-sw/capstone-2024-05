# AWS::KinesisFirehose::D eliveryStream ElasticsearchDestinationConfiguration

`ElasticsearchDestinationConfiguration` 속성 유형은 Amazon Kinesis Data Firehose(Kinesis Data Firehose)가 데이터를 전송하는 Amazon Elasticsearch Service(Amazon ES) 도메인을 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "BufferingHints" : ElasticsearchBufferingHints,
  "CloudWatchLoggingOptions" : CloudWatchLoggingOptions,
  "ClusterEndpoint" : String,
  "DocumentIdOptions" : DocumentIdOptions,
  "DomainARN" : String,
  "IndexName" : String,
  "IndexRotationPeriod" : String,
  "ProcessingConfiguration" : ProcessingConfiguration,
  "RetryOptions" : ElasticsearchRetryOptions,
  "RoleARN" : String,
  "S3BackupMode" : String,
  "S3Configuration" : S3DestinationConfiguration,
  "TypeName" : String,
  "VpcConfiguration" : VpcConfiguration
}
```

## 속성

`BufferingHints`  
Kinesis Data Firehose가 수신 데이터를 Amazon ES 도메인으로 전송하는 동안 이를 버퍼링하는 방법을 구성합니다.  
*필수 항목 *여부: 아니요  
_형식_: ElasticsearchBufferingHints  
_업데이트 필요 사항_: 중단 없음

`CloudWatchLoggingOptions`  
전송 스트림에 대한 Amazon CloudWatch Logs 로깅 옵션입니다.  
*필수 항목 *여부: 아니요  
_유형_: CloudWatchLoggingOptions  
_업데이트 필요 사항_: 중단 없음

`ClusterEndpoint`  
클러스터와 통신할 때 사용할 끝점입니다. this `ClusterEndpoint` 또는 `DomainARN` 필드를 지정합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DocumentIdOptions`  
속성 설명을 사용할 수 없습니다.  
*필수 항목 *여부: 아니요  
_형식_: DocumentIdOptions  
_업데이트 필요 사항_: 중단 없음

`DomainARN`  
Amazon ES 도메인의 ARN입니다. IAM 역할에는 `DescribeElasticsearchDomain`, `DescribeElasticsearchDomains`, 및 `DescribeElasticsearchDomainConfig`RoleARN**에 지정된 역할을 수임한 후에 ** 대한 권한이 있어야 합니다.  
`ClusterEndpoint` 또는 `DomainARN`를 지정합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `1`  
_최대_: `512`  
_패턴_: `arn:.*`  
_업데이트 필요 사항_: 중단 없음

`IndexName`  
Kinesis Data Firehose가 인덱싱을 위해 데이터를 추가하는 Elasticsearch 인덱스의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `80`  
_패턴_: `.*`  
_업데이트 필요 사항_: 중단 없음

`IndexRotationPeriod`  
Elasticsearch 인덱스 교체 빈도입니다. 인덱스 교체를 활성화하면 Kinesis Data Firehose가 UTC 도착 타임스탬프의 일부를 지정된 인덱스 이름에 추가하고 그에 따라 추가된 타임스탬프를 교체합니다. 자세한 내용은 _Amazon Kinesis Data Firehose 개발자 안내서의 Amazon ES 대상에 대한 인덱스 교체를 참조하십시오_.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `NoRotation | OneDay | OneHour | OneMonth | OneWeek`  
_업데이트 필요 사항_: 중단 없음

`ProcessingConfiguration`  
Kinesis Data Firehose 전송 스트림에 대한 데이터 처리 구성입니다.  
*필수 항목 *여부: 아니요  
_형식_: ProcessingConfiguration  
_업데이트 필요 사항_: 중단 없음

`RetryOptions`  
Kinesis Data Firehose가 Amazon ES로 데이터를 전송할 수 없는 경우의 재시도 동작입니다.  
*필수 항목 *여부: 아니요  
_유형_: ElasticsearchRetryOptions  
_업데이트 필요 사항_: 중단 없음

`RoleARN`  
Amazon ES 구성 API를 호출하고 문서를 인덱싱하기 위해 Kinesis Data Firehose에서 수임할 IAM 역할의 Amazon 리소스 이름(ARN)입니다. 자세한 내용은 Amazon Kinesis Data Firehose를 사용한 액세스 제어 단원을 참조하십시오.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `512`  
_패턴_: `arn:.*`  
_업데이트 필요 사항_: 중단 없음

`S3BackupMode`  
Kinesis Data Firehose가 Amazon Simple Storage Service(Amazon S3)로 데이터를 전송하는 조건입니다. Amazon S3에 모든 문서(모든 데이터)를 전송하거나 Kinesis Data Firehose가 Amazon ES 대상으로 전송할 수 없는 문서만 전송할 수 있습니다. 자세한 내용과 유효한 값은 `S3BackupMode`Amazon Kinesis Data Firehose API Reference*의 ElasticsearchDestinationConfiguration 데이터 형식에 대한 내용을 * 참조하십시오.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `AllDocuments | FailedDocumentsOnly`  
_업데이트 필요 사항_: 중단 없음

`S3Configuration`  
Kinesis Data Firehose가 수신 데이터를 백업하는 S3 버킷입니다.  
_필수 항목 여부_: 예  
_유형_: S3DestinationConfiguration  
_업데이트 필요 사항_: 중단 없음

`TypeName`  
Amazon ES가 데이터를 인덱싱할 때 문서에 추가하는 Elasticsearch 유형 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `0`  
_최대_: `100`  
_패턴_: `.*`  
_업데이트 필요 사항_: 중단 없음

`VpcConfiguration`  
Amazon ES 대상의 VPC에 대한 세부 정보입니다.  
*필수 항목 *여부: 아니요  
_유형_: VpcConfiguration  
_업데이트 필요 사항_: 바꿔 놓음
