# AWS::Lightsail::컨테이너 컨테이너

`Container` 는 ContainerServiceDeployment 속성의 속성입니다. Amazon Lightsail 컨테이너 서비스에서 시작되거나 시작될 컨테이너의 설정에 대해 설명합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Command" : [ String, ... ],
  "ContainerName" : String,
  "Environment" : [ EnvironmentVariable, ... ],
  "Image" : String,
  "Ports" : [ PortInfo, ... ]
}
```

## 속성

`Command`  
컨테이너에 대한 시작 명령입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`ContainerName`  
컨테이너의 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Environment`  
컨테이너의 환경 변수입니다.  
*필수 항목 *여부: 아니요  
_유형_: EnvironmentVariable 목록  
_업데이트 필요 사항_: 중단 없음

`Image`  
컨테이너에 사용되는 이미지의 이름입니다.  
컨테이너 서비스에서 제공(등록 및 저장)된 컨테이너 이미지는 콜론()으로 시작합니다`:`. 예를 들어 컨테이너 서비스 이름이 인 경우 `container-service-1`컨테이너 이미지 레이블이 `mystaticsite` 인 경우 등록된 컨테이너 이미지의 세 번째 버전(`3`)을 사용하려면 를 지정해야 합니다`:container-service-1.mystaticsite.3`. 최신 버전의 컨테이너 이미지를 사용하려면 `latest` 버전 번호 대신 지정합니다(예: `:container-service-1.mystaticsite.latest`). 컨테이너 서비스는 등록된 컨테이너 이미지의 가장 높은 번호 버전을 자동으로 사용합니다.  
Docker Hub와 같은 공용 레지스트리에서 제공되는 컨테이너 이미지는 콜론으로 시작하지 않습니다. 예를 들어, `nginx:latest` 또는 `nginx`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Ports`  
컨테이너의 열려 있는 방화벽 포트 및 프로토콜을 설명하는 개체입니다.  
*필수 항목 *여부: 아니요  
_유형_: PortInfo 목록  
_업데이트 필요 사항_: 중단 없음
