# AWS::ECS::서비스 배포 컨트롤러

서비스에 사용할 배포 컨트롤러입니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Amazon ECS 배포 유형을 참조하세요_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : String
}
```

## 속성

`Type`  
사용할 배포 컨트롤러 유형입니다. 다음과 같은 세 가지 배포 컨트롤러 유형을 사용할 수 있습니다.  
증권 시세 표시기  
롤링 업데이트(`ECS`) 배포 유형에는 현재 실행 중인 컨테이너 버전을 최신 버전으로 교체하는 작업이 포함됩니다. 롤링 업데이트 중에 Amazon ECS가 서비스에서 추가하거나 제거하는 컨테이너 수는 DeploymentConfiguration에 지정된 대로 서비스 배포 중에 허용되는 최소 및 최대 정상 작업 수를 조정하여 제어됩니다.  
CODE_DEPLOY  
블루/그린(`CODE_DEPLOY`) 배포 유형은 AWS CodeDeploy에서 제공하는 블루/그린 배포 모델을 사용하며, 이를 통해 프로덕션 트래픽을 전송하기 전에 서비스의 새 배포를 확인할 수 있습니다.  
외부  
외부(`EXTERNAL`) 배포 유형을 사용하면 타사 배포 컨트롤러를 사용하여 Amazon ECS 서비스의 배포 프로세스를 완전히 제어할 수 있습니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `CODE_DEPLOY | ECS | EXTERNAL`  
_업데이트 필요 사항_: 바꿔 놓음
