# AWS::ECS::CapacityProvider AutoScalingGroupProvider

용량 공급자에 대한 오토 스케일링의 세부 정보입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AutoScalingGroupArn" : String,
  "ManagedScaling" : ManagedScaling,
  "ManagedTerminationProtection" : String
}
```

## 속성

`AutoScalingGroupArn`  
오토 스케일링을 식별하는 Amazon 리소스 이름(ARN) 또는 오토 스케일링 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`ManagedScaling`  
오토 스케일링 용량 공급자에 대한 관리형 조정 설정입니다.  
*필수 항목 *여부: 아니요  
_유형_: ManagedScaling  
_업데이트 필요 사항_: 중단 없음

`ManagedTerminationProtection`  
오토 스케일링 용량 공급자에 사용할 관리형 종료 방지 설정입니다. 이는 오토 스케일링에 관리형 종료 방지 기능이 있는지 여부를 결정합니다. 기본값은 off입니다.  
관리형 종료 방지를 사용하는 경우 관리형 크기 조정도 사용해야 하며, 그렇지 않으면 관리형 종료 방지가 작동하지 않습니다.
관리형 종료 방지 기능이 켜져 있으면 Amazon ECS는 축소 작업 중에 태스크가 포함된 오토 스케일링의 Amazon EC2 인스턴스가 종료되는 것을 방지합니다. 오토 스케일링과 오토 스케일링의 각 인스턴스에는 축소 작업으로부터 인스턴스를 보호할 수 있는 기능도 있어야 합니다. 자세한 내용은 * AWS Auto Scaling 사용 설명서*의 인스턴스 보호를 참조하세요.  
관리형 종료 방지 기능이 꺼져 있으면 오토 스케일링이 축소될 때 Amazon EC2 인스턴스가 종료로부터 보호되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `DISABLED | ENABLED`  
_업데이트 필요 사항_: 중단 없음
