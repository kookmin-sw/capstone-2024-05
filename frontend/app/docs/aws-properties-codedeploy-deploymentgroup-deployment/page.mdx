# AWS::CodeDeploy::D eploymentGroup 배포

`Deployment` 는 배포 그룹의 인스턴스에 배포할 AWS CodeDeploy 애플리케이션 개정을 지정하는 DeploymentGroup 리소스의 속성입니다. 응용 프로그램 수정 버전을 지정하면 프로비저닝 프로세스가 완료되는 즉시 대상 수정 버전이 배포됩니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Description" : String,
  "IgnoreApplicationStopFailures" : Boolean,
  "Revision" : RevisionLocation
}
```

## 속성

`Description`  
배포에 대한 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`IgnoreApplicationStopFailures`  
 true인 경우, 인스턴스에 대한 `ApplicationStop`, `BeforeBlockTraffic`또는 `AfterBlockTraffic` 배포 수명 주기 이벤트가 실패하면 배포가 다음 배포 수명 주기 이벤트로 계속됩니다. 예를 들어 `ApplicationStop` 실패하면 DownloadBundle을 사용하여 배포가 계속됩니다. `BeforeBlockTraffic` 실패하면 배포가 계속됩니다`BlockTraffic`. `AfterBlockTraffic` 실패하면 배포가 계속됩니다`ApplicationStop`.  
 false이거나 지정되지 않은 경우, 인스턴스에 배포하는 동안 수명 주기 이벤트가 실패하면 해당 배포가 실패합니다. 해당 인스턴스에 대한 배포가 전체 배포의 일부이고 정상 호스트 수가 최소 정상 호스트 수보다 작지 않은 경우 다음 인스턴스로의 배포가 시도됩니다.  
 배포 중에 AWS CodeDeploy 에이전트는 `ApplicationStop`, `BeforeBlockTraffic`, 에 `AfterBlockTraffic` 지정된 스크립트 와 이전에 성공한 배포의 AppSpec 파일에 지정된 스크립트를 실행합니다. (다른 모든 스크립트는 현재 배포의 AppSpec 파일에서 실행됩니다.) 이러한 스크립트 중 하나에 오류가 포함되어 있고 성공적으로 실행되지 않으면 배포가 실패할 수 있습니다.  
 실패의 원인이 성공적으로 실행되지 않는 마지막으로 성공한 배포의 스크립트인 경우 새 배포를 만들고 를 사용하여 `ignoreApplicationStopFailures` `ApplicationStop`, `BeforeBlockTraffic`및 `AfterBlockTraffic` 실패를 무시하도록 지정합니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Revision`  
저장된 애플리케이션 아티팩트의 위치 및 이를 검색할 서비스에 대한 정보입니다.  
_필수 항목 여부_: 예  
_형식_: RevisionLocation  
_업데이트 필요 사항_: 중단 없음
