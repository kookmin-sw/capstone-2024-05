# AWS::AutoScaling::LaunchConfiguration 메타데이터 옵션

`MetadataOptions` 는 인스턴스에 대한 메타데이터 옵션을 설명하는 AWS::AutoScaling::LaunchConfiguration의 속성입니다.

자세한 내용은 Amazon EC2 Auto Scaling 사용 설명서*의 인스턴스 메타데이터 옵션 구성을 참조하세요*.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "HttpEndpoint" : String,
  "HttpPutResponseHopLimit" : Integer,
  "HttpTokens" : String
}
```

## 속성

`HttpEndpoint`  
이 파라미터는 인스턴스에서 HTTP 메타데이터 엔드포인트를 활성화하거나 비활성화합니다. 매개 변수를 지정하지 않으면 기본 상태는 `enabled`입니다.  
값을 지정 `disabled` 하면 인스턴스 메타데이터에 액세스할 수 없습니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `disabled | enabled`  
_업데이트 필요 사항_: 바꿔 놓음

`HttpPutResponseHopLimit`  
인스턴스 메타데이터 요청에 대해 원하는 HTTP PUT 응답 홉 제한입니다. 숫자가 클수록 인스턴스 메타데이터 요청이 더 멀리 이동할 수 있습니다.  
기본값 : 1  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_최소_: `1`  
_최대_: `64`  
_업데이트 필요 사항_: 바꿔 놓음

`HttpTokens`  
인스턴스 메타데이터 요청에 대한 토큰 사용 상태입니다. 요청에 매개 변수가 지정되지 않은 경우 기본 상태는 `optional`입니다.  
상태가 인 경우 `optional`요청에서 서명된 토큰 헤더를 사용하거나 사용하지 않고 인스턴스 메타데이터를 검색하도록 선택할 수 있습니다. 토큰 없이 IAM 역할 자격 증명을 검색하면 버전 1.0 역할 자격 증명이 반환됩니다. 유효한 서명된 토큰을 사용하여 IAM 역할 자격 증명을 검색하면 버전 2.0 역할 자격 증명이 반환됩니다.  
상태가 인 경우 `required`인스턴스 메타데이터 검색 요청과 함께 서명된 토큰 헤더를 전송해야 합니다. 이 상태에서 IAM 역할 자격 증명을 검색하면 항상 버전 2.0 자격 증명이 반환됩니다. 버전 1.0 자격 증명을 사용할 수 없습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `optional | required`  
_업데이트 필요 사항_: 바꿔 놓음
