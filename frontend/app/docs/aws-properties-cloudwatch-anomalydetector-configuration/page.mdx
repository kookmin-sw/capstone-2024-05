# AWS::CloudWatch::AnomalyDetector 구성

모델을 학습하고 업데이트할 때 제외할 시간 범위를 포함하여 변칙 검색 모델을 학습하는 방법에 대한 세부 정보를 지정합니다. 구성에는 메트릭에 사용할 표준 시간대도 포함될 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "ExcludedTimeRanges" : [ Range, ... ],
  "MetricTimeZone" : String
}
```

## 속성

`ExcludedTimeRanges`  
변칙 검색 모델을 학습하고 업데이트할 때 사용하지 않도록 제외할 시간 범위의 배열을 지정합니다. 이를 사용하여 CloudWatch가 모델을 생성하거나 업데이트할 때 배포와 같이 지표에 대해 비정상적인 값을 발생시킬 수 있는 이벤트가 사용되지 않도록 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 범위 목록  
_업데이트 필요 사항_: 중단 없음

`MetricTimeZone`  
메트릭에 사용할 표준 시간대입니다. 이는 메트릭이 이러한 시간 변경에 민감한 경우 모델이 일광 절약 시간 변경을 자동으로 고려할 수 있도록 하는 데 유용합니다.  
시간대를 지정하려면 표준 tz 데이터베이스에 지정된 시간대 이름을 사용합니다. 자세한 내용은 tz 데이터베이스를 참조하세요.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
