# AWS CloudFormation 템플릿 작업

스택 리소스를 프로비저닝하고 구성하려면 JSON 형식의 텍스트 파일인 AWS CloudFormation 템플릿을 이해해야 합니다. 이러한 템플릿은 AWS CloudFormation 스택에서 프로비저닝할 리소스를 설명합니다. AWS CloudFormation Designer 또는 텍스트 편집기를 사용하여 템플릿을 생성하고 저장할 수 있습니다. 템플릿의 구조와 구문에 대한 자세한 내용은 Template anatomy를 참조하십시오.

JSON에 익숙하지 않은 경우 AWS CloudFormation Designer를 사용하여 AWS CloudFormation 템플릿을 시작할 수 있습니다. AWS CloudFormation Designer는 템플릿을 시각적으로 생성 및 수정할 수 있는 도구입니다. 자세한 내용은 AWS CloudFormation Designer란 무엇입니까?를 참조하십시오.

템플릿 코드 조각은 특정 리소스에 대한 템플릿을 작성하는 방법을 보여주는 예제를 제공합니다. 예를 들어 Amazon EC2 인스턴스, Amazon S3 도메인, AWS CloudFormation 매핑 등에 대한 코드 조각을 볼 수 있습니다. 코드 조각은 리소스별로 그룹화되며, 일반 템플릿 코드 조각에 범용 AWS CloudFormation 코드 조각이 있습니다.

템플릿에서 사용할 수 있는 지원되는 리소스, 형식 이름, 내장 함수 및 의사 매개 변수에 대한 자세한 내용은 템플릿 참조를 참조하세요.

**항목**

- AWS CloudFormation 템플릿 형식
- 템플릿 해부학
- AWS CloudFormation Designer란 무엇입니까?
- 연습
- 템플릿 스니펫
- 사용자 지정 리소스
- AWS CloudFormation 매크로를 사용하여 템플릿에 대한 사용자 지정 처리 수행
- 모듈을 사용하여 리소스 구성 캡슐화 및 재사용
- AWS CloudFormation을 사용하여 CodeDeploy를 통해 ECS 블루/그린 배포 수행
- AWS CloudFormation 템플릿에서 정규식 사용
