# AWS::Lightsail::인스턴스 포트

`Port` 는 Networking 속성의 속성입니다. 인스턴스의 포트에 대한 정보를 설명합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AccessDirection" : String,
  "AccessFrom" : String,
  "AccessType" : String,
  "CidrListAliases" : [ String, ... ],
  "Cidrs" : [ String, ... ],
  "CommonName" : String,
  "FromPort" : Integer,
  "Ipv6Cidrs" : [ String, ... ],
  "Protocol" : String,
  "ToPort" : Integer
}
```

## 속성

`AccessDirection`  
액세스 방향(`inbound` 또는 `outbound`).  
Lightsail은 현재 `inbound` 액세스 방향만 지원합니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`AccessFrom`  
액세스가 허용되는 위치입니다. 예를 들어 `Anywhere (0.0.0.0/0)`특정 `Custom` IP 주소 또는 IP 주소 범위가 허용되는 경우입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`AccessType`  
액세스 유형(`Public` 또는 `Private`).  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`CidrListAliases`  
미리 구성된 IP 주소 범위에 대한 액세스를 정의하는 별칭입니다.  
현재 지원되는 유일한 별칭은 `lightsail-connect`Lightsail 콘솔에 있는 브라우저 기반 RDP/SSH 클라이언트의 IP 주소를 인스턴스에 연결할 수 있도록 허용하는 입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Cidrs`  
포트 및 프로토콜을 통해 인스턴스에 연결할 수 있는 IPv4 주소 또는 IPv4 주소 범위(CIDR 표기법)입니다.  
이 `ipv6Cidrs` 파라미터는 인스턴스에 연결할 수 있는 IPv6 주소를 나열합니다.
예제:

- IP 주소를 허용하려면 `192.0.2.44` 또는 `192.0.2.44`을 `192.0.2.44/32`지정합니다.
- IP 주소를 허용하려면 `192.0.2.0` `192.0.2.255`를 지정합니다`192.0.2.0/24`.
  *필수 항목 *여부: 아니요  
  _유형_: 문자열 목록  
  _업데이트 필요 사항_: 중단 없음

`CommonName`  
포트 정보의 일반 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`FromPort` 인스턴스에서 열려 있는 포트 범위의 첫 번째 포트입니다.  
허용되는 포트:

- TCP 및 UDP - `0` 대상 `65535`
- ICMP - IPv4 주소의 ICMP 유형입니다. 예를 들어, `8` `fromPort` ICMP Ping을 활성화하려면 (ICMP 유형) 및 `-1` (ICMP 코드)`toPort`로 지정합니다.
- ICMPv6 - IPv6 주소의 ICMP 유형입니다. 예를 들어 `128` `fromPort` (ICMPv6 유형) 및 `0` `toPort` (ICMPv6 코드)로 지정합니다.
  *필수 항목 *여부: 아니요  
  _유형_: Integer  
  _업데이트 필요 사항_: 중단 없음

`Ipv6Cidrs`  
포트 및 프로토콜을 통해 인스턴스에 연결할 수 있는 IPv6 주소 또는 IPv6 주소 범위(CIDR 표기법)입니다. IPv6 주소가 있는 디바이스만 IPv6를 통해 인스턴스에 연결할 수 있습니다. 그렇지 않으면 IPv4를 사용해야 합니다.  
이 `cidrs` 파라미터는 인스턴스에 연결할 수 있는 IPv4 주소를 나열합니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Protocol` IP 프로토콜 이름입니다.  
이름은 다음 중 하나일 수 있습니다.

- `tcp` - TCP(Transmission Control Protocol)는 IP 네트워크로 통신하는 호스트에서 실행되는 애플리케이션 간에 스트리밍된 데이터를 안정적이고 순서가 정해져 있으며 오류 검사된 방식으로 전달합니다. 신뢰할 수 있는 데이터 스트림 서비스가 필요하지 않은 애플리케이션이 있는 경우 대신 UDP를 사용합니다.
- `all` - 모든 전송 계층 프로토콜 유형.
- `udp` - UDP(User Datagram Protocol)를 사용하면 컴퓨터 애플리케이션이 IP(인터넷 프로토콜) 네트워크의 다른 호스트에 메시지(또는 데이터그램)를 보낼 수 있습니다. 전송 채널 또는 데이터 경로를 설정하기 위해 사전 통신이 필요하지 않습니다. 신뢰할 수 있는 데이터 스트림 서비스가 필요하지 않은 애플리케이션은 안정성보다 대기 시간 단축을 강조하는 연결 없는 데이터그램 서비스를 제공하는 UDP를 사용할 수 있습니다. 신뢰할 수 있는 데이터 스트림 서비스가 필요한 경우 TCP를 대신 사용합니다.
- `icmp` - ICMP(Internet Control Message Protocol)는 인스턴스와 통신할 때 성공 또는 실패를 나타내는 오류 메시지 및 운영 정보를 보내는 데 사용됩니다. 예를 들어, 인스턴스에 연결할 수 없는 경우 오류가 표시됩니다. 로 지정하는 경우 `icmp` `protocol` 매개 변수를 사용하여 ICMP 유형을 지정 `fromPort` 하고 매개 변수를 사용하여 ICMP 코드를 지정해야 합니다 `toPort` .
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _업데이트 필요 사항_: 중단 없음

`ToPort` 인스턴스에서 열려 있는 포트 범위의 마지막 포트입니다.  
허용되는 포트:

- TCP 및 UDP - `0` 대상 `65535`
- ICMP - IPv4 주소의 ICMP 코드입니다. 예를 들어, `8` `fromPort` ICMP Ping을 활성화하려면 (ICMP 유형) 및 `-1` (ICMP 코드)`toPort`로 지정합니다.
- ICMPv6 - IPv6 주소에 대한 ICMP 코드입니다. 예를 들어 `128` `fromPort` (ICMPv6 유형) 및 `0` `toPort` (ICMPv6 코드)로 지정합니다.
  *필수 항목 *여부: 아니요  
  _유형_: Integer  
  _업데이트 필요 사항_: 중단 없음
