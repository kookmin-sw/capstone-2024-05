# AWS::EC2::P lacementGroup

인스턴스를 시작할 배치 그룹을 지정합니다. 배치 그룹의 전략에 따라 그룹 내에서 인스턴스가 구성되는 방식이 결정됩니다.

`cluster` 배치 그룹은 단일 가용 영역 내에 있는 인스턴스의 논리적 그룹으로, 짧은 네트워크 지연 시간과 높은 네트워크 처리량의 이점을 누릴 수 있습니다. `spread` 배치 그룹은 인스턴스를 별개의 하드웨어에 배치합니다. `partition` 배치 그룹은 인스턴스 그룹을 서로 다른 파티션에 배치하며, 한 파티션의 인스턴스는 다른 파티션의 인스턴스와 동일한 하드웨어를 공유하지 않습니다.

자세한 내용은 Amazon EC2 사용 설명서*의 배치 그룹을 참조하십시오*.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::PlacementGroup",
  "Properties" : {
      "PartitionCount" : Integer,
      "SpreadLevel" : String,
      "Strategy" : String,
      "Tags" : [Tag, ... ]
    }
}
```

## 속성

`PartitionCount`  
파티션 수입니다. **Strategy**가 로 설정된 경우에만 유효합니다`partition`.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 바꿔 놓음

`SpreadLevel` 배치 그룹이 인스턴스를 분산하는 방법을 결정합니다.

- 호스트 – `host` Outpost 배치 그룹에만 사용할 수 있습니다.
- 랙 – 사용 제한이 없습니다.
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _허용되는 값_: `host | rack`  
  _업데이트 필요 사항_: 바꿔 놓음

`Strategy`  
배치 전략입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `cluster | partition | spread`  
_업데이트 필요 사항_: 바꿔 놓음

`Tags`  
새 게재위치 그룹에 적용할 태그입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`배치 그룹의 이름을 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`GroupName` 배치 그룹의 이름입니다.
