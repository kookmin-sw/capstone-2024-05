# AWS::EC2::VerifiedAccessTrustProvider

Verified Access 트러스트 공급자에 대해 설명합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::VerifiedAccessTrustProvider",
  "Properties" : {
      "Description" : String,
      "DeviceOptions" : DeviceOptions,
      "DeviceTrustProviderType" : String,
      "OidcOptions" : OidcOptions,
      "PolicyReferenceName" : String,
      "Tags" : [Tag, ... ],
      "TrustProviderType" : String,
      "UserTrustProviderType" : String
    }
}
```

## 속성

`Description`  
AWS Verified Access 트러스트 공급자에 대한 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`DeviceOptions`  
디바이스 ID 신뢰 공급자에 대한 옵션입니다.  
*필수 항목 *여부: 아니요  
_형식_: DeviceOptions  
_업데이트 필요 사항_: 바꿔 놓음

`DeviceTrustProviderType`  
장치 기반 신뢰 공급자의 유형입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `crowdstrike | jamf`  
_업데이트 필요 사항_: 바꿔 놓음

`OidcOptions`  
OpenID Connect 호환 사용자 ID 신뢰 제공자에 대한 옵션입니다.  
*필수 항목 *여부: 아니요  
_형식_: OidcOptions  
_업데이트 필요 사항_: 중단 없음

`PolicyReferenceName`  
정책 규칙으로 작업할 때 사용할 식별자입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Tags`  
태그.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`TrustProviderType`  
Verified Access 신뢰 공급자의 유형입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `device | user`  
_업데이트 필요 사항_: 바꿔 놓음

`UserTrustProviderType`  
사용자 기반 신뢰 공급자의 유형입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `iam-identity-center | oidc`  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`Verified Access 신뢰 공급자의 ID를 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`CreationTime` 생성 시간입니다.

`LastUpdatedTime` 마지막 업데이트 시간입니다.

`VerifiedAccessTrustProviderId` Verified Access 트러스트 공급자의 ID입니다.
