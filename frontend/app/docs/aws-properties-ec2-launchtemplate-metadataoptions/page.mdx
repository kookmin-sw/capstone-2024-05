# AWS::EC2::LaunchTemplate 메타데이터 옵션

인스턴스에 대한 메타데이터 옵션입니다. 자세한 내용은 _Amazon EC2 사용 설명서의 인스턴스 메타데이터 및 사용자 데이터를 참조하세요_.

`MetadataOptions` 은 AWS::EC2::LaunchTemplate LaunchTemplateData의 속성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "HttpEndpoint" : String,
  "HttpProtocolIpv6" : String,
  "HttpPutResponseHopLimit" : Integer,
  "HttpTokens" : String,
  "InstanceMetadataTags" : String
}
```

## 속성

`HttpEndpoint`  
인스턴스에서 HTTP 메타데이터 엔드포인트를 활성화하거나 비활성화합니다. 매개 변수를 지정하지 않으면 기본 상태는 `enabled`입니다.  
값을 지정 `disabled` 하면 인스턴스 메타데이터에 액세스할 수 없습니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `disabled | enabled`  
_업데이트 필요 사항_: 중단 없음

`HttpProtocolIpv6`  
인스턴스 메타데이터 서비스에 대한 IPv6 끝점을 활성화하거나 비활성화합니다.  
기본값: `disabled`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `disabled | enabled`  
_업데이트 필요 사항_: 중단 없음

`HttpPutResponseHopLimit`  
인스턴스 메타데이터 요청에 대해 원하는 HTTP PUT 응답 홉 제한입니다. 숫자가 클수록 인스턴스 메타데이터 요청이 더 멀리 이동할 수 있습니다.  
기본값: `1`  
가능한 값: 1에서 64 사이의 정수  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`HttpTokens`  
IMDSv2는 토큰 지원 세션을 사용합니다. HTTP 토큰의 사용을 (즉 `optional` , IMDSv2의 사용을 )로 설정하거나(즉, IMDSv2 `optional` `required` 의 사용을 로 설정)`required`

- `optional` - IMDSv2가 선택 사항인 경우 요청에 세션 토큰을 포함하거나 포함하지 않고 인스턴스 메타데이터를 검색하도록 선택할 수 있습니다. 토큰 없이 IAM 역할 자격 증명을 검색하면 IMDSv1 역할 자격 증명이 반환됩니다. 유효한 세션 토큰을 사용하여 IAM 역할 자격 증명을 검색하면 IMDSv2 역할 자격 증명이 반환됩니다.
- `required` - IMDSv2가 필요한 경우 인스턴스 메타데이터 검색 요청과 함께 세션 토큰을 전송해야 합니다. 이 상태에서 IAM 역할 자격 증명을 검색하면 항상 IMDSv2 자격 증명이 반환됩니다. IMDSv1 자격 증명은 사용할 수 없습니다.
  기본값: `optional`  
  _필수_: 아니요  
  _유형_: 문자열  
  _허용되는 값_: `optional | required`  
  _업데이트 필요 사항_: 중단 없음

`InstanceMetadataTags`  
 `enabled` 인스턴스 메타데이터에서 인스턴스 태그에 대한 액세스를 허용하려면 로 설정합니다. `disabled` 인스턴스 메타데이터에서 인스턴스 태그에 대한 액세스를 끄려면 로 설정합니다. 자세한 내용은 인스턴스 메타데이터를 사용하여 인스턴스 태그 작업을 참조하세요.  
기본값: `disabled`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `disabled | enabled`  
_업데이트 필요 사항_: 중단 없음
