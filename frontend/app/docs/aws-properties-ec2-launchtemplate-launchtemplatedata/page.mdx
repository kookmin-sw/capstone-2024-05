# AWS::EC2::LaunchTemplate Launch템플릿 데이터

시작 템플릿에 포함할 정보입니다.

**메모**  
시작 템플릿 데이터에 대해 하나 이상의 파라미터를 지정해야 합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "BlockDeviceMappings" : [ BlockDeviceMapping, ... ],
  "CapacityReservationSpecification" : CapacityReservationSpecification,
  "CpuOptions" : CpuOptions,
  "CreditSpecification" : CreditSpecification,
  "DisableApiStop" : Boolean,
  "DisableApiTermination" : Boolean,
  "EbsOptimized" : Boolean,
  "ElasticGpuSpecifications" : [ ElasticGpuSpecification, ... ],
  "ElasticInferenceAccelerators" : [ LaunchTemplateElasticInferenceAccelerator, ... ],
  "EnclaveOptions" : EnclaveOptions,
  "HibernationOptions" : HibernationOptions,
  "IamInstanceProfile" : IamInstanceProfile,
  "ImageId" : String,
  "InstanceInitiatedShutdownBehavior" : String,
  "InstanceMarketOptions" : InstanceMarketOptions,
  "InstanceRequirements" : InstanceRequirements,
  "InstanceType" : String,
  "KernelId" : String,
  "KeyName" : String,
  "LicenseSpecifications" : [ LicenseSpecification, ... ],
  "MaintenanceOptions" : MaintenanceOptions,
  "MetadataOptions" : MetadataOptions,
  "Monitoring" : Monitoring,
  "NetworkInterfaces" : [ NetworkInterface, ... ],
  "Placement" : Placement,
  "PrivateDnsNameOptions" : PrivateDnsNameOptions,
  "RamDiskId" : String,
  "SecurityGroupIds" : [ String, ... ],
  "SecurityGroups" : [ String, ... ],
  "TagSpecifications" : [ TagSpecification, ... ],
  "UserData" : String
}
```

## 속성

`BlockDeviceMappings`  
블록 장치 매핑입니다.  
*필수 항목 *여부: 아니요  
_형식_: BlockDeviceMapping 목록  
_업데이트 필요 사항_: 중단 없음

`CapacityReservationSpecification`  
용량 예약 대상 지정 옵션입니다. 이 파라미터를 지정하지 않으면 인스턴스의 용량 예약 기본 설정이 기본적으로 로 설정되며, `open`이를 통해 일치하는 속성(인스턴스 유형, 플랫폼, 가용 영역)이 있는 열려 있는 모든 용량 예약에서 인스턴스를 실행할 수 있습니다.  
*필수 항목 *여부: 아니요  
_형식_: CapacityReservationSpecification  
_업데이트 필요 사항_: 중단 없음

`CpuOptions`  
인스턴스의 CPU 옵션입니다. 자세한 내용은 _Amazon Elastic Compute Cloud 사용 설명서의 CPU 옵션 최적화를 참조하세요_.  
*필수 항목 *여부: 아니요  
_형식_: CpuOptions  
_업데이트 필요 사항_: 중단 없음

`CreditSpecification`  
인스턴스의 CPU 사용량에 대한 크레딧 옵션입니다. T 인스턴스에만 유효합니다.  
*필수 항목 *여부: 아니요  
_유형_: CreditSpecification  
_업데이트 필요 사항_: 중단 없음

`DisableApiStop`  
중지 보호를 위해 인스턴스를 활성화할지 여부를 나타냅니다. 자세한 내용은 _Amazon Elastic Compute Cloud 사용 설명서의 보호 중지를 참조하세요_.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`DisableApiTermination`  
이 파라미터를 `true`로 설정하면 Amazon EC2 콘솔, CLI 또는 API를 사용하여 인스턴스를 종료할 수 없으며, 그렇지 않으면 종료할 수 있습니다. 시작 후 이 속성을 변경하려면 ModifyInstanceAttribute를 사용합니다. 또는 로 설정하면 `InstanceInitiatedShutdownBehavior` `terminate`인스턴스에서 shutdown 명령을 실행하여 인스턴스를 종료할 수 있습니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`EbsOptimized`  
인스턴스가 Amazon EBS I/O에 최적화되어 있는지 여부를 나타냅니다. 이 최적화는 최적의 Amazon EBS I/O 성능을 제공하기 위해 Amazon EBS에 전용 처리량과 최적화된 구성 스택을 제공합니다. 이 최적화는 모든 인스턴스 유형에서 사용할 수 있는 것은 아닙니다. EBS 최적화 인스턴스를 사용하는 경우 추가 사용 요금이 적용됩니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`ElasticGpuSpecifications`  
인스턴스와 연결할 탄력적 GPU입니다.  
*필수 항목 *여부: 아니요  
_유형_: ElasticGpuSpecification 목록  
_업데이트 필요 사항_: 중단 없음

`ElasticInferenceAccelerators`  
인스턴스와 연결할 탄력적 추론 액셀러레이터입니다. Elastic 추론 액셀러레이터는 딥 러닝(DL) 추론 워크로드를 가속화하기 위해 Amazon EC2 인스턴스에 연결할 수 있는 리소스입니다.  
동일한 요청에서 서로 다른 세대의 액셀러레이터를 지정할 수 없습니다.  
2023년 4월 15일부터 AWS는 신규 고객을 Amazon Elastic Inference(EI)에 온보딩하지 않으며, 기존 고객이 더 나은 가격과 성능을 제공하는 옵션으로 워크로드를 마이그레이션할 수 있도록 지원합니다. 2023년 4월 15일 이후에는 신규 고객이 Amazon SageMaker, Amazon ECS 또는 Amazon EC2에서 Amazon EI 액셀러레이터를 사용하여 인스턴스를 시작할 수 없습니다. 그러나 지난 30일 동안 Amazon EI를 한 번 이상 사용한 고객은 현재 고객으로 간주되며 서비스를 계속 사용할 수 있습니다.
*필수 항목 *여부: 아니요  
_형식_: LaunchTemplateElasticInferenceAccelerator 목록  
_업데이트 필요 사항_: 중단 없음

`EnclaveOptions`  
인스턴스가 AWS Nitro Enclaves에 대해 활성화되었는지 여부를 나타냅니다. 자세한 내용은 AWS Nitro 엔클레이브란 무엇입니까?를 참조하십시오. 의AWSNitro 엔클레이브 사용 설명서*.  
동일한 인스턴스에서 AWS Nitro Enclaves 및 최대 절전 모드를 활성화할 수 없습니다.  
*필수 항목 *여부: 아니요  
*형식*: EnclaveOptions  
*업데이트 필요 사항\*: 중단 없음

`HibernationOptions`  
인스턴스가 최대 절전 모드를 사용할 수 있는지 여부를 나타냅니다. 이 파라미터는 인스턴스가 최대 절전 모드 사전 요구 사항을 충족하는 경우에만 유효합니다. 자세한 내용은 _Amazon Elastic Compute Cloud 사용 설명서의 인스턴스 하이버네이트를 참조하세요_.  
*필수 항목 *여부: 아니요  
_형식_: HibernationOptions  
_업데이트 필요 사항_: 중단 없음

`IamInstanceProfile`  
IAM 인스턴스 프로파일의 이름 또는 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_형식_: IamInstanceProfile  
_업데이트 필요 사항_: 중단 없음

`ImageId` AMI의 ID입니다. 또는 시작 시 AMI ID로 확인되는 Systems Manager 파라미터를 지정할 수 있습니다.  
유효한 형식:

- `ami-17characters00000`
- `resolve:ssm:parameter-name`
- `resolve:ssm:parameter-name:version-number`
- `resolve:ssm:parameter-name:label`
  자세한 내용은 _Amazon Elastic Compute Cloud 사용 설명서의 Systems Manager 파라미터를 사용하여 AMI 찾기를 참조하세요_.  
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _업데이트 필요 사항_: 중단 없음

`InstanceInitiatedShutdownBehavior`  
인스턴스에서 종료를 시작할 때 인스턴스가 중지되는지 또는 종료되는지를 나타냅니다(시스템 종료를 위해 운영 체제 명령 사용).  
기본값: `stop`  
_필수_: 아니요  
_유형_: 문자열  
_허용되는 값_: `stop | terminate`  
_업데이트 필요 사항_: 중단 없음

`InstanceMarketOptions`  
인스턴스에 대한 시장(구매) 옵션입니다.  
*필수 항목 *여부: 아니요  
_형식_: InstanceMarketOptions  
_업데이트 필요 사항_: 중단 없음

`InstanceRequirements`  
인스턴스 유형에 대한 속성입니다. 인스턴스 속성을 지정하면 Amazon EC2가 이러한 속성을 가진 인스턴스 유형을 식별합니다.  
및 를 지정해야 합니다 `VCpuCount` `MemoryMiB`. 다른 모든 특성은 선택 사항입니다. 지정되지 않은 선택적 특성은 기본값으로 설정됩니다.  
여러 속성을 지정하면 지정된 속성을 모두 충족하는 인스턴스 유형을 얻을 수 있습니다. 속성에 대해 여러 값을 지정하면 지정된 값 중 하나를 충족하는 인스턴스 유형을 가져옵니다.  
Amazon EC2가 일치하는 인스턴스 유형을 식별할 수 있는 인스턴스 유형 목록을 제한하려면 다음 파라미터 중 하나를 사용할 수 있지만 동일한 요청에서 둘 다 사용할 수는 없습니다.

- `AllowedInstanceTypes` - 목록에 포함할 인스턴스 유형입니다. 다른 모든 인스턴스 유형은 지정된 속성과 일치하더라도 무시됩니다.
- `ExcludedInstanceTypes` - 목록에서 제외할 인스턴스 유형(지정된 속성과 일치하는 경우에도 해당).
  를 지정`InstanceRequirements`하면 를 지정할 수 없습니다`InstanceType`.  
  속성 기반 인스턴스 유형 선택은 Auto Scaling 그룹, EC2 집합 및 스팟 집합을 사용하여 인스턴스를 시작하는 경우에만 지원됩니다. 인스턴스 시작 마법사에서 시작 템플릿을 사용하거나 RunInstances API 또는 AWS::EC2::Instance AWS CloudFormation 리소스와 함께 사용하려는 경우 를 지정할 수 없습니다`InstanceRequirements`.
  자세한 내용은 Amazon EC2 사용 설명서의 EC2 집합에 대한 속성 기반 인스턴스 유형 선택, 스팟 집합에 대한 속성 기반 인스턴스 유형 선택 및 스팟 배치 점수를 참조하세요\**.  
  *필수 항목 *여부: 아니요  
  *형식*: InstanceRequirements  
  *업데이트 필요 사항\*: 중단 없음

`InstanceType`  
인스턴스 유형입니다. 자세한 내용은 *Amazon Elastic Compute Cloud 사용 설명서*의 인스턴스 유형을 참조하세요.
