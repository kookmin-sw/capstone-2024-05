# AWS::ECS::TaskDefinition ContainerDependency

이 `ContainerDependency` 속성은 컨테이너 시작 및 종료에 대해 정의된 종속성을 지정합니다. 컨테이너에는 여러 종속성이 포함될 수 있습니다. 컨테이너 시작에 대한 종속성이 정의되면 컨테이너 종료의 경우 반대가 됩니다.

컨테이너 종속성을 활성화하려면 Amazon ECS 컨테이너 인스턴스에 컨테이너 에이전트 버전 1.26.0 이상이 필요합니다. 그러나 최신 컨테이너 에이전트 버전을 사용하는 것이 좋습니다. 에이전트 버전 확인 및 최신 버전 업데이트에 대한 자세한 내용은 *Amazon Elastic Container Service 개발자 안내서*의 Amazon ECS 컨테이너 에이전트 업데이트를 참조하세요. Amazon ECS 최적화 Linux AMI를 사용하는 경우 인스턴스에 패키지 버전 1.26.0-1 이상이 필요합니다 `ecs-init` . 컨테이너 인스턴스가 버전 이상에서 시작되는 경우 `20190301` 컨테이너 에이전트의 필수 버전과 . `ecs-init` 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Amazon ECS 최적화 Linux AMI를 참조하세요_.

**메모**  
Fargate 시작 유형을 사용하는 태스크의 경우 이 파라미터를 사용하려면 태스크 또는 서비스가 플랫폼 버전 1.3.0 이상을 사용해야 합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Condition" : String,
  "ContainerName" : String
}
```

## 속성

`Condition` 컨테이너의 종속성 조건입니다. 사용 가능한 조건과 해당 동작은 다음과 같습니다.

- `START` - 이 조건은 현재 링크 및 볼륨의 동작을 에뮬레이트합니다. 다른 컨테이너의 시작을 허용하기 전에 종속 컨테이너가 시작되었는지 확인합니다.
- `COMPLETE` - 이 조건은 다른 컨테이너의 시작을 허용하기 전에 종속 컨테이너가 완료(종료)될 때까지 실행되는지 확인합니다. 이는 스크립트를 실행한 다음 종료하는 필수적이지 않은 컨테이너에 유용할 수 있습니다. 이 조건은 필수 컨테이너에서 설정할 수 없습니다.
- `SUCCESS` - 이 조건은 와 동일`COMPLETE`하지만 컨테이너가 상태로 종료되어야 합니다 `zero` . 이 조건은 필수 컨테이너에서 설정할 수 없습니다.
- `HEALTHY` - 이 조건은 다른 컨테이너의 시작을 허용하기 전에 종속 컨테이너가 Docker 상태 확인을 통과하는지 확인합니다. 이를 위해서는 종속 컨테이너에 상태 확인이 구성되어 있어야 합니다. 이 조건은 작업 시작 시에만 확인됩니다.
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _허용되는 값_: `COMPLETE | HEALTHY | START | SUCCESS`  
  _업데이트 필요 사항_: 바꿔 놓음

`ContainerName`  
컨테이너의 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음
