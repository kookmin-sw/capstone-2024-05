# AWS::EC2::VPCDHCPOptionsAssociation

DHCP 옵션 집합을 VPC에 연결하거나 DHCP 옵션을 VPC에 연결하지 않습니다.

옵션을 VPC와 연결한 후 기존 인스턴스와 해당 VPC에서 시작하는 모든 새 인스턴스는 옵션을 사용합니다. 인스턴스를 다시 시작하거나 다시 시작할 필요가 없습니다. 인스턴스가 DHCP 임대를 갱신하는 빈도에 따라 몇 시간 내에 변경 사항을 자동으로 선택합니다. 인스턴스의 운영 체제를 사용하여 임대를 명시적으로 갱신할 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::VPCDHCPOptionsAssociation",
  "Properties" : {
      "DhcpOptionsId" : String,
      "VpcId" : String
    }
}
```

## 속성

`DhcpOptionsId`  
DHCP 옵션 집합의 ID이거나 `default` DHCP 옵션을 VPC와 연결하지 않습니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`VpcId`  
VPC의 ID입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`DHCP 옵션 연결의 ID가 반환됩니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

## 예제

### VPC DHCP 옵션 연결

다음 예제에서는 `Ref` intrinsic 함수를 사용하여 myDHCPOptions DHCP 옵션을 myVPC VPC와 연결합니다. VPC 및 DHCP 옵션은 동일한 템플릿에서 선언하거나 입력 파라미터로 추가할 수 있습니다. VPC 또는 DHCP 옵션 리소스에 대한 자세한 내용은 AWS::EC2::VPC 또는 AWS::EC2::D HCPOptions 단원을 참조하십시오.

#### JSON (영문)

```
"myVPCDHCPOptionsAssociation" : {
     "Type" : "AWS::EC2::VPCDHCPOptionsAssociation",
      "Properties" : {
         "VpcId" : {"Ref" : "myVPC"},
         "DhcpOptionsId" : {"Ref" : "myDHCPOptions"}
         }
}
```
