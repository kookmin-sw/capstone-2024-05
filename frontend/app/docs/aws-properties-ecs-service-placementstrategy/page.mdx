# AWS::ECS::서비스 배치 전략

이 `PlacementStrategy` 속성은 작업 또는 서비스에 대한 작업 배치 전략을 지정합니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 작업 배치 전략을 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Field" : String,
  "Type" : String
}
```

## 속성

`Field`  
배치 전략을 적용할 필드입니다. 배치 전략의 경우 `spread` 유효한 값은 `instanceId` (또는 `host`동일한 효과가 있음) 또는 컨테이너 인스턴스에 적용되는 플랫폼 또는 사용자 지정 속성(예: )입니다`attribute:ecs.availability-zone`. `binpack` 배치 전략의 경우 유효한 값은 `CPU` 및 `MEMORY`입니다. `random` 배치 전략의 경우 이 필드는 사용되지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Type`  
배치 전략의 유형입니다. `random` 배치 전략은 사용 가능한 후보자에게 작업을 무작위로 배치합니다. `spread` 배치 전략은 매개변수에 따라 사용 가능한 후보자 간에 배치를 균등하게 분산합니다 `field` . 이 전략은 `binpack` 매개 변수 로 지정된 자원의 사용 가능한 양이 가장 적은 사용 가능한 후보에게 작업을 배치합니다 `field` . 예를 들어, 메모리에 binpack을 사용하는 경우 남은 메모리 양이 가장 적지만 작업을 실행하기에 충분한 인스턴스가 인스턴스에 작업이 배치됩니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `binpack | random | spread`  
_업데이트 필요 사항_: 중단 없음
