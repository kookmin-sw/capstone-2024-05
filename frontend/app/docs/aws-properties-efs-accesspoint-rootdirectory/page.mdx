# AWS::EFS::액세스 포인트 루트 디렉터리

액세스 포인트가 액세스를 제공하는 Amazon EFS 파일 시스템의 디렉터리를 지정합니다. 액세스 포인트는 지정된 파일 시스템 경로를 액세스 포인트를 사용하는 애플리케이션에 파일 시스템의 루트 디렉터리로 노출합니다. 액세스 포인트를 사용하는 NFS 클라이언트는 액세스 포인트 및 액세스 포인트의 하위 디렉토리에 있는 데이터에만 액세스할 수 있습니다 `RootDirectory` .

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "CreationInfo" : CreationInfo,
  "Path" : String
}
```

## 속성

`CreationInfo`  
(선택 사항) 액세스 포인트의 `RootDirectory`. 지정된 `RootDirectory` > `Path` 존재하지 않는 경우 EFS는 `CreationInfo` 클라이언트가 액세스 포인트에 연결할 때 설정을 사용하여 루트 디렉터리를 생성합니다. 를 지정할 때는 `CreationInfo`모든 속성에 대한 값을 제공해야 합니다.  
제공하지 않고 `CreationInfo` 지정된 `RootDirectory` > `Path` 존재하지 않는 경우 액세스 포인트를 사용하여 파일 시스템을 마운트하려는 시도가 실패합니다.
*필수 항목 *여부: 아니요  
_형식_: CreationInfo  
_업데이트 필요 사항_: 바꿔 놓음

`Path`  
EFS 파일 시스템에 액세스하기 위해 액세스 포인트를 사용하여 NFS 클라이언트에 루트 디렉터리로 노출할 EFS 파일 시스템의 경로를 지정합니다. 경로에는 최대 4개의 하위 디렉터리가 있을 수 있습니다. 지정된 경로가 없으면 `CreationInfo`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
*최소*값: `1`  
*최대*값: `100`  
_패턴_: `^(/|(/(?!.)+[^$#<>;`|&?{} ^*/n]+){1,4})$'  
*업데이트 필요 사항\*: 바꿔 놓음
