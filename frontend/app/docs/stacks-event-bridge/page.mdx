# AWS CloudFormation 및 Amazon EventBridge를 사용하여 이벤트 관리

AWS CloudFormation은 스택에서 생성, 업데이트, 삭제 또는 드리프트 감지 작업이 수행될 때마다 Amazon EventBridge로 이벤트를 전송할 수 있습니다. 또한 AWS CloudFormation은 StackSets의 상태 변경을 위해 Amazon EventBridge에 이벤트를 보냅니다. EventBridge 규칙을 사용하여 정의된 대상으로 이벤트를 라우팅할 수 있습니다. 이러한 이벤트는 최선을 다해 전달되며 순서에 맞지 않게 전달될 수 있습니다.

CloudFormation 이벤트를 검색하고 해당 이벤트를 기반으로 추가 워크플로를 설정합니다. CloudFormation은 CloudFormation 스택 또는 StackSets와 해당 리소스의 변경 사항에 대한 정보를 제공하므로 각 이벤트와 연결된 워크플로를 구독하고 모방할 수 있습니다. 예를 들어:

- AWS CloudFormation을 통해 프로비저닝된 모든 리소스에 대해 스택 또는 StackSet 특정 태그를 생성합니다.
- CloudFormation 스택 또는 StackSet과 Amazon WorkSpaces Application Manager(Amazon WAM) 간의 연결을 설정합니다.
- 생성된 스택 또는 StackSet에 대해 AppRegistry와의 연결을 지정합니다.

## 지원되는 이벤트

모든 이벤트는 스택 및 StackSet 작업과 관련된 생성, 업데이트, 삭제(CUD) 및 드리프트 감지 작업에 의해 제공됩니다. 자세한 정보는 이벤트 유형 메시지 구조를 참조하십시오.

CloudFormation에서 지원되는 이벤트는 다음과 같습니다.

| 이벤트 유형                 | 묘사                                                                                                                                                                                                   |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 리소스 상태                 | 기본 리소스 속성을 변경하는 스택에서 수행되는 모든 업데이트입니다. 지원되는 AWS 리소스 유형의 전체 목록은 AWS 리소스 및 속성 유형 참조 섹션을 참조하세요                                               |
| 스택 상태                   | 사용자가 프로비저닝한 스택에 대한 모든 상태 변경 업데이트. 스택 상태 코드의 전체 목록은 스택 상태 코드 섹션을 참조하세요.                                                                              |
| 드리프트 감지 상태          | 사용자가 스택에서 드리프트 감지 업데이트를 시작했습니다. 드리프트 감지를 지원하는 완전히 변경 가능하고 변경할 수 없는 유형의 전체 목록은 가져오기 및 드리프트 감지 작업을 지원하는 리소스를 참조하세요 |
| StackSet 상태               | 특정 스택 세트에 대한 상태 변경. 상태는 ACTIVE 및 DELETED로 제한됩니다.                                                                                                                                |
| StackSet 스택 인스턴스 상태 | 특정 StackSet Stack 인스턴스에 대한 상태 변경. StackSet 내 스택 인스턴스에 대한 상태 코드의 전체 목록은 스택 인스턴스 상태 코드 섹션을 참조하세요.                                                     |
| StackSet 작업 상태          | 지정된 StackSet 작업에 대한 상태 변경. StackSet 작업에 대한 상태 코드의 전체 목록은 StackSet 상태 코드 섹션을 참조하세요.                                                                              |
