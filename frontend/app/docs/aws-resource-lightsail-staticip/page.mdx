# AWS::Lightsail::정적 IP

리소스는 `AWS::Lightsail::StaticIp` 동일한 AWS 리전 및 가용 영역에 있는 Amazon Lightsail 인스턴스에 연결할 수 있는 고정 IP를 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Lightsail::StaticIp",
  "Properties" : {
      "AttachedTo" : String,
      "StaticIpName" : String
    }
}
```

## 속성

`AttachedTo`  
고정 IP가 연결된 인스턴스입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`StaticIpName`  
고정 IP의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`이 `Ref`리소스에 대한 고유 식별자를 반환합니다.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`IpAddress` 고정 IP의 IP 주소입니다.

`IsAttached` 고정 IP가 인스턴스에 연결되어 있는지 여부를 나타내는 부울 값입니다.

`StaticIpArn`  
고정 IP의 Amazon 리소스 이름(ARN)입니다(예: `arn:aws:lightsail:us-east-2:123456789101:StaticIp/244ad76f-8aad-4741-809f-12345EXAMPLE`).

## 발언

_인스턴스가 고정 IP를 연결하려면 실행 중 상태여야 합니다_

인스턴스에 고정 IP를 연결하려면 인스턴스가 상태여야 합니다 `running` . `running` 고정 IP 연결 요청을 수행한 후 15분 이내에 인스턴스가 상태가 되지 않으면 고정 IP 연결 요청 시간이 초과됩니다.

_인스턴스에는 하나의 고정 IP만 연결할 수 있습니다_

단일 인스턴스에 하나의 고정 IP를 연결할 수 있습니다. 하나의 인스턴스에 여러 고정 IP를 연결할 수 없습니다. 여러 고정 IP의 매개 변수에 동일한 인스턴스가 있는 경우 `AttachedTo` 동작을 예측할 수 없으며 고정 IP(하나만)가 인스턴스에 연결될 수 있습니다. 이렇게 하면 고정 IP 중 하나만 인스턴스에 연결되지만 템플릿에는 여러 개가 표시되기 때문에 스택이 드리프트됩니다.
