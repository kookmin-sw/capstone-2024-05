# AWS::백업::프레임워크

하나 이상의 컨트롤이 있는 프레임워크를 만듭니다. 프레임워크는 백업 방법을 평가하는 데 사용할 수 있는 컨트롤 모음입니다. 미리 빌드된 사용자 지정 가능한 컨트롤을 사용하여 정책을 정의하면 백업 방법이 정책을 준수하는지 여부와 아직 준수하지 않는 리소스를 평가할 수 있습니다.

샘플 AWS CloudFormation 템플릿은 AWS Backup 개발자 안내서를 참조하십시오.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Backup::Framework",
  "Properties" : {
      "FrameworkControls" : [ FrameworkControl, ... ],
      "FrameworkDescription" : String,
      "FrameworkName" : String,
      "FrameworkTags" : [Tag, ... ]
    }
}
```

## 속성

`FrameworkControls`  
프레임워크의 모든 컨트롤에 대한 자세한 정보를 포함합니다. 각 프레임워크에는 하나 이상의 컨트롤이 포함되어야 합니다.  
_필수 항목 여부_: 예  
_형식_: FrameworkControl 목록  
_업데이트 필요 사항_: 중단 없음

`FrameworkDescription`  
최대 1,024자의 프레임워크에 대한 선택적 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `0`  
_최대_: `1024`  
_패턴_: `.*S.*`  
_업데이트 필요 사항_: 중단 없음

`FrameworkName`  
프레임워크의 고유한 이름입니다. 이 이름은 1자에서 256자 사이이며 문자로 시작하고 문자(a-z, A-Z), 숫자(0-9) 및 밑줄(\_)로 구성됩니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `1`  
_최대_: `256`  
_패턴_: `[a-zA-Z][_a-zA-Z0-9]*`  
_업데이트 필요 사항_: 바꿔 놓음

`FrameworkTags`  
프레임워크에 태그를 지정하는 데 사용할 태그 목록입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

### Fn::GetAtt

####

`CreationTime` 프레임워크를 만든 UTC 시간입니다.

`DeploymentStatus`  
Depolyment 상태는 프레임워크가 배포를 완료했는지 여부를 나타냅니다. 이 상태는 일반적으로 `Completed`이지만 `Create in progress` 또는 다른 상태일 수도 있습니다. 상태 목록은 AWS Backup의 프레임워크 규정 준수 상태 섹션을 참조하세요 _. 개발자 안내서_.

`FrameworkArn` 프레임워크의 Amazon 리소스 이름(ARN)입니다.

`FrameworkStatus`  
프레임워크 상태는 모든 리소스에 대해 리소스 추적을 설정했는지 여부를 나타냅니다. 이 상태는 `Active` 프레임워크가 평가하는 모든 리소스를 켤 때입니다. 다른 상태 및 수정 단계는 AWS Backup의 프레임워크 규정 준수 상태 섹션을 참조하세요 _. 개발자 안내서_.
