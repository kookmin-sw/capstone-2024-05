# AWS::EC2::Launch템플릿 스팟옵션

스팟 인스턴스에 대한 옵션을 지정합니다.

`SpotOptions` 는 AWS::EC2::LaunchTemplate InstanceMarketOptions의 속성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "BlockDurationMinutes" : Integer,
  "InstanceInterruptionBehavior" : String,
  "MaxPrice" : String,
  "SpotInstanceType" : String,
  "ValidUntil" : String
}
```

## 속성

`BlockDurationMinutes`  
되지 않는.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`InstanceInterruptionBehavior`  
스팟 인스턴스가 중단될 때의 동작입니다. 기본값은 `terminate`입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `hibernate | stop | terminate`  
_업데이트 필요 사항_: 중단 없음

`MaxPrice`  
스팟 인스턴스에 대해 지불할 의사가 있는 최대 시간당 요금입니다. 이 매개 변수는 중단이 증가할 수 있으므로 사용하지 않는 것이 좋습니다. 이 파라미터를 지정하지 않으면 현재 스팟 가격을 지불하게 됩니다.  
최고 가격을 지정하면 이 파라미터를 지정하지 않을 때보다 스팟 인스턴스가 더 자주 중단됩니다.
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`SpotInstanceType`  
스팟 인스턴스 요청 유형입니다.  
오토 스케일링과 함께 스팟 인스턴스를 사용하는 경우 `one-time` 그룹이 원하는 용량 미만일 때마다 Amazon EC2 Auto Scaling 서비스가 새 스팟 인스턴스 요청을 처리하므로 requests를 사용합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `one-time | persistent`  
_업데이트 필요 사항_: 중단 없음

`ValidUntil`  
UTC 형식(*YYYY-MM-DD*T*HH:MM:SS* Z)의 요청 종료 날짜입니다. 영구 요청에 대해서만 지원됩니다.

- 지속적 요청의 경우 요청은 `ValidUntil` 날짜 및 시간에 도달할 때까지 활성 상태로 유지됩니다. 그렇지 않으면 요청을 취소할 때까지 요청이 활성 상태로 유지됩니다.
- 일회성 요청의 경우 `ValidUntil` 지원되지 않습니다. 요청은 모든 인스턴스가 시작되거나 사용자가 요청을 취소할 때까지 활성 상태로 유지됩니다.
  기본값: 현재 날짜로부터 7일  
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _업데이트 필요 사항_: 중단 없음
