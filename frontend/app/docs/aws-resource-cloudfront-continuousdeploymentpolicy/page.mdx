# AWS::CloudFront::ContinuousDeploymentPolicy

운영 트래픽의 하위 집합을 기본 배포에서 스테이징 배포로 라우팅하는 지속적 배포 정책을 생성합니다.

스테이징 배포를 생성하고 업데이트한 후 지속적 배포 정책을 사용하여 트래픽을 스테이징 배포로 점진적으로 이동할 수 있습니다. 이렇게 하면 모든 프로덕션 트래픽을 새 구성으로 이동하기 전에 배포 구성에 대한 변경 사항을 테스트할 수 있습니다.

자세한 내용은 Amazon CloudFront 개발자 안내서*의 CloudFront 지속적 배포를 사용하여 CDN 구성 변경 사항을 안전하게 테스트하기를 참조하세요*.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::CloudFront::ContinuousDeploymentPolicy",
  "Properties" : {
      "ContinuousDeploymentPolicyConfig" : ContinuousDeploymentPolicyConfig
    }
}
```

## 속성

`ContinuousDeploymentPolicyConfig`  
지속적인 배포 정책에 대한 구성을 포함합니다.  
_필수 항목 여부_: 예  
_형식_: ContinuousDeploymentPolicyConfig  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`지속적 배포 정책에 대한 고유 식별자가 반환됩니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Id` 연속 배포 정책의 식별자입니다.

`LastModifiedTime` 지속적인 배포 정책을 마지막으로 수정한 날짜 및 시간입니다.
