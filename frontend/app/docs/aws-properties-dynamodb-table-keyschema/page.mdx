# AWS::D ynamoDB::테이블 키 스키마

키 스키마의 단일 요소를* 나타냅니다 * . 키 스키마는 테이블의 기본 키를 구성하는 속성 또는 인덱스의 키 속성을 지정합니다.

A `KeySchemaElement` 는 기본 키의 정확히 하나의 특성을 나타냅니다. 예를 들어 단순 기본 키는 파티션 키의 경우 하나로 표시됩니다 `KeySchemaElement` . 복합 기본 키에는 `KeySchemaElement` 파티션 키에 대한 기본 키와 `KeySchemaElement` 정렬 키에 대한 기본 키가 필요합니다.

A `KeySchemaElement` 는 스칼라의 최상위 특성(중첩 특성이 아님)이어야 합니다. 데이터 형식은 String, Number 또는 Binary 중 하나여야 합니다. 애트리뷰트는 목록 또는 맵 내에 중첩될 수 없습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AttributeName" : String,
  "KeyType" : String
}
```

## 속성

`AttributeName`  
키 특성의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `255`  
_업데이트 필요 사항_: 중단 없음

`KeyType` 이 키 속성이 맡는 역할은 다음과 같습니다.

- `HASH` - 파티션 키
- `RANGE` - 정렬 키
  항목의 파티션 키는 *해시 특성*이라고도 합니다. "해시 속성"이라는 용어는 DynamoDB가 내부 해시 함수를 사용하여 파티션 키 값을 기반으로 데이터 항목을 파티션 간에 균등하게 분산하는 데서 유래했습니다.  
  항목의 정렬 키는 *range 속성*이라고도 합니다. "범위 속성"이라는 용어는 DynamoDB가 동일한 파티션 키를 가진 항목을 정렬 키 값을 기준으로 정렬된 순서로 물리적으로 서로 가깝게 저장하는 방식에서 파생됩니다.
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _업데이트 필요 사항_: 중단 없음
