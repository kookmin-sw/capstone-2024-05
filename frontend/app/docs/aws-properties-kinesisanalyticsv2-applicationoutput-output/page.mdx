# AWS::KinesisAnalyticsV2::ApplicationOutput 출력

SQL 기반 Managed Service for Apache Flink 애플리케이션의 출력 구성에 대해 설명하며, 여기서 인애플리케이션 스트림 및 인애플리케이션 스트림 데이터를 기록할 대상을 식별합니다. 대상은 Kinesis 데이터 스트림 또는 Kinesis Data Firehose 전송 스트림일 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "DestinationSchema" : DestinationSchema,
  "KinesisFirehoseOutput" : KinesisFirehoseOutput,
  "KinesisStreamsOutput" : KinesisStreamsOutput,
  "LambdaOutput" : LambdaOutput,
  "Name" : String
}
```

## 속성

`DestinationSchema`  
레코드가 대상에 기록될 때의 데이터 형식에 대해 설명합니다.  
_필수 항목 여부_: 예  
_형식_: DestinationSchema  
_업데이트 필요 사항_: 중단 없음

`KinesisFirehoseOutput`  
Kinesis Data Firehose 전송 스트림을 대상으로 식별합니다.  
*필수 항목 *여부: 아니요  
_유형_: KinesisFirehoseOutput  
_업데이트 필요 사항_: 중단 없음

`KinesisStreamsOutput`  
Kinesis 데이터 스트림을 대상으로 식별합니다.  
*필수 항목 *여부: 아니요  
_유형_: KinesisStreamsOutput  
_업데이트 필요 사항_: 중단 없음

`LambdaOutput`  
Amazon Lambda 함수를 대상으로 식별합니다.  
*필수 항목 *여부: 아니요  
_유형_: LambdaOutput  
_업데이트 필요 사항_: 중단 없음

`Name`  
인애플리케이션 스트림의 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `1`  
_최대_: `32`  
_패턴_: `[^-s<>&]*`  
_업데이트 필요 사항_: 바꿔 놓음

## 참고 항목

- Amazon Kinesis Data Analytics API 참조\* \*의 출력
