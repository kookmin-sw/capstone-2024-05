# AWS::Cognito::UserPoolGroup

식별된 사용자 풀에서 새 그룹을 지정합니다.

이 작업을 호출하려면 개발자 자격 증명이 필요합니다.

**메모**  
파라미터 값을 지정하지 않으면 Amazon Cognito가 파라미터 값을 기본값으로 설정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Cognito::UserPoolGroup",
  "Properties" : {
      "Description" : String,
      "GroupName" : String,
      "Precedence" : Double,
      "RoleArn" : String,
      "UserPoolId" : String
    }
}
```

## 속성

`Description`  
그룹에 대한 설명이 포함된 문자열입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `2048`  
_업데이트 필요 사항_: 중단 없음

`GroupName`  
그룹의 이름입니다. 고유해야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `1`  
_최대_: `128`  
_패턴_: `[p{L}p{M}p{S}p{N}p{P}]+`  
_업데이트 필요 사항_: 바꿔 놓음

`Precedence`  
음수가 아닌 정수 값으로, 사용자가 사용자 풀에서 속할 수 있는 다른 그룹에 상대적인 이 그룹의 우선 순위를 지정합니다. 0은 가장 높은 우선 순위 값입니다. 값이 낮은 그룹은 `Precedence` 값이 높거나 null 값인 그룹보다 우선합니다 `Precedence` . 사용자가 두 개 이상의 그룹에 속하는 경우 우선 순위 값이 가장 낮은 그룹이며, 해당 그룹의 역할 ARN은 및 클레임에 대한 사용자의 토큰에 제공됩니다 `cognito:roles` `cognito:preferred_role` .  
두 그룹이 동일한 값을 가질 수 있습니다 `Precedence` . 이 경우 두 그룹 모두 다른 그룹보다 우선하지 않습니다. 동일한 두 그룹의 `Precedence` 역할 ARN이 동일한 경우 해당 역할은 `cognito:preferred_role` 각 그룹의 사용자에 대한 토큰의 클레임에 사용됩니다. 두 그룹의 역할 ARN이 서로 다른 경우 `cognito:preferred_role` 클레임이 사용자의 토큰에 설정되지 않습니다.  
기본값은 `Precedence` null입니다. 최대 `Precedence` 값은 `2^31-1`입니다.  
*필수 항목 *여부: 아니요  
_유형_: 더블  
_최소_: `0`  
_업데이트 필요 사항_: 중단 없음

`RoleArn`  
그룹의 역할 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최소_: `20`  
_최대_: `2048`  
_패턴_: `arn:[w+=/,.@-]+:[w+=/,.@-]+:([w+=/,.@-]*)?:[0-9]+:[w+=/,.@-]+(:[w+=/,.@-]+)?(:[w+=/,.@-]+)?`  
_업데이트 필요 사항_: 중단 없음

`UserPoolId`  
사용자 풀의 사용자 풀 ID입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `55`  
_패턴_: `[w-]+_[0-9a-zA-Z]+`  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`사용자 풀 그룹의 이름이 반환됩니다. 예를 들면 다음과 같습니다 `Admins`.
