# AWS::EC2::IPAMResourceDiscovery

리소스 검색은 IPAM이 소유 계정에 속한 리소스를 관리하고 모니터링할 수 있도록 하는 IPAM 구성 요소입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::IPAMResourceDiscovery",
  "Properties" : {
      "Description" : String,
      "OperatingRegions" : [ IpamOperatingRegion, ... ],
      "Tags" : [Tag, ... ]
    }
}
```

## 속성

`Description`  
자원 발견 설명입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`OperatingRegions`  
리소스 검색을 위한 운영 지역입니다. 운영 리전은 IPAM이 IP 주소 CIDR을 관리할 수 있는 AWS 리전입니다. IPAM은 운영 리전으로 선택한 AWS 리전의 리소스만 검색하고 모니터링합니다.  
*필수 항목 *여부: 아니요  
_형식_: IpamOperatingRegion 목록  
_업데이트 필요 사항_: 중단 없음

`Tags`  
태그는 AWS 리소스에 할당하는 레이블입니다. 각 태그는 키와 선택적 값으로 구성됩니다. 태그를 사용하여 리소스를 검색 및 필터링하거나 AWS 비용을 추적할 수 있습니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`리소스 검색 ID가 반환됩니다. 예를 들면 다음과 같습니다 `ipam-res-disco-111122223333`.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`IpamResourceDiscoveryArn` 리소스 검색 ARN입니다.

`IpamResourceDiscoveryId` 리소스 검색 ID입니다.

`IpamResourceDiscoveryRegion` 리소스 검색 리전입니다.

`IsDefault` 자원 검색이 기본값인지 여부를 정의합니다. 기본 리소스 검색은 IPAM을 생성할 때 자동으로 생성되는 리소스 검색입니다.

`OwnerId` 소유자 ID입니다.

`State` 리소스 검색의 상태입니다.

- `create-in-progress` - 리소스 검색이 생성되고 있습니다.
- `create-complete` - 리소스 검색 만들기가 완료되었습니다.
- `create-failed` - 리소스 검색 만들기에 실패했습니다.
- `modify-in-progress` - 리소스 검색이 수정되고 있습니다.
- `modify-complete` - 리소스 검색 수정이 완료되었습니다.
- `modify-failed` - 리소스 검색 수정에 실패했습니다.
- `delete-in-progress` - 리소스 검색이 삭제되고 있습니다.
- `delete-complete` - 리소스 검색 삭제가 완료되었습니다.
- `delete-failed` - 리소스 검색 삭제에 실패했습니다.
- `isolate-in-progress` - 리소스 검색을 생성한 AWS 계정이 제거되었으며 리소스 검색이 격리되고 있습니다.
- `isolate-complete` - 리소스 검색 격리가 완료되었습니다.
- `restore-in-progress` - 리소스 검색을 생성하고 격리된 AWS 계정이 복원되었습니다.
