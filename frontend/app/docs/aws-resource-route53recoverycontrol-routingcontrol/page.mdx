# AWS::Route53RecoveryControl::라우팅 제어

Amazon Route 53 Application Recovery Controller에서 라우팅 제어를 생성합니다. 라우팅 제어 상태는 매우 안정적인 클러스터 데이터 플레인에서 유지 관리됩니다.

라우팅 제어의 상태를 가져오거나 업데이트하려면 엔드포인트 URL과 AWS 리전인 클러스터 엔드포인트를 지정해야 합니다. 자세한 내용은 Amazon Route 53 Application Recovery Controller 개발자 안내서의 코드 예제를 참조하십시오.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Route53RecoveryControl::RoutingControl",
  "Properties" : {
      "ClusterArn" : String,
      "ControlPanelArn" : String,
      "Name" : String
    }
}
```

## 속성

`ClusterArn`  
라우팅 제어를 호스팅하는 클러스터의 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`ControlPanelArn`  
라우팅 제어를 포함하는 제어판의 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Name`  
라우팅 제어의 이름입니다. 이름에 공백이 아닌 문자를 사용할 수 있습니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`개체 `Ref`가 반환됩니다 `RoutingControlArn` .

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`RoutingControlArn` 라우팅 제어의 Amazon 리소스 이름(ARN)입니다.

`Status` 라우팅 제어의 배포 상태입니다. 상태는 PENDING, DEPLOYED PENDING_DELETION 중 하나일 수 있습니다.
