# AWS::EFS::액세스 포인트 생성 정보

지정된 > `RootDirectory` 없는 경우 필수 `Path` 입니다. 액세스 포인트의 >에 적용할 POSIX ID 및 권한을 지정합니다 `RootDirectory` `Path`. 액세스 포인트 루트 디렉터리가 없는 경우 EFS는 클라이언트가 액세스 포인트에 연결할 때 이러한 설정을 사용하여 루트 디렉터리를 생성합니다. 를 지정할 때는 `CreationInfo`모든 속성의 값을 포함해야 합니다.

Amazon EFS는 디렉터리에 대한 CreationInfo: OwnUid, OwnGID 및 권한을 제공한 경우에만 루트 디렉터리를 생성합니다. 이 정보를 제공하지 않으면 Amazon EFS가 루트 디렉터리를 생성하지 않습니다. 루트 디렉토리가 없으면 액세스 포인트를 사용하여 마운트하려는 시도가 실패합니다.

**중요하다**  
제공하지 않고 `CreationInfo` 지정된 항목이 `RootDirectory` 존재하지 않는 경우 액세스 포인트를 사용하여 파일 시스템을 마운트하려는 시도가 실패합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "OwnerGid" : String,
  "OwnerUid" : String,
  "Permissions" : String
}
```

## 속성

`OwnerGid`  
에 적용할 POSIX 그룹 ID를 지정합니다 `RootDirectory`. 0에서 2^32(4294967295) 사이의 값을 허용합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`OwnerUid`  
에 적용할 POSIX 사용자 ID를 지정합니다 `RootDirectory`. 0에서 2^32(4294967295) 사이의 값을 허용합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Permissions`  
에 적용할 POSIX 권한을 `RootDirectory`파일의 모드 비트를 나타내는 8진수 형식으로 지정합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `3`  
_최대_: `4`  
_패턴_: `^[0-7]{3,4}$`  
_업데이트 필요 사항_: 바꿔 놓음
