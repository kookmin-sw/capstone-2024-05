# AWS::ApiGateway::D omainName MutualTlsAuthentication

사용자 지정 도메인 이름에 대한 상호 TLS 인증 구성입니다. 지정된 경우 API Gateway는 클라이언트와 서버 간에 양방향 인증을 수행합니다. 클라이언트는 API에 액세스하기 위해 신뢰할 수 있는 인증서를 제시해야 합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "TruststoreUri" : String,
  "TruststoreVersion" : String
}
```

## 속성

`TruststoreUri`  
상호 TLS 인증을 위한 트러스트 스토어를 지정하는 Amazon S3 URL(예: `s3://bucket-name/key-name`. 신뢰 저장소에는 공용 또는 개인 인증 기관의 인증서가 포함될 수 있습니다. 트러스트 스토어를 업데이트하려면 새 버전을 S3에 업로드한 다음 새 버전을 사용하도록 사용자 지정 도메인 이름을 업데이트합니다. 신뢰 저장소를 업데이트하려면 S3 객체에 액세스할 수 있는 권한이 있어야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`TruststoreVersion`  
신뢰 저장소가 포함된 S3 객체의 버전입니다. 버전을 지정하려면 S3 버킷에 대해 버전 관리를 활성화해야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음
