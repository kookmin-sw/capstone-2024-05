# AWS::KinesisAnalyticsV2::ApplicationCloudWatchLogging옵션

애플리케이션 구성 오류를 모니터링하기 위해 Amazon CloudWatch 로그 스트림을 추가합니다.

**메모**  
 애플리케이션당 하나의 _ApplicationCloudWatchLoggingOption_ 리소스만 연결할 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::KinesisAnalyticsV2::ApplicationCloudWatchLoggingOption",
  "Properties" : {
      "ApplicationName" : String,
      "CloudWatchLoggingOption" : CloudWatchLoggingOption
    }
}
```

## 속성

`ApplicationName`  
응용 프로그램의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `128`  
_패턴_: `[a-zA-Z0-9_.-]+`  
_업데이트 필요 사항_: 바꿔 놓음

`CloudWatchLoggingOption`  
로그 스트림 Amazon 리소스 이름(ARN)을 포함하여 Amazon CloudWatch 로깅 옵션에 대한 설명을 제공합니다.  
_필수 항목 여부_: 예  
_유형_: CloudWatchLoggingOption  
_업데이트 필요 사항_: 중단 없음

## 예제

### ApplicationCloudWatchLoggingOption 리소스 만들기

#### JSON (영문)

```
{
    "BasicApplicationV2CloudWatchLoggingOption": {
        "Type": "AWS::KinesisAnalyticsV2::ApplicationCloudWatchLoggingOption",
        "Properties": {
            "ApplicationName": {
                "Ref": "BasicApplication"
            },
            "CloudWatchLoggingOption": {
                "LogStreamARN": {
                    "Fn::Join": [
                        ":",
                        [
                            "arn:aws:logs",
                            {
                                "Ref": "AWS::Region"
                            },
                            {
                                "Ref": "AWS::AccountId"
                            },
                            "log-group",
                            {
                                "Ref": "TestCWLogGroup"
                            },
                            "log-stream",
                            {
                                "Ref": "TestCWLogStream"
                            }
                        ]
                    ]
                }
            }
        }
    }
}
```
