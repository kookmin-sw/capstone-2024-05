# AWS::RedshiftServerless::작업 그룹

Amazon Redshift Serverless의 컴퓨팅 리소스 모음입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::RedshiftServerless::Workgroup",
  "Properties" : {
      "BaseCapacity" : Integer,
      "ConfigParameters" : [ ConfigParameter, ... ],
      "EnhancedVpcRouting" : Boolean,
      "NamespaceName" : String,
      "Port" : Integer,
      "PubliclyAccessible" : Boolean,
      "SecurityGroupIds" : [ String, ... ],
      "SubnetIds" : [ String, ... ],
      "Tags" : [Tag, ... ],
      "WorkgroupName" : String
    }
}
```

## 속성

`BaseCapacity`  
Redshift 처리 장치(RPU)로 단위의 작업 그룹 기본 컴퓨팅 용량입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`ConfigParameters`  
데이터베이스를 보다 세밀하게 제어하기 위해 설정할 매개 변수 목록입니다. 사용 가능한 옵션은 `datestyle`, `enable_user_activity_logging`, `query_group`, `search_path`및 `max_query_execution_time`입니다.  
*필수 항목 *여부: 아니요  
_유형_: ConfigParameter 목록  
_업데이트 필요 사항_: 중단 없음

`EnhancedVpcRouting`  
Amazon Redshift Serverless가 VPC를 통해 트래픽을 라우팅하도록 강제하는 향상된 Virtual Private Cloud(VPC) 라우팅을 활성화할지 여부를 지정하는 값입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`NamespaceName`  
작업 그룹이 연결된 네임스페이스입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Port`  
작업 그룹에 연결할 때 사용할 사용자 지정 포트입니다. 유효한 포트 범위는 5431-5455 및 8191-8215입니다. 기본값은 5439입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`PubliclyAccessible`  
공용 네트워크에서 작업 그룹에 액세스할 수 있는지 여부를 지정하는 값입니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`SecurityGroupIds`  
작업 그룹과 연결할 보안 그룹 ID 목록입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`SubnetIds`  
작업 그룹이 연결된 서브넷 ID 목록입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Tags`  
작업 그룹에 태그를 지정하는 데 사용되는 키-값 쌍의 맵입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

`WorkgroupName`  
작업 그룹의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID가 Ref 내장 함수에 제공되면 Ref는 다음과 같이 WorkgroupName을 반환합니다 `sample-workgroup.` . Ref 함수 사용에 대한 자세한 내용은 Ref를 참조하십시오.

### Fn::GetAtt

GetAtt는 이 형식의 지정된 특성에 대한 값을 반환합니다. 자세한 내용은 Fn::GetAtt 단원을 참조하십시오. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

####

`Workgroup`  
값을 반환합니다 `Workgroup` .

`Workgroup.BaseCapacity` Redshift 처리 장치(RPU) 단위의 작업 그룹 기본 데이터 웨어하우스 용량입니다.

`Workgroup.ConfigParameters` 속성 설명을 사용할 수 없습니다.

`Workgroup.CreationDate` 작업 그룹을 만든 날짜입니다.

`Workgroup.Endpoint` 속성 설명을 사용할 수 없습니다.

`Workgroup.Endpoint.Address` VPC 엔드포인트의 DNS 주소입니다.

`Workgroup.Endpoint.Port` 작업 그룹에 연결할 때 사용할 사용자 지정 포트입니다. 유효한 포트 범위는 5431-5455 및 8191-8215입니다. 기본값은 5439입니다.

`Workgroup.Endpoint.VpcEndpoints` 속성 설명을 사용할 수 없습니다.

`Workgroup.EnhancedVpcRouting` Amazon Redshift Serverless가 VPC를 통해 트래픽을 라우팅하도록 강제하는 향상된 Virtual Private Cloud(VPC) 라우팅을 활성화할지 여부를 지정하는 값입니다.

`Workgroup.NamespaceName` 작업 그룹이 연결된 네임스페이스입니다.

`Workgroup.PubliclyAccessible` 공용 네트워크에서 작업 그룹에 액세스할 수 있는지 여부를 지정하는 값입니다.

`Workgroup.SecurityGroupIds` 작업 그룹과 연결할 보안 그룹 ID의 배열입니다.

`Workgroup.Status` 작업 그룹의 상태입니다.

`Workgroup.SubnetIds` 작업 그룹이 연결된 서브넷 ID의 배열입니다.

`Workgroup.WorkgroupArn` 작업 그룹에 연결되는 Amazon 리소스 이름(ARN)입니다.

`Workgroup.WorkgroupId` 작업 그룹의 고유 식별자입니다.

`Workgroup.WorkgroupName` 작업 그룹의 이름입니다.
