# AWS::Route53RecoveryControl::SafetyRule 어설션 규칙

어설션 규칙은 라우팅 제어 상태를 변경할 때 규칙 구성에서 설정한 기준이 충족되도록 합니다. 그렇지 않으면 라우팅 제어에 대한 변경이 허용되지 않습니다. 예를 들어, 트래픽이 `On` 애플리케이션에 대해 하나 이상의 셀로 계속 흐르도록 하나 이상의 라우팅 제어 상태가 트랜잭션 이후에 있는 것이 기준일 수 있습니다. 이렇게 하면 fail-open 시나리오를 방지할 수 있습니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AssertedControls" : [ String, ... ],
  "WaitPeriodMs" : Integer
}
```

## 속성

`AssertedControls`  
라우팅 제어 상태 변경 요청이 허용되는지 여부를 결정하기 위해 평가되는 트랜잭션의 일부인 라우팅 제어입니다. 예를 들어, 3개의 AWS 리전 각각에 하나씩 3개의 라우팅 제어를 포함할 수 있습니다.  
_필수 항목 여부_: 예  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`WaitPeriodMs`  
대상 라우팅 제어에 대한 모든 요청이 실패하는 평가 기간(밀리초(ms))입니다. 이렇게 하면 상태 플래핑을 방지할 수 있습니다. 대기 기간은 기본적으로 5000ms이지만 사용자 지정 값을 선택할 수 있습니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음
