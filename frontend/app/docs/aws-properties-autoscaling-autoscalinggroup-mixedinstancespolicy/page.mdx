# AWS::AutoScaling::AutoScalingGroup MixedInstancesPolicy

이 구조를 사용하여 단일 Auto Scaling 그룹 내에서 여러 인스턴스 유형과 온디맨드 인스턴스 및 스팟 인스턴스를 시작할 수 있습니다.

혼합 인스턴스 정책에는 Amazon EC2 Auto Scaling이 인스턴스를 시작하고 비용을 최적화하는 데 사용할 수 있는 정보가 포함되어 있습니다. 자세한 내용은 Amazon EC2 Auto Scaling 사용 설명서의 여러 인스턴스 유형 및 구매 옵션이 있는 Auto Scaling 그룹을 참조하세요\*\*.

신규 및 기존 Auto Scaling 그룹에 대한 혼합 인스턴스 정책을 생성할 수 있습니다. 시작 템플릿을 사용하여 정책을 구성해야 합니다. 시작 구성은 사용할 수 없습니다.

스팟 인스턴스와 온디맨드 인스턴스 사이에는 다음과 같은 주요 차이점이 있습니다.

- 스팟 인스턴스의 가격은 수요에 따라 달라집니다
- Amazon EC2는 스팟 인스턴스의 가용성 또는 가격이 변경됨에 따라 개별 스팟 인스턴스를 종료할 수 있습니다

스팟 인스턴스가 종료되면 Amazon EC2 Auto Scaling 그룹은 그룹에 대해 원하는 용량을 유지하기 위해 대체 인스턴스를 시작하려고 시도합니다.

`MixedInstancesPolicy` 은 AWS::AutoScaling::AutoScalingGroup 리소스의 속성입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "InstancesDistribution" : InstancesDistribution,
  "LaunchTemplate" : LaunchTemplate
}
```

## 속성

`InstancesDistribution`  
인스턴스 배포입니다.  
*필수 항목 *여부: 아니요  
_형식_: InstancesDistribution  
_업데이트 필요 사항_: 중단 없음

`LaunchTemplate`  
온디맨드 및 스팟 용량을 충족하기 위해 EC2 인스턴스를 시작하는 데 사용되는 하나 이상의 시작 템플릿과 인스턴스 유형(재정의).  
_필수 항목 여부_: 예  
_형식_: LaunchTemplate  
_업데이트 필요 사항_: 중단 없음
