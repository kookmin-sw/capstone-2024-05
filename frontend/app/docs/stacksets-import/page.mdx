# AWS CloudFormation StackSets로 스택 가져오기

AWS CloudFormation _스택 가져오기_ 작업은 기존 스택을 새 스택 세트 또는 기존 스택 세트로 가져올 수 있으므로 한 번의 작업으로 기존 스택을 스택 세트로 마이그레이션할 수 있습니다. _StackSets_ 는 스택의 기능을 확장하므로 단일 작업으로 여러 계정 및 리전에서 스택을 생성, 업데이트 또는 삭제할 수 있습니다.

자체 관리형* 또는 *서비스 관리형* StackSets에 스택 가져오기 작업을 사용합니다 * . _자체 관리형_ StackSets의 경우 가져오기 작업을 통해 관리자 계정 또는 다른 대상 계정 및 AWS 리전의 스택을 가져올 수 있습니다. _서비스 관리형_ StackSets의 경우 가져오기 작업은 관리 계정과 동일한 AWS Organizations에 있는 모든 스택을 가져올 수 있습니다. 가져오기 작업은 인라인 스택 ID를 사용하여 최대 10개의 스택을 가져오거나 Amazon S3 객체를 사용하여 최대 200개의 스택을 가져올 수 있습니다.

스택 가져오기는 StackSets가 지원되는 모든 곳에서 사용할 수 있습니다. StackSets 리전 지원에 대한 자세한 내용은 StackSets 리전 지원 섹션을 참조하세요.

## 스택 가져오기에 대한 요구 사항

스택 세트는 여러 계정에 걸쳐 스택 작업을 수행하기 때문에 첫 번째 스택 세트를 생성하기 전에 AWS 계정에 정의된 필수 권한이 필요합니다.

- 자체 관리형 권한**으로 스택 세트를 생성하는 데 필요한 권한을 설정하려면 ** 기본적으로 비활성화된 리전과 관련된 스택 세트 작업 수행 및 자체 관리형 권한 부여 섹션을 참조하세요.
- 서비스 관리형 권한**으로 스택 세트를 생성하는 데 필요한 권한을 설정하려면 ** 기본적으로 비활성화된 리전과 관련된 스택 세트 작업 수행 및 AWS Organizations를 사용하여 신뢰할 수 있는 액세스 활성화 섹션을 참조하세요.

**중요하다**  
 이 `NoEcho` 속성은 StackSet 가져오기에서 지원되지 않습니다. 포함된 스택 `NoEcho` 은 StackSet 가져오기를 통해 새 스택 세트로 가져오지 않습니다.

StackSets 요구 사항에 대한 자세한 내용은 스택 세트 작업을 위한 사전 조건 섹션을 참조하세요.

스택을 스택 세트로 가져오기 전에 다음 요구 사항을 이해해야 합니다.

- 스택은 하나의 스택 세트에만 속할 수 있습니다.
- 스택 가져오기 작업에서 태그를 파라미터로 명시적으로 지정하여 스택 세트에 스택 태그를 구현할 수 있습니다.
- 스택의 사용자 지정 파라미터 재정의는 가져오기 작업 중에 영향을 받지 않습니다.
- StackSets 할당량 및 스택 인스턴스는 스택을 가져올 때 적용됩니다. 할당량에 대한 자세한 내용은 AWS CloudFormation 할당량 섹션을 참조하세요.
