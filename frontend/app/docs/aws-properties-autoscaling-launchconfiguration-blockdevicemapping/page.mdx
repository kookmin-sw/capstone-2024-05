# AWS::AutoScaling::LaunchConfiguration BlockDeviceMapping

`BlockDeviceMapping` AWS::AutoScaling::LaunchConfiguration 리소스의 속성`BlockDeviceMappings`에 대한 블록 디바이스 매핑을 지정합니다.

시작된 각 인스턴스에는 연결된 루트 디바이스 볼륨(Amazon EBS 볼륨 또는 인스턴스 스토어 볼륨)이 있습니다. 블록 디바이스 매핑을 사용하여 시작 시 인스턴스에 연결할 추가 EBS 볼륨 또는 인스턴스 스토어 볼륨을 지정할 수 있습니다.

자세한 내용은 Linux 인스턴스용 Amazon EC2 사용 설명서*의 블록 디바이스 매핑 예제를 참조하세요*.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "DeviceName" : String,
  "Ebs" : BlockDevice,
  "NoDevice" : Boolean,
  "VirtualName" : String
}
```

## 속성

`DeviceName`  
볼륨에 할당된 장치 이름(예: `/dev/sdh` 또는 `xvdh`). 자세한 내용은 *Linux 인스턴스용 Amazon EC2 사용 설명서*의 Linux 인스턴스에서 디바이스 이름 지정을 참조하세요.  
블록 장치 매핑을 정의하려면 장치 이름과 정확히 다음 속성 중 하나를 설정 `Ebs` `NoDevice` 하십시오`VirtualName`.
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Ebs`  
시작 시 EBS 볼륨을 인스턴스에 연결하기 위한 정보입니다.  
*필수 항목 *여부: 아니요  
_형식_: BlockDevice  
_업데이트 필요 사항_: 바꿔 놓음

`NoDevice`  
이 값을 설정하면 `true` AMI의 블록 디바이스 매핑에 포함된 볼륨이 시작 시 지정된 디바이스 이름에 매핑되지 않습니다.  
`NoDevice` `true` 루트 디바이스용인 경우 인스턴스가 EC2 상태 확인에 실패할 수 있습니다. 이 경우 Amazon EC2 Auto Scaling은 대체 인스턴스를 시작합니다.  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 바꿔 놓음

`VirtualName`  
시작 시 인스턴스에 연결할 인스턴스 스토어 볼륨(가상 디바이스)의 이름입니다. 이름은 임시 X 형식이어야 하며\**, 여기서 *X*는 영(0)부터 시작하는 숫자입니다(예: `ephemeral0`).  
*필수 항목 *여부: 아니요  
*유형*: 문자열  
*업데이트 필요 사항\*: 바꿔 놓음
