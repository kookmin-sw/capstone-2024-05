# AWS::ECS::서비스 배치 제약 조건

이 `PlacementConstraint` 속성은 작업 정의에서 작업 배치에 대한 제약 조건을 나타내는 개체를 지정합니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 작업 배치 제약 조건을 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Expression" : String,
  "Type" : String
}
```

## 속성

`Expression`  
제약 조건에 적용할 클러스터 쿼리 언어 식입니다. 표현식의 최대 길이는 2000자입니다. 제약 조건 유형이 인 경우 표현식을 지정할 수 없습니다`distinctInstance`. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 클러스터 쿼리 언어를 참조하세요_.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 중단 없음

`Type`  
제약 조건의 유형입니다. `distinctInstance` 특정 그룹의 각 작업이 다른 컨테이너 인스턴스에서 실행되고 있는지 확인하는 데 사용합니다. `memberOf` 선택을 유효한 후보 그룹으로 제한하는 데 사용합니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_허용되는 값_: `distinctInstance | memberOf`  
_업데이트 필요 사항_: 중단 없음
