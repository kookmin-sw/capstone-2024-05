# AWS::CloudFront::D istribution ForwardedValues

이 필드는 더 이상 사용되지 않습니다. 이 필드 대신 캐시 정책 또는 오리진 요청 정책을 사용하는 것이 좋습니다.

캐시 키에 값을 포함하려면 캐시 정책을 사용합니다. 자세한 내용은 _Amazon CloudFront 개발자 안내서의 캐시 정책 생성을 참조하세요_.

오리진에 값을 전송하되 캐시 키에 포함하지 않으려면 오리진 요청 정책을 사용합니다. 자세한 내용은 _Amazon CloudFront 개발자 안내서의 오리진 요청 정책 생성을 참조하세요_.

CloudFront에서 쿼리 문자열, 쿠키 및 HTTP 헤더를 처리하는 방법을 지정하는 복합 형식입니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Cookies" : Cookies,
  "Headers" : [ String, ... ],
  "QueryString" : Boolean,
  "QueryStringCacheKeys" : [ String, ... ]
}
```

## 속성

`Cookies`

캐시 키에 쿠키를 포함하려면 캐시 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 캐시 정책 생성을 참조하세요.  
오리진에 쿠키를 전송하되 캐시 키에 포함하지 않으려면 오리진 요청 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 오리진 요청 정책 생성을 참조하세요.  
CloudFront에서 오리진에 쿠키를 전달할지 여부와 전달하려는 경우 어떤 쿠키를 전달할지를 지정하는 복합 유형입니다. 오리진에 쿠키를 전달하는 방법에 대한 자세한 내용은 Amazon CloudFront 개발자 안내서의 CloudFront 쿠키 전달, 캐시 및 로그 방법을 참조하십시오\**.

*유형*: 쿠키  
*업데이트 필요 사항\*: 중단 없음

`Headers`  
이 필드는 더 이상 사용되지 않습니다. 이 필드 대신 캐시 정책 또는 오리진 요청 정책을 사용하는 것이 좋습니다.  
캐시 키에 헤더를 포함하려면 캐시 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 캐시 정책 생성을 참조하세요.  
오리진에 헤더를 전송하되 캐시 키에 포함하지 않으려면 오리진 요청 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 오리진 요청 정책 생성을 참조하세요.  
`Headers`CloudFront에서 이 캐시 동작(화이트리스트 헤더)에 대해 오리진에 전달할 (있는 경우)를 지정하는 복합 유형입니다. 또한 지정한 헤더의 경우, CloudFront는 최종 사용자 요청의 헤더 값을 기반으로 지정된 객체의 개별 버전을 캐싱합니다.  
자세한 내용은 Amazon CloudFront 개발자 안내서의 요청 헤더에 따라 콘텐츠 캐싱을 참조하십시오\**.  
*필수 항목 *여부: 아니요  
*유형*: 문자열 목록  
*업데이트 필요 사항\*: 중단 없음

`QueryString`  
이 필드는 더 이상 사용되지 않습니다. 이 필드 대신 캐시 정책 또는 오리진 요청 정책을 사용하는 것이 좋습니다.  
캐시 키에 쿼리 문자열을 포함하려면 캐시 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 캐시 정책 생성을 참조하세요.  
오리진에 쿼리 문자열을 전송하되 캐시 키에 포함하지 않으려면 오리진 요청 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 오리진 요청 정책 생성을 참조하세요.  
CloudFront에서 쿼리 문자열 파라미터를 기반으로 이 캐시 동작 및 캐시와 연결된 오리진에 쿼리 문자열을 전달할지 여부를 나타냅니다. CloudFront 동작은 의 값 `QueryString` 과 에 대해 지정한 값`QueryStringCacheKeys`(있는 경우)에 따라 달라집니다.  
true를 지정 `QueryString` 하고 에 대한 값을 지정하지 않는 `QueryStringCacheKeys` 경우, CloudFront에서는 모든 쿼리 문자열 파라미터를 오리진에 전달하고 모든 쿼리 문자열 파라미터를 기반으로 캐싱합니다. 보유한 쿼리 문자열 파라미터 및 값의 수에 따라 CloudFront가 오리진에 더 많은 요청을 전달해야 하기 때문에 성능이 저하될 수 있습니다.  
true for `QueryString` 를 지정하고 에 대해 하나 이상의 값을 지정하는 `QueryStringCacheKeys` 경우, CloudFront에서는 모든 쿼리 문자열 파라미터를 오리진에 전달하지만, 사용자가 지정한 쿼리 문자열 파라미터에 따라서만 캐싱합니다.  
에 false를 지정할 경우`QueryString`, CloudFront는 쿼리 문자열 파라미터를 오리진에 전달하지 않으며, 쿼리 문자열 파라미터를 기반으로 캐싱하지 않습니다.  
자세한 내용은 Amazon CloudFront 개발자 안내서의 쿼리 문자열 파라미터를 기반으로 캐싱하도록 CloudFront 구성을 참조하십시오\**.  
*필수 항목 여부*: 예  
*유형*: Boolean  
*업데이트 필요 사항\*: 중단 없음

`QueryStringCacheKeys`

캐시 키에 쿼리 문자열을 포함하려면 캐시 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 캐시 정책 생성을 참조하세요.  
오리진에 쿼리 문자열을 전송하되 캐시 키에 포함하지 않으려면 오리진 요청 정책을 사용합니다. 자세한 내용은 *Amazon CloudFront 개발자 안내서*의 오리진 요청 정책 생성을 참조하세요.  
CloudFront에서 이 캐시 동작에 대한 캐싱에 사용할 쿼리 문자열 파라미터에 대한 정보를 포함하는 복합 유형입니다.
