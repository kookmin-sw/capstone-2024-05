# AWS::EC2::CapacityReservationFleet

지정된 속성을 사용하여 새 용량 예약 플릿을 생성합니다. 자세한 내용은 _Amazon EC2 사용 설명서의 용량 예약 플릿을 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::EC2::CapacityReservationFleet",
  "Properties" : {
      "AllocationStrategy" : String,
      "EndDate" : String,
      "InstanceMatchCriteria" : String,
      "InstanceTypeSpecifications" : [ InstanceTypeSpecification, ... ],
      "NoRemoveEndDate" : Boolean,
      "RemoveEndDate" : Boolean,
      "TagSpecifications" : [ TagSpecification, ... ],
      "Tenancy" : String,
      "TotalTargetCapacity" : Integer
    }
}
```

## 속성

`AllocationStrategy`  
용량 예약 집합이 지정된 인스턴스 유형 중 사용할 인스턴스를 결정하는 데 사용하는 전략입니다. 현재는 `prioritized` 할당 전략만 지원됩니다. 자세한 내용은 Amazon EC2 사용 설명서의 할당 전략을 참조하세요.  
유효한 값: `prioritized`  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`EndDate`  
용량 예약 집합이 만료되는 날짜 및 시간입니다. 용량 예약 플릿이 만료되면 상태가 로 변경 `expired` 되고 플릿의 모든 용량 예약이 만료됩니다.  
용량 예약 집합은 지정된 시간 이후 한 시간 이내에 만료됩니다. 예를 들어, 를 지정하면 `5/31/2019` `13:30:55`용량 예약 집합은 와 다음 사이에 만료되도록 보장됩니다 `13:30:55` `14:30:55` `5/31/2019`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`InstanceMatchCriteria`  
용량 예약 집합이 수락하는 인스턴스 시작 유형을 나타냅니다. 플릿의 모든 용량 예약은 이 인스턴스 일치 기준을 상속합니다.  
현재 용량 예약 플릿은 `open` 기준과 일치하는 인스턴스만 지원합니다. 즉, 일치하는 속성(인스턴스 유형, 플랫폼 및 가용 영역)이 있는 인스턴스는 용량 예약에서 자동으로 실행됩니다. 인스턴스는 예약 용량을 사용하기 위해 용량 예약 플릿을 명시적으로 대상으로 지정할 필요가 없습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `open`  
_업데이트 필요 사항_: 바꿔 놓음

`InstanceTypeSpecifications`  
용량을 예약할 인스턴스 유형에 대한 정보입니다.  
*필수 항목 *여부: 아니요  
_Type_: InstanceTypeSpecification 목록  
_업데이트 필요 사항_: 바꿔 놓음

`NoRemoveEndDate`  
종료 날짜 및 시간이 없는 용량 예약 집합에 종료 날짜를 추가하는 데 사용됩니다. 용량 예약 플릿에 종료 날짜를 추가하려면 `true` 이 파라미터에 대해 지정하고 EndDate 파라미터에 대한 종료 날짜 및 시간(UTC 시간 형식)을 지정합니다 \**\*\* .  
*필수 항목 *여부: 아니요  
*유형*: Boolean  
*업데이트 필요 사항\*: 중단 없음

`RemoveEndDate`  
특정 날짜 및 시간에 자동으로 종료되도록 구성된 용량 예약 집합에서 종료 날짜를 제거하는 데 사용됩니다. 용량 예약 집합에서 종료 날짜를 제거하려면 `true` 이 파라미터에 대해 지정하고 **EndDate** 파라미터를 생략합니다 .  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`TagSpecifications`  
용량 예약 플릿에 할당할 태그입니다. 태그는 플릿의 용량 예약에 자동으로 할당됩니다.  
*필수 항목 *여부: 아니요  
_유형_: TagSpecification 목록  
_업데이트 필요 사항_: 바꿔 놓음

`Tenancy` 용량 예약 플릿의 테넌시를 나타냅니다. 플릿의 모든 용량 예약은 이 테넌시를 상속합니다. 용량 예약 플릿은 다음 테넌시 설정 중 하나를 가질 수 있습니다.

- `default` - 용량 예약 플릿은 다른 AWS 계정과 공유되는 하드웨어에서 생성됩니다.
- `dedicated` - 용량 예약은 단일 AWS 계정 전용의 단일 테넌트 하드웨어에서 생성됩니다.
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _허용되는 값_: `default`  
  _업데이트 필요 사항_: 바꿔 놓음

`TotalTargetCapacity`  
용량 예약 플릿에서 예약할 총 용량 단위 수입니다. 이 값은 플릿에서 사용하는 각 인스턴스 유형에 할당하는 인스턴스 유형 가중치와 함께 플릿이 용량을 예약하는 인스턴스 수를 결정합니다. 두 값 모두 워크로드에 적합한 단위를 기반으로 합니다. 자세한 내용은 Amazon EC2 사용 설명서의 총 목표 용량을 참조하세요.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`는 `Ref`와 같은 리소스 ID를 반환합니다 `crf-abcdef01234567890`.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`CapacityReservationFleetId` 용량 예약 플릿의 ID입니다.
