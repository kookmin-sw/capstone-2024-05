# AWS::Route53::CidrCollection

현재 AWS 계정에 CIDR 컬렉션을 생성합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Route53::CidrCollection",
  "Properties" : {
      "Locations" : [ Location, ... ],
      "Name" : String
    }
}
```

## 속성

`Locations`  
CIDR 위치 목록에 대한 정보를 포함하는 복합 형식입니다.  
*필수 항목 *여부: 아니요  
_유형_: 위치 목록  
_업데이트 필요 사항_: 중단 없음

`Name`  
CIDR 컬렉션의 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `64`  
_패턴_: `[0-9A-Za-z_-]+`  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`ID `Ref`를 반환합니다 `CidrCollection` .

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Arn` "AWS 리소스를 고유하게 식별하기 위한 Amazon 리소스 이름(ARN)입니다.

`Id` CIDR 컬렉션의 UUID입니다.

## 예제

### CIDR 컬렉션 생성

다음 예제에서는 각각 두 개의 CIDR 블록을 포함하는 두 개의 위치로 CIDR 컬렉션을 생성합니다.

#### JSON (영문)

```
{
  "MyCidrCollection":{
    "Type":"AWS::Route53::CidrCollection",
    "Properties":{
      "Name":"my-first-cidr-collection",
      "Locations":[
      {
        "LocationName":"location-1",
        "CidrList":[
          "1.1.0.0/24",
          "2.1.0.0/16"
        ]
      },
      {
        "LocationName":"location-2",
        "CidrList":[
          "2002::1234:0:0:0:0:0/48",
          "1002::/32"
        ]
      }
    ]
   }
  }
}
```
