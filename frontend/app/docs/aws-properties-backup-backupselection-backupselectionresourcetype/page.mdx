# AWS::백업::백업선택 BackupSelectionResourceType

자원 집합을 백업 계획에 할당하는 데 사용되는 속성이 포함된 개체를 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Conditions" : Conditions,
  "IamRoleArn" : String,
  "ListOfTags" : [ ConditionResourceType, ... ],
  "NotResources" : [ String, ... ],
  "Resources" : [ String, ... ],
  "SelectionName" : String
}
```

## 속성

`Conditions`  
태그를 사용하여 백업 계획에 리소스를 할당하기 위해 정의하는 조건 목록입니다. 예를 들어, `"StringEquals": { "ConditionKey": "aws:ResourceTag/CreatedByCryo", "ConditionValue": "true" },`. 조건 연산자는 대소문자를 구분합니다.  
`Conditions` 차이점은 `ListOfTags` 다음과 같습니다.

- 둘 이상의 조건을 지정하는 경우 ALL 조건과 일치하는 리소스만 할당합니다(AND 논리 사용).
- `Conditions`, `StringEquals`, `StringLike`, 및 `StringNotEquals` 를 지원합니다`StringNotLike`. `ListOfTags` 만 지원합니다`StringEquals`.
  *필수 항목 *여부: 아니요  
  _유형_: 조건  
  _업데이트 필요 사항_: 바꿔 놓음

`IamRoleArn`  
AWS Backup이 대상 리소스를 백업할 때 인증하는 데 사용하는 IAM 역할의 ARN입니다. 예를 들어, `arn:aws:iam::123456789012:role/S3Access`.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`ListOfTags`  
태그를 사용하여 백업 계획에 리소스를 할당하기 위해 정의하는 조건 목록입니다. 예를 들어, `"StringEquals": { "ConditionKey": "aws:ResourceTag/CreatedByCryo", "ConditionValue": "true" },`. 조건 연산자는 대소문자를 구분합니다.  
`ListOfTags` 차이점은 `Conditions` 다음과 같습니다.

- 둘 이상의 조건을 지정하는 경우 하나 이상의 조건과 일치하는 모든 리소스를 할당합니다(OR 로직 사용).
- `ListOfTags` 만 지원합니다`StringEquals`. `Conditions`, `StringEquals`, `StringLike`, 및 `StringNotEquals` 를 지원합니다`StringNotLike`.
  *필수 항목 *여부: 아니요  
  _유형_: ConditionResourceType 목록  
  _업데이트 필요 사항_: 바꿔 놓음

`NotResources`  
백업 계획에서 제외할 Amazon 리소스 이름(ARN) 목록입니다. 최대 ARN 수는 와일드카드가 없는 경우 500개, 와일드카드가 있는 ARN은 30개입니다.  
백업 계획에서 많은 리소스를 제외해야 하는 경우 하나 또는 몇 개의 리소스 유형만 할당하거나 태그를 사용하여 리소스 선택을 구체화하는 것과 같은 다른 리소스 선택 전략을 고려합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 바꿔 놓음

`Resources`  
백업 계획에 할당할 리소스의 Amazon 리소스 이름(ARN)을 포함하는 문자열 배열입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 바꿔 놓음

`SelectionName`  
자원 선택 문서의 표시 이름입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음
