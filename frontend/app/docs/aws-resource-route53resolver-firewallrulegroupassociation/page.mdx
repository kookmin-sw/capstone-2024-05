# AWS::Route53Resolver::FirewallRuleGroup연결

방화벽 규칙 그룹과 VPC 간의 연결로, VPC에 대한 DNS 필터링을 활성화합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::Route53Resolver::FirewallRuleGroupAssociation",
  "Properties" : {
      "FirewallRuleGroupId" : String,
      "MutationProtection" : String,
      "Name" : String,
      "Priority" : Integer,
      "Tags" : [Tag, ... ],
      "VpcId" : String
    }
}
```

## 속성

`FirewallRuleGroupId`  
방화벽 규칙 그룹의 고유 식별자입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `64`  
_업데이트 필요 사항_: 바꿔 놓음

`MutationProtection`  
이 설정을 사용하도록 설정하면 DNS 방화벽 보호가 실수로 변경되는 것을 방지하기 위해 연결을 수정하거나 제거할 수 없습니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_허용되는 값_: `DISABLED | ENABLED`  
_업데이트 필요 사항_: 중단 없음

`Name`  
연결의 이름입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_최대_: `64`  
_패턴_: `(?!^[0-9]+$)([a-zA-Z0-9-_' ']+)`  
_업데이트 필요 사항_: 중단 없음

`Priority`  
단일 VPC와 연결된 규칙 그룹 중에서 규칙 그룹의 처리 순서를 결정하는 설정입니다. DNS 방화벽은 숫자 우선 순위가 가장 낮은 규칙 그룹에서 시작하는 VPC 트래픽을 필터링합니다.  
단일 VPC와 연결하는 각 규칙 그룹에 대해 고유한 우선 순위를 지정해야 합니다. 나중에 규칙 그룹을 더 쉽게 삽입하려면 숫자 사이에 공백을 둡니다(예: 101, 200 등). 규칙 그룹 연결을 생성한 후 해당 우선 순위 설정을 변경할 수 있습니다.  
허용되는 값은 `Priority` 100에서 9900 사이(100에서 9900 제외)입니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`Tags`  
규칙 그룹과 연결하려는 태그 키 및 값의 목록입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_최대_: `200`  
_업데이트 필요 사항_: 중단 없음

`VpcId`  
규칙 그룹과 연결된 VPC의 고유 식별자입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_최소_: `1`  
_최대_: `64`  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`ID `Ref`를 반환합니다 `FirewallRuleGroupAssociation` .

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Arn` 방화벽 규칙 그룹 연결의 Amazon 리소스 이름(ARN)입니다.

`CreationTime` 연결이 생성된 날짜 및 시간(Unix 시간 형식 및 UTC(협정 세계시))입니다.

`CreatorRequestId` 요청을 식별하기 위해 사용자가 정의한 고유한 문자열입니다. 이렇게 하면 작업을 두 번 실행할 위험 없이 실패한 요청을 다시 시도할 수 있습니다. 고유한 문자열(예: 타임스탬프)이 될 수 있습니다.

`Id` 연결의 식별자입니다.

`ManagedOwnerName` 사용자가 관리하지 않는 연결에만 사용되는 연결의 소유자입니다. AWS Firewall Manager를 사용하여 DNS 방화벽에서 방화벽을 관리하는 경우 Firewall Manager가 관리형 소유자로 보고됩니다.

`ModificationTime` 연결이 마지막으로 수정된 날짜 및 시간(Unix 시간 형식 및 UTC(협정 세계시))입니다.

`Status` 연결의 현재 상태입니다.

`StatusMessage` 응답 상태에 대한 추가 정보(사용 가능한 경우)입니다.
