# AWS::VpcLattice::리스너

서비스에 대한 수신기를 만듭니다. Amazon VPC Lattice 서비스 사용을 시작하기 전에 하나 이상의 리스너를 추가해야 합니다. 수신기는 서비스에 대한 연결 요청을 확인하는 프로세스입니다. 자세한 내용은 _Amazon VPC Lattice 사용 설명서의 리스너를 참조하세요_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::VpcLattice::Listener",
  "Properties" : {
      "DefaultAction" : DefaultAction,
      "Name" : String,
      "Port" : Integer,
      "Protocol" : String,
      "ServiceIdentifier" : String,
      "Tags" : [Tag, ... ]
    }
}
```

## 속성

`DefaultAction`  
기본 규칙에 대한 작업입니다. 각 수신기에는 기본 규칙이 있습니다. 일치하는 다른 규칙이 없는 경우 기본 규칙이 사용됩니다.  
_필수 항목 여부_: 예  
_유형_: DefaultAction  
_업데이트 필요 사항_: 중단 없음

`Name`  
수신기의 이름입니다. 수신기 이름은 서비스 내에서 고유해야 합니다. 유효한 문자는 a-z, 0-9 및 하이픈(-)입니다. 하이픈은 첫 번째 또는 마지막 문자로 사용하거나 다른 하이픈 바로 뒤에 사용할 수 없습니다.  
이름을 지정하지 않으면 CloudFormation에서 이름을 생성합니다. 그러나 이름을 지정하고 나중에 자원을 바꾸려면 새 이름을 지정해야 합니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Port`  
수신기 포트입니다. 1에서 65535 사이의 값을 지정할 수 있습니다. HTTP의 경우 기본값은 80입니다. HTTPS의 경우 기본값은 443입니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 바꿔 놓음

`Protocol`  
수신기 프로토콜입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`ServiceIdentifier`  
서비스의 ID 또는 Amazon 리소스 이름(ARN)입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

`Tags`  
수신기에 대한 태그입니다.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref`리스 `Ref`너의 Amazon 리소스 이름(ARN)을 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`Arn` 리스너의 Amazon 리소스 이름(ARN)입니다.

`Id` 수신기의 ID입니다.

`ServiceArn` 서비스의 Amazon 리소스 이름(ARN)입니다.

`ServiceId` 서비스의 ID입니다.
