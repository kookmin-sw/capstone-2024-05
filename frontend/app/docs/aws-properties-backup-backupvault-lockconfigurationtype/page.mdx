# AWS::백업::BackupVault LockConfigurationType

`LockConfigurationType` 속성 유형은 AWS Backup 저장소 잠금에 대한 구성을 지정합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "ChangeableForDays" : Integer,
  "MaxRetentionDays" : Integer,
  "MinRetentionDays" : Integer
}
```

## 속성

`ChangeableForDays`  
잠금 날짜까지 남은 일 수를 지정하는 AWS Backup 저장소 잠금 구성입니다. 예를 들어 `ChangeableForDays` 2022년 1월 1일 오후 8시(UTC)에 30으로 설정하면 잠금 날짜가 2022년 1월 31일 오후 8시(UTC)로 설정됩니다.  
AWS Backup은 볼트 잠금이 적용되어 변경할 수 없게 되기 전에 72시간의 냉각 기간을 적용합니다. 따라서 `ChangeableForDays` 3 이상으로 설정해야 합니다.  
잠금 날짜 이전에는 를 사용하여 볼트에서 볼트 잠금을 삭제 `DeleteBackupVaultLockConfiguration` 하거나 를 사용하여 볼트 잠금 구성을 변경할 `PutBackupVaultLockConfiguration` 수 있습니다. 잠금 날짜 이후에는 볼트 잠금을 변경할 수 없게 되며 변경하거나 삭제할 수 없습니다.  
이 매개변수를 지정하지 않으면 언제든지 를 사용하여 볼트에서 볼트 잠금을 삭제 `DeleteBackupVaultLockConfiguration` 하거나 를 사용하여 볼트 잠금 구성을 변경할 수 있습니다 `PutBackupVaultLockConfiguration` .  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`MaxRetentionDays`  
저장소가 복구 지점을 유지하는 최대 보존 기간을 지정하는 AWS Backup 저장소 잠금 구성입니다. 이 설정은 예를 들어 조직의 정책에 따라 특정 데이터를 4년(1460일) 동안 보존한 후 삭제해야 하는 경우에 유용할 수 있습니다.  
이 매개 변수가 포함되지 않은 경우 자격 증명 모음 잠금은 자격 증명 모음의 복구 지점에 최대 보존 기간을 적용하지 않습니다. 이 매개 변수를 값 없이 포함하면 볼트 잠금이 최대 보존 기간을 적용하지 않습니다.  
이 파라미터를 지정하면 볼트에 대한 모든 백업 또는 복사 작업에는 보존 기간이 최대 보존 기간보다 짧거나 같은 수명 주기 정책이 있어야 합니다. 작업의 보존 기간이 최대 보존 기간보다 길면 자격 증명 모음이 백업 또는 복사 작업에 실패하므로 수명 주기 설정을 수정하거나 다른 자격 증명 모음을 사용해야 합니다. 볼트 잠금 이전에 볼트에 이미 저장된 복구 지점은 영향을 받지 않습니다.  
*필수 항목 *여부: 아니요  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`MinRetentionDays`  
볼트가 복구 지점을 보존하는 최소 보존 기간을 지정하는 AWS Backup 볼트 잠금 구성입니다. 이 설정은 예를 들어 조직의 정책에 따라 특정 데이터를 최소 7년(2555일) 동안 보존해야 하는 경우에 유용할 수 있습니다.  
이 매개 변수를 지정하지 않으면 Vault Lock에서 최소 보존 기간을 적용하지 않습니다.  
이 파라미터를 지정하면 자격 증명 모음에 대한 모든 백업 또는 복사 작업에는 보존 기간이 최소 보존 기간보다 길거나 같은 수명 주기 정책이 있어야 합니다. 작업의 보존 기간이 최소 보존 기간보다 짧으면 볼트가 해당 백업 또는 복사 작업에 실패하므로 수명 주기 설정을 수정하거나 다른 볼트를 사용해야 합니다. 볼트 잠금 이전에 볼트에 이미 저장된 복구 지점은 영향을 받지 않습니다.  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음
