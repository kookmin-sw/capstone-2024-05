# AWS::Cognito::UserPool 디바이스 구성

사용자 풀에 대한 디바이스 기억 구성입니다. DescribeUserPool 요청은 사용자 풀이 디바이스를 기억하도록 구성되지 않은 경우 이 객체에 대해 null 값을 반환합니다. 디바이스 기억이 활성화되면 ConfirmDevice API 요청으로 사용자의 디바이스를 기억할 수 있습니다. 또한. 속성이 인 경우 `DeviceOnlyRememberedOnUserPrompt` 사용자의 디바이스를 `true` 또는 `ConfirmDevice`로`remembered` 설정하는 UpdateDeviceStatus API 요청을 `not_remembered`따라야 합니다.

기억된 디바이스로 로그인하려면 `DEVICE_KEY` 사용자의 InitiateAuth 요청에 인증 파라미터를 포함합니다. 앱에 파라미터가 포함되어 있지 않은 경우 `DEVICE_KEY` Amazon Cognito의 응답에는 새로 생성된 `DEVICE_KEY` 값과 `DEVICE_GROUP_KEY` . `NewDeviceMetadata` 향후 디바이스 인증 요청에서 사용할 수 있도록 이러한 값을 저장합니다.

**메모**  
의 속성에 대한 값을 제공 `DeviceConfiguration`하면 사용자 풀을 기억하는 디바이스가 활성화됩니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "ChallengeRequiredOnNewDevice" : Boolean,
  "DeviceOnlyRememberedOnUserPrompt" : Boolean
}
```

## 속성

`ChallengeRequiredOnNewDevice`  
true인 경우 기억된 디바이스는 SMS 및 다단계 인증(MFA)을 위한 시간 기반 일회용 암호(TOTP) 요소 대신 디바이스 인증을 사용하여 로그인할 수 있습니다.  
사실이든 아니든 `ChallengeRequiredOnNewDevice` , 확인되지 않았거나 기억되지 않은 디바이스로 로그인하는 사용자는 여전히 MFA가 필요한 사용자 풀에 두 번째 팩터를 제공해야 합니다.
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`DeviceOnlyRememberedOnUserPrompt`  
true인 경우 Amazon Cognito는 앱이 ConfirmDevice API 요청을 보낼 때 사용자의 디바이스를 자동으로 기억하지 않습니다. 앱에서 사용자가 기기를 기억할지 여부를 선택할 수 있는 프롬프트를 만듭니다. UpdateDeviceStatus API 요청에서 사용자의 선택 사항을 반환합니다.  
인 경우 `DeviceOnlyRememberedOnUserPrompt` `false`Amazon Cognito는 API 요청에 등록한 디바이스를 즉시 기억 합니다 `ConfirmDevice` .  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음
