# AWS::VpcLattice::인증 정책

인증 정책을 생성하거나 업데이트합니다. JSON의 정책 문자열에는 줄 바꿈이나 빈 줄이 포함되어서는 안 됩니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Type" : "AWS::VpcLattice::AuthPolicy",
  "Properties" : {
      "Policy" : Json,
      "ResourceIdentifier" : String
    }
}
```

## 속성

`Policy`  
인증 정책입니다.  
_필수 항목 여부_: 예  
_형식_: Json  
_업데이트 필요 사항_: 중단 없음

`ResourceIdentifier`  
정책이 생성된 서비스 네트워크 또는 서비스의 ID 또는 Amazon 리소스 이름(ARN)입니다.  
_필수 항목 여부_: 예  
_유형_: 문자열  
_업데이트 필요 사항_: 바꿔 놓음

## 반환 값

### 참고문헌

이 리소스의 논리적 ID를 내장 함수에 전달하면 `Ref` `Ref`인증 정책의 Amazon 리소스 이름(ARN)을 반환합니다.

함수 사용에 대한 자세한 `Ref`내용은 Ref를 참조하세요.

### Fn::GetAtt

`Fn::GetAtt`내장 함수는 이 형식의 지정된 특성에 대한 값을 반환합니다. 다음은 사용 가능한 특성 및 샘플 반환 값입니다.

내장 함수 사용에 대한 자세한 `Fn::GetAtt`내용은 Fn::GetAtt 단원을 참조하십시오.

####

`State`  
인증 정책의 상태입니다. 인증 정책은 인증 유형이 로 설정된 경우에만 활성화됩니다 `AWS_IAM`. 정책을 제공하면 이 정책과 클라이언트의 IAM 정책에 따라 인증 및 권한 부여 결정이 내려집니다. 인증 유형이 인 경우 `NONE`사용자가 제공하는 모든 인증 정책은 비활성 상태로 유지됩니다.
