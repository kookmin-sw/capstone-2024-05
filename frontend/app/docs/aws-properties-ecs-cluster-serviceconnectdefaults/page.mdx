# AWS::ECS::클러스터 ServiceConnect기본값

이 매개 변수를 사용하여 기본 Service Connect 네임스페이스를 설정합니다. 기본 Service Connect 네임스페이스를 설정한 후 클러스터에서 만든 Service Connect가 켜져 있는 모든 새 서비스는 네임스페이스에 클라이언트 서비스로 추가됩니다. 이 설정은 `enabled` 매개 변수를 `true` `ServiceConnectConfiguration`. 에서 각 서비스의 네임스페이스를 개별적으로 설정하여 `ServiceConnectConfiguration` 이 기본 매개 변수를 재정의할 수 있습니다.

네임스페이스에서 실행되는 작업은 짧은 이름을 사용하여 네임스페이스의 서비스에 연결할 수 있습니다. 태스크는 네임스페이스의 모든 클러스터에서 서비스에 연결할 수 있습니다. 작업은 가시성을 높이기 위해 로그와 메트릭을 수집하는 관리형 프록시 컨테이너를 통해 연결됩니다. Amazon ECS 서비스가 생성하는 작업만 Service Connect에서 지원됩니다. 자세한 내용은 _Amazon Elastic Container Service 개발자 안내서의 Service Connect를 참조하십시오_.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Namespace" : String
}
```

## 속성

`Namespace`  
서비스를 생성하고 Service Connect 구성을 지정하지 않을 때 사용되는 AWS Cloud Map 네임스페이스의 네임스페이스 이름 또는 전체 Amazon 리소스 이름(ARN)입니다. 네임스페이스 이름은 최대 1024자를 포함할 수 있습니다. 이름은 대소문자를 구분합니다. 이름에는 하이픈(-), 물결표(~), 보다 큼(>), 보다 작음(&lt;) 또는 슬래시(/)를 포함할 수 없습니다.  
기존 네임스페이스 이름 또는 ARN을 입력하면 해당 네임스페이스가 사용됩니다. 모든 네임스페이스 형식이 지원됩니다. 네임스페이스는 이 계정과 이 AWS 리전에 있어야 합니다.  
새 이름을 입력하면 AWS Cloud Map 네임스페이스가 생성됩니다. Amazon ECS는 인스턴스 검색의 "API 호출" 메서드만 사용하여 AWS Cloud Map 네임스페이스를 생성합니다. 이 인스턴스 검색 방법은 AWS Command Line Interface의 "HTTP" 네임스페이스 유형입니다. 다른 유형의 인스턴스 검색은 Service Connect에서 사용되지 않습니다.  
 네임스페이스 이름에 빈 문자열로 클러스터를 업데이트하면 `""` Service Connect에 대한 클러스터 구성이 제거됩니다. 네임스페이스는 AWS Cloud Map에 남아 있으며 별도로 삭제해야 합니다.  
AWS Cloud Map에 대한 자세한 내용은 * AWS Cloud Map 개발자 안내서*의 서비스 작업을 참조하십시오.
