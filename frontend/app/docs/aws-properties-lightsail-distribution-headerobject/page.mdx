# AWS::Lightsail::D istribution HeaderObject

`HeaderObject` 은 CacheSettings 속성의 속성입니다. 배포에서 사용하는 요청 헤더에 대해 설명하며, 요청 헤더를 기반으로 콘텐츠를 캐싱합니다.

지정한 헤더의 경우, 배포는 최종 사용자 요청의 헤더 값에 따라 지정된 콘텐츠의 개별 버전을 캐시합니다. 예를 들어, logo.jpg에 대한 최종 사용자 요청에 acme 또는 apex 값을 가진 사용자 지정 제품 헤더가 포함되어 있다고 가정해 보겠습니다. 또한 product 헤더의 값을 기반으로 콘텐츠를 캐시하도록 배포를 구성한다고 가정해 보겠습니다. 배포는 제품 헤더를 오리진에 전달하고 각 헤더 값에 대해 오리진의 응답을 한 번씩 캐싱합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "HeadersAllowList" : [ String, ... ],
  "Option" : String
}
```

## 속성

`HeadersAllowList`  
배포의 오리진으로 전달할 특정 헤더입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Option` 배포에서 오리진으로 전달하려는 헤더입니다. 배포는 이러한 헤더를 기반으로 콘텐츠를 캐시합니다.  
배포에 대해 다음 구성 중 하나를 사용합니다.

- ** `all` ** - 모든 헤더를 오리진으로 전달합니다.
- ** `none` ** - 기본 헤더만 전달합니다.
- ** `allow-list` ** - 매개 변수를 사용하여 지정한 헤더만 전달합니다 `HeadersAllowList` .
  *필수 항목 *여부: 아니요  
  _유형_: 문자열  
  _허용되는 값_: `all | allow-list | none`  
  _업데이트 필요 사항_: 중단 없음
