# AWS::백업::프레임워크 제어 범위

프레임워크는 하나 이상의 컨트롤로 구성됩니다. 각 컨트롤에는 고유한 컨트롤 범위가 있습니다. 제어 범위에는 하나 이상의 리소스 유형, 태그 키와 값의 조합 또는 하나의 리소스 유형과 하나의 리소스 ID의 조합이 포함될 수 있습니다. 범위를 지정하지 않으면 레코딩 그룹의 리소스가 구성에서 변경될 때 규칙에 대한 평가가 트리거됩니다.

**메모**  
특정 리소스를 모두 포함하는 컨트롤 범위를 설정하려면 `ControlScope` 를 비워 두거나 호출할 때 전달하지 마십시오 `CreateFramework`.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "ComplianceResourceIds" : [ String, ... ],
  "ComplianceResourceTypes" : [ String, ... ],
  "Tags" : [Tag, ... ]
}
```

## 속성

`ComplianceResourceIds`  
컨트롤 범위에 포함할 유일한 AWS 리소스의 ID입니다.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_최대_: `100`  
_업데이트 필요 사항_: 중단 없음

`ComplianceResourceTypes`  
컨트롤 범위에 또는 와 같은 하나 이상의 리소스 형식이 포함되어 있는지 여부를 설명합니다 `EFS` `RDS`.  
*필수 항목 *여부: 아니요  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`Tags`  
규칙에 대한 평가를 트리거하려는 AWS 리소스에 적용되는 태그 키-값 페어입니다. 최대 하나의 키-값 쌍을 제공할 수 있습니다. 태그 값은 선택 사항이지만 빈 문자열일 수 없습니다. 태그를 할당하는 구조는 다음과 같습니다`[{"Key":"string","Value":"string"}]`.  
*필수 항목 *여부: 아니요  
_유형_: 태그 목록  
_업데이트 필요 사항_: 중단 없음
