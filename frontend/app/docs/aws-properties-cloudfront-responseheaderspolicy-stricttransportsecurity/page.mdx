# AWS::CloudFront::ResponseHeadersPolicy StrictTransportSecurity

CloudFront에 HTTP 응답 헤더와 헤더의 값이`Strict-Transport-Security` 포함되어 있는지 여부를 결정합니다.

HTTP 응답 헤더에 대한 자세한 `Strict-Transport-Security` 내용은 MDN Web Docs의 Strict-Transport-Security를 참조하십시오.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "AccessControlMaxAgeSec" : Integer,
  "IncludeSubdomains" : Boolean,
  "Override" : Boolean,
  "Preload" : Boolean
}
```

## 속성

`AccessControlMaxAgeSec`  
CloudFront에서 HTTP 응답 헤더의 `max-age` 지시문 `Strict-Transport-Security` 값으로 사용하는 숫자입니다 .  
_필수 항목 여부_: 예  
_유형_: Integer  
_업데이트 필요 사항_: 중단 없음

`IncludeSubdomains`  
CloudFront에서 HTTP 응답 헤더에 지시문을 포함하는지 여부를 결정하는 부울입니다 `includeSubDomains` `Strict-Transport-Security` .  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Override`  
CloudFront가 `Strict-Transport-Security` 오리진에서 수신한 HTTP 응답 헤더를 이 응답 헤더 정책에 지정된 헤더로 재정의할지 여부를 결정하는 부울입니다.  
_필수 항목 여부_: 예  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음

`Preload`  
CloudFront에서 HTTP 응답 헤더에 지시문을 포함하는지 여부를 결정하는 부울입니다 `preload` `Strict-Transport-Security` .  
*필수 항목 *여부: 아니요  
_유형_: Boolean  
_업데이트 필요 사항_: 중단 없음
