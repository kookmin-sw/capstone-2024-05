# AWS::CloudFront::D istribution 지리적 제한

콘텐츠가 배포되는 국가를 제어하는 복합 형식입니다. CloudFront는 GeoIP 데이터베이스를 사용하여 사용자의 위치를 결정합니다 `MaxMind` . 지리적 제한을 비활성화하려면 스택 템플릿에서 Restrictions 속성을 제거합니다.

## 통사론

AWS CloudFormation 템플릿에서 이 엔터티를 선언하려면 다음 구문을 사용합니다.

### JSON (영문)

```
{
  "Locations" : [ String, ... ],
  "RestrictionType" : String
}
```

## 속성

`Locations`  
 CloudFront에서 콘텐츠를 배포(`Location`)하거나 콘텐츠를 배포하지 않으`whitelist`려는 각 국가에 대한 요소를 포함하는 복합 유형(`blacklist`)입니다().  
요소는 `Location` or 에 `blacklist` 포함할 국가의 두 글자 대문자 국가 코드 `whitelist` 입니다. `Location` 각 국가에 대해 하나의 요소를 포함합니다.  
CloudFront와 `MaxMind` 둘 다 `ISO 3166` 국가 코드를 사용합니다. 현재 국가 목록과 해당 코드는 `ISO 3166-1-alpha-2` _국제 표준화 기구_ 웹 사이트의 코드를 참조하십시오. CloudFront 콘솔에서 국가 이름과 코드가 모두 포함된 국가 목록을 참조할 수도 있습니다.  
_필수_: 조건부  
_유형_: 문자열 목록  
_업데이트 필요 사항_: 중단 없음

`RestrictionType` 국가별로 콘텐츠 배포를 제한하는 데 사용할 방법:

- `none`: 지리적 제한이 활성화되지 않았으며, 이는 콘텐츠에 대한 액세스가 클라이언트 지리적 위치에 의해 제한되지 않음을 의미합니다.
- `blacklist`: `Location` 요소는 CloudFront에서 콘텐츠를 배포하지 않으려는 국가를 지정합니다.
- `whitelist`: `Location` 요소는 CloudFront에서 콘텐츠를 배포할 국가를 지정합니다.
  _필수 항목 여부_: 예  
  _유형_: 문자열  
  _허용되는 값_: `blacklist | none | whitelist`  
  _업데이트 필요 사항_: 중단 없음
